========================================================
  새 블로그 글 쓰는 법 (github.dev — 브라우저에서)
========================================================

[열기]
  저장소 페이지에서 키보드 '.'(마침표)를 누른다.
  또는 주소창에서 github.com → github.dev 로 바꾼다.
  → 브라우저에 VSCode가 그대로 뜬다. (설치·npm 불필요)

[새 글 만들기]
  src/content/blog/<카테고리폴더>/ 안에
  파일명: 2026-06-29-제목.md  형태로 새 .md 파일 생성
  (폴더는 아무 데나 OK — 분류는 아래 category 값으로 정해짐)

  파일 맨 위에 아래를 복사해 채운다 ↓↓↓

--------------------------------------------------------
---
title: "여기에 글 제목"
date: 2026-06-29
category: "Etc"
tags: ["태그1", "태그2"]
description: "목록·검색에 보일 한 줄 요약"
permalink: "2026/06/29/my-post"
---

여기부터 본문을 마크다운으로 자유롭게 작성.

## 소제목

- 목록
- 항목

```js
console.log("코드블록도 됨");
```
--------------------------------------------------------

[필수 규칙]
  · title / date / permalink 는 반드시 있어야 함 (없으면 빌드 실패)
  · permalink = 글 주소 (확장자 없이). 예: "2026/06/29/my-post"
      → 사이트에서 /2026/06/29/my-post.html 로 열림. 다른 글과 겹치지 않게.
  · category = 사이드바 분류. 기존 값 중 하나 권장:
      Spring · Etc · CS · Springboot · Java · JS · Algorithm ·
      Backend · React-Vue · Front · C · SQL · Docker · Claude · Git · Python
  · description / tags 는 비워도 되지만 채우면 목록·검색이 깔끔.

[발행]
  좌측 'Source Control'(가지 아이콘) → 메시지 입력 →
  'Commit & Push'(체크표시) → master에 바로 커밋됨.
  → GitHub Actions가 빌드·배포 (2~4분 뒤 사이트 반영).

[비공개/보관함 글]
  이건 위 방법 말고 /tools/lock.html (암호화 도구)에서 만든다.
  (이 일반 블로그 글은 누구나 보이는 공개 글)
