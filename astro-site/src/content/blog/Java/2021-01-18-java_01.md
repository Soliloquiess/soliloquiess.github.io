## JRE와 JDK의 차이

- **JDK**(Java Development Kit): 자바용 소프트웨어 개발 키트. 자바 애플리케이션 도구를 만드는 데 필요한 라이브러리를 제공한다.
- **JRE**(Java Runtime Environment): 프로그램을 실행하는 장소. 가장 기본적인 실행 환경이다.

> 환경 변수 등록이란, 운영체제가 컴퓨터의 어떤 경로에서든 특정 파일(파일 형식으로 되어 있는 애플리케이션)을 인식할 수 있도록 하는 것이다.

정리하면 JDK는 소프트웨어를 **개발**하기 위한 도구이고, JRE는 자바 프로그램을 **실행**하기 위한 장소다. JRE가 JVM에 더 가깝다.

### 서버에는 JDK와 JRE 중 무엇을 설치할까?

→ JRE를 설치한다. 서버는 실행을 위한 환경이므로 개발용 JDK를 설치할 필요가 없다.

- 서버: JRE 설치
- 개발자: JDK 설치 (굳이 말하면 JDK가 좀 더 큰 의미)

## 타입이란?

### 머신의 메모리 할당 및 관리 (Memory Allocation & Management)

- **primitive type**: 미리 정해진 크기의 메모리 사이즈로 표현
- **reference type**: 미리 정해질 수 없는 데이터의 표현

## 변수(Variable)란?

- 타입별로 선언(declaration)한다.
- 값을 할당(assign)한 후 변경 및 사용한다.
- 로컬 변수는 디폴트값이 아니다.
- long, float, double 타입 값.

## 형 변환이란?

서로 다른 타입의 변수에 값을 할당하는 것을 말한다.

- **큰 타입 ← 작은 타입: 자동 형 변환**

```java
byte b = 2;
int i = b;
```

- **작은 타입 ← 큰 타입: 오류, 명시적 형 변환 필요**

```java
int i = -2000;
byte b = (byte)i; // -49 (value loss)
```

- **정수형은 실수형으로 자동 형 변환**

```java
long var = 100;
float fvar = var;
```

## for vs while

| --- | --- |
