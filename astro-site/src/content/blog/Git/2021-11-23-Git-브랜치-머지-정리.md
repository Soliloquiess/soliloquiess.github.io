## 머지(merge)와 페치(fetch)

- `git merge` 시 **squash 옵션**을 쓰면, rebase처럼 일일이 수정할 필요가 없다.
- `git fetch`는 전체 가지(브랜치)를 동기화한 뒤, 필요한 것만 골라 merge시킨다.

## 자주 쓰는 명령 흐름

1. 최초에는 `git clone`이 제일 좋다.
2. 원격(remote)에 참조할 브랜치(ref)가 있다고 가정한다.
3. 나머지가 필요하면 `git checkout -b dev origin/master`로 동기화한다.
4. `git checkout -b topic origin/master`
