이미 여러 IntelliJ 한글 설정 글이 있지만, 따라 해도 안 되는 경우가 많았다.
일반 Java 코드 실행 시에는 콘솔 한글이 잘 나오는데, **Tomcat으로 실행할 때만** 깨지는 현상이 반복되어 직접 해결한 방법을 기록해 둔다.

---

## 방법 1 — File Encoding을 UTF-8로 변경

`File > Settings` (단축키 `Ctrl + Alt + S`) → 좌측 트리에서 **Editor > File Encodings** 로 이동한다.
화면에 보이는 Encoding 설정들을 모두 **UTF-8**로 바꿔준다.

![20220630_112219](/assets/20220630_112219.png)

그런데 이것만으로는 여전히 아래처럼 한글이 깨지는 경우가 있었다.

![20220630_104649](/assets/20220630_104649.png)

---

## 방법 2 — Tomcat VM 옵션에 인코딩 플래그 추가 (핵심)

Tomcat 서버 설정에서 **VM options** 항목에 아래 한 줄을 추가한다.

`-Dfile.encoding=UTF-8`

![20220630_110407](/assets/20220630_110407_ye9cdj160.png)

이 옵션을 넣으면 Tomcat 실행 시 콘솔에 한글이 **정상 출력**된다.

![20220630_112433](/assets/20220630_112433.png)
