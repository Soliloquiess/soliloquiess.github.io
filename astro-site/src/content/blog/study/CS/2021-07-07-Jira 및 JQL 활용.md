## Issue Tracking Systems

**이슈(Issue)** 란 개발해야 할 항목이나 처리해야 할 작업 단위를 말한다. 개발 관련이 아니어도 해결해야 할 문제라면 이슈가 된다. 다른 프로그램에서는 **티켓(Ticket)** 이라 부르기도 한다.

![Jira 이슈 트래킹 화면](/assets/20210707_160129.png)

Jira는 사용자에게 친숙한 UI와 다양한 지표 관리 기능을 제공한다.

> 이슈 관리만 하면 되는데, 왜 이런 도구가 필요한가?

## DevOps

**DevOps** 는 개발(Development)과 운영(Operations)을 합친 개념이다.

- 개발팀과 운영팀을 분리하면 각 팀이 서로의 사정을 모를 수 있다.
- 한 팀에서 개발·운영·배포까지 처리하면 이런 문제가 해결된다.

## JQL (Jira Query Language)

| 항목 | 설명 |
|------|------|
| 정의 | Jira Issue를 구조적으로 검색하기 위해 제공하는 언어 |
| 문법 | SQL(Standard Query Language)과 유사 |
| 특징 | Jira 각 필드에 맞는 특수 예약어 제공 |
| 활용 | 쌓인 이슈를 재가공해 유의미한 데이터 도출 (Gadget, Agile Board) |
