## 왜 리액트가 필요한가

리액트를 배우는 이유는 결국 **웹앱(Web App)** 을 만들기 위해서입니다.

Vibe, Flipkart, Instagram 같은 사이트가 대표적인 웹앱입니다. 다른 페이지로 넘어가거나 글을 발행해도 새로고침 없이 부드럽게 동작해서, 모바일 앱 같은 느낌을 준다고 해서 "앱"이라고 부릅니다.

요즘은 대기업이든 중소기업이든 사이트를 새로 만들 때 웹앱 방식을 자주 채택합니다. 그 이유는 다음과 같습니다.

- 좋은 사용자 경험(UX)을 제공한다
- HTML 관리가 편해진다
- 구매 전환율이 높아진다
- 같은 문법으로 iOS / Android 모바일 앱도 만들 수 있다
- 서버 개발자의 부담이 줄어든다

이런 웹앱을 순수 자바스크립트로 만든다면, AJAX와 HTML 조작 코드를 잔뜩 짜야 합니다. 코드가 수백~수천 줄로 늘어나면서 고생하게 되죠.

그래서 **React, Angular, Vue** 같은 라이브러리를 쓰면 훨씬 짧고 간결한 코드로 개발할 수 있습니다. 그중에서도 가장 유명하고 사용자가 많은 **React** 를 배워두면 취업, 개인 프로젝트, 사업 등 여러 방면에 큰 도움이 됩니다. 요즘 웹 개발자의 사실상 기본 스펙이라고 볼 수 있습니다.

### 필요한 자바스크립트 사전지식

본격적으로 시작하기 전에 다음 자바스크립트 기초는 알고 있는 것이 좋습니다.

- `var` `let` `const`
- `function` / `return`
- `for`
- `if` / `else`
- `array`, `object`
- `addEventListener`
- 약간의 HTML / CSS

### 터미널 에러 해결

터미널에 명령어를 입력할 때 종종 에러가 발생합니다. 아래로 대부분 해결되며, 그 외에는 정확한 에러 메시지를 구글에 검색하는 것이 답입니다.

**Q1. 프로젝트 설치가 10분 이상 오래 걸려요**

네트워크 문제일 수 있습니다. 안정적인 환경에서 설치합시다.

**Q2. `npx command not found` 에러**

- 터미널을 껐다 켜보세요.
- 대부분 Node.js가 제대로 설치되지 않았거나 옛날 버전이라 그렇습니다.
- 맥북이면 brew 대신 공식 사이트에서 다운받으세요.
- 윈도우는 C 드라이브에 그대로 설치하세요.
- 리눅스는 Node.js 설치/버전 업데이트 명령어를 직접 입력하면 됩니다.

**Q3. 맥북 `permission이 없어요` 에러**

터미널에 아래처럼 `sudo`를 붙여 실행하고, 비밀번호를 물으면 맥북 비밀번호를 입력합니다.

```
sudo npx create-react-app blog
```

**Q4. 윈도우 `허가되지 않은 스크립트입니다` 에러**

윈도우 검색에서 PowerShell을 찾아 **관리자 권한으로 실행**한 뒤, 아래 명령을 대소문자 그대로 입력하고 엔터를 칩니다. 이후 선택지가 나오면 `y`를 누릅니다.

```
Set-ExecutionPolicy Unrestricted
```

이제 `npx`, `npm` 명령이 잘 동작합니다.

**Q5. `The engine "node" is incompatible with this module` 에러**

Node.js 버전이 너무 낮거나 높다는 뜻입니다. 모듈이 요구하는 버전으로 Node.js를 재설치하세요.

**Q6. 윈도우에서 여전히 터미널 명령이 안 될 때**

- 보안 프로그램(특히 AhnLab Security 류)을 꺼보세요.
- 작업 폴더를 연 다음 `파일 - PowerShell 열기 - 관리자 권한으로 열기`로 실행한 뒤 프로젝트 생성을 시도해보세요.

이외에도 에러 경우의 수가 매우 많으므로, 정확한 에러 메시지를 직접 검색하는 것이 가장 확실합니다.

> **(참고) 강의와 같은 리액트 버전으로 맞추고 싶다면**
> 1. 첨부파일을 압축 푼 뒤 그 폴더를 에디터로 엽니다.
> 2. `Terminal - New Terminal`을 열고 `npm install`을 입력하면 필요한 라이브러리가 설치됩니다.
> 3. 그대로 코딩하면 됩니다.
>
> (문제가 생기는 경우가 아니면 굳이 버전을 똑같이 맞출 필요는 없습니다.)

## JSX 문법 3가지

리액트라고 해서 어렵고 복잡하게 코드를 짜야 하는 것은 아닙니다. 기존처럼 HTML / CSS로 웹페이지를 만들어 나가면 됩니다. 다만 HTML 대신 **JSX** 라는 것을 쓰는데, 그 사용법을 알아봅시다.

리액트 프로젝트의 `App.js`로 들어갑니다. `App.js`가 메인 페이지입니다. 기본으로 채워져 있던 HTML은 싹 비우고 `<div>` 하나만 남긴 채 깨끗한 백지에서 시작합시다.

```
import 어쩌구;

function App(){
  return (
    <div className="App">
      //다지움 ㅅㄱ
    </div>
  )
}
```

> 코드를 수정했으면 **파일 저장**을 해야 미리보기 화면에 반영됩니다.

본격적으로 블로그 상단 nav를 만들어봅시다. 레이아웃은 옛 방식 그대로 `<div>`로 짜면 됩니다.

`App.js`:

```
function App(){
  return (
    <div className="App">
      <div className="black-nav">
        <h4>블로그임</h4>
      </div>
    </div>
  )
}

```

`App.css`:

```
.black-nav {
  background : black;
  width : 100%;
  display : flex;
  color : white;
  padding : 20px;
}
```

CSS는 `App.css` 파일에 넣으면 됩니다. 저장하면 검은색 상단 메뉴가 생성됩니다.

### 문법 1. class 대신 className

평소 HTML/CSS와 다른 점이 보입니다. class명을 줄 때 `class=" "`가 아니라 `className=" "`을 씁니다.

`App.js`에 작성하는 것은 사실 HTML이 아니라 **JSX** 라는 언어이기 때문입니다. 원래 리액트에서 `<div>` 하나를 만들려면 자바스크립트로 아래처럼 짜야 합니다.

```
React.createElement('div', null)
```

이렇게 하면 너무 불편하므로 JSX라는 언어를 대신 사용합니다. JSX는 HTML과 사용법이 비슷하지만 일종의 자바스크립트이기 때문에, 자바스크립트 예약어인 `class`를 그대로 쓸 수 없습니다. 그래서 `class=" "` 대신 `className`을 써야 합니다.

### 문법 2. 변수를 꽂아넣을 때는 {중괄호}

자바스크립트 변수에 담긴 자료를 HTML 중간에 꽂아 보여주고 싶을 때가 많습니다.

```
function App(){

  let post = '강남 우동 맛집';
  return (
    <div className="App">
      <div className="black-nav">
        <div>블로그임</div>
        <div>여기에 저 변수에 있던거 꽂고 싶으면?</div>
      </div>
    </div>
  )
}
```

위에서 `post` 변수에 잠깐 문자를 저장해뒀습니다.

> **변수란?** 길고 복잡한 자료를 잠깐 한 단어에 저장해서 쓸 수 있는 문법입니다. `var` `let` `const` 키워드로 만들면 됩니다.

옛날 자바스크립트라면 `document.getElementById().innerHTML = ??` 같은 식이었겠지만, 리액트에서는 **중괄호** 로 더 쉽게 꽂을 수 있습니다.

```
function App(){

  let post = '강남 우동 맛집';
  return (
    <div className="App">
      <div className="black-nav">
        <div>블로그임</div>
        <div>{ post }</div>
      </div>
    </div>
  )
}
```

중괄호 안에 데이터 바인딩하고 싶은 변수명만 담으면, 미리보기 화면에 아래처럼 출력됩니다.

```
<div>강남 우동 맛집</div>
```

중괄호는 텍스트뿐 아니라 `href`, `id`, `className`, `src` 등 여러 HTML 속성에도 사용할 수 있습니다.

```
function App(){

  var data = 'red';
  return (
    <div className="App">
      <div className="black-nav">
        <div>개발 blog</div>
        <div className={data}>안녕하세요</div>
      </div>
    </div>
  )
}
```

위처럼 쓰면 아래처럼 됩니다.

```
<div className="red">
```

> 변수에 있던 값을 HTML에 꽂아넣는 작업을 **데이터 바인딩(data binding)** 이라고 합니다.

### 문법 3. style 속성 넣는 법

일반 HTML에서 `<div style="color : blue">`처럼 쓰는 인라인 스타일은, JSX에서는 `style={ }` 안에 **객체(object) 자료형** 으로 넣어야 합니다.

```
<div style={ {color : 'blue', fontSize : '30px'} }> 글씨 </div>
```

- `{ 속성명 : '속성값' }` 형태로 넣습니다.
- `font-size`처럼 속성명에 대시(`-`)를 쓸 수 없습니다. 대시 대신 단어를 붙여 쓰되, 뒷 단어의 첫 글자를 대문자로 바꿉니다(예: `fontSize`).

CSS 파일을 열기 귀찮을 때 쓰면 됩니다.

## 중요한 데이터는 변수 말고 state에 담는다

![블로그 글 목록 3개 레이아웃 예시](/assets/20220426_102640.png)

> **오늘의 숙제**
> 위 사진처럼 블로그 글 목록 3개를 HTML 레이아웃으로 만들고, 제목 3개를 state에 저장한 뒤 HTML에 데이터 바인딩해보세요.

중요한 데이터를 저장할 땐 변수 대신 **state** 를 만들어 씁니다. state가 왜 필요한지, 어떻게 만드는지 알아봅시다.

먼저 블로그 글 레이아웃을 만들어봅시다.

```
function App(){

  let posts = '강남 우동 맛집';
  return (
    <div className="App">
      <div className="black-nav">
        <div>개발 blog</div>
      </div>
      <div className="list">
        <h4>글제목</h4>
        <p>2월 17일 발행</p>
      </div>
    </div>
  )
}
```

`App.css`:

```
div {
  box-sizing : border-box
}
.list {
  text-align : left;
  padding-left : 20px;
  border-bottom : 1px solid grey;
}
```

이렇게 하면 메인 페이지에 블로그 글 목록이 하나 생성됩니다.

### state 만드는 법

리액트에선 변수 말고 state를 만들어서 데이터를 저장할 수 있습니다.

```
import { useState } from 'react';
import './App.css'

function App(){

  let [a,b] = useState('남자 코트 추천');
  let posts = '강남 우동 맛집';
  return (
    <div className="App">
      <div className="black-nav">
        <div>개발 blog</div>
      </div>
      <div className="list">
        <h4>글제목</h4>
        <p>2월 17일 발행</p>
        <hr/>
      </div>
    </div>
  )
}
```

- 맨 윗줄에 `import { useState } from 'react'`를 작성합니다.
- 원하는 곳에서 `useState('보관할 자료')`를 쓰면 state에 자료를 잠깐 저장할 수 있습니다.
- `let [a, b] = useState('남자 코트 추천')`에서 `a` 자리에 state 이름을 자유롭게 작명한 뒤 나중에 사용합니다.

#### Spoiler: 저 `[a, b]` 문법은 뭔가요?

자바스크립트 **destructuring(구조 분해)** 문법입니다. array 안의 데이터를 변수로 쉽게 꺼낼 때 씁니다.

예를 들어 `['Kim', 20]` 같은 array에서 각각 변수에 담고 싶으면, 예전에는 이렇게 했습니다.

```
let array = ['Kim', 20];

let name = array[0];
let age = array[1];
```

요즘은 이렇게 합니다.

```
let [name, age] = ['Kim', 20]
```

그럼 `name = 'Kim'`, `age = 20`이라는 변수가 생성됩니다. 등호를 여러 번 쓸 필요 없이 왼쪽과 오른쪽 형식을 맞춰주면 자동으로 변수가 생성됩니다.

`useState()`를 쓰면 그 자리에 `[데이터1, 데이터2]` 형태의 array가 남습니다.

- **데이터1**: `'남자 코트 추천'` 같은 저장한 자료
- **데이터2**: state 변경을 도와주는 함수

그래서 아래처럼 destructuring으로 각각 변수로 빼는 것일 뿐입니다.

```
let [a, b] = useState('남자 코트 추천')
```

변수명을 직관적으로 짓고 HTML에 데이터 바인딩해보면 실제로 출력됩니다.

```
function App(){

  let [글제목, b] = useState('남자 코트 추천');
  let posts = '강남 우동 맛집';
  return (
    <div className="App">
      <div className="black-nav">
        <div>개발 blog</div>
      </div>
      <div className="list">
        <h4>{ 글제목 }</h4>
        <p>2월 17일 발행</p>
      </div>
    </div>
  )
}
```

저장하면 글제목 부분에 `'남자 코트 추천'`이 나옵니다. 결론은, 리액트에선 일반 변수 대신 state로도 자료를 잠깐 저장해둘 수 있다는 것입니다.

### 왜 변수 말고 state를 쓰는가

state도 용도는 변수와 똑같이 "자료를 잠깐 보관"하는 것인데, 왜 굳이 state를 쓸까요?

> **state는 값이 바뀌면, 그 state를 쓰는 HTML도 자동으로 재렌더링됩니다.**

```
function App(){
  let post = '강남 우동 맛집'

  return (
    <h4>{ post }</h4>
  )
}
```

위처럼 일반 변수 `post`를 데이터 바인딩한 상태에서 값을 `'강남 우동 맛집' → '강남 고기 맛집'`으로 바꿨다고 합시다. 이 변경을 HTML에 반영하려면, 순수 자바스크립트에서는 "변수 바뀌었으니 HTML도 고쳐줘"라고 직접 코드를 짜야 합니다.

```
function App(){
  let [글제목, b] = useState('남자 코트 추천');

  return (
    <h4>{ 글제목 }</h4>
  )
}
```

반면 state는 그럴 필요가 없습니다. `'남자 코트 추천' → '여자 코트 추천'`으로 바꾸면, 개입하지 않아도 HTML이 자동으로 바뀝니다. state가 포함된 HTML을 자동으로 재렌더링해주기 때문입니다.

덕분에 UI 기능 개발이 편리해지고, 사이트가 스무스하게 동작합니다.

> **Q. 블로그 로고처럼 안 바뀌는 데이터도 state로 만들까요?**
> 그래도 되지만, 로고명은 거의 바뀌지 않으니 state의 장점(자동 재렌더링)이 의미가 없습니다. 안 바뀌는 데이터는 그냥 변수에 둬도 됩니다.

#### 정리

- **자주 변경될 것 같은 데이터** → state에 저장 후 HTML에 데이터 바인딩 (예: 상품명, 글제목, 가격)
- 변경할 일이 없는 데이터 / HTML 표기가 불필요한 데이터 → 그냥 변수에 저장

지금은 변수와 state의 차이가 잘 안 느껴질 수 있는데, 다음 장에서 state를 변경하는 법을 배우면 진짜 차이를 체감할 수 있습니다.

## 버튼 기능 개발 & state 변경하는 법

> **오늘의 숙제**
> 버튼을 누르면 첫 글 제목이 '여자 코트 추천'으로 바뀌는 기능을 만들어봅시다.

버튼을 눌렀을 때 자바스크립트를 실행시키는 법, 그리고 state를 수정하는 법을 알아봅시다.

먼저 저번 숙제(글 목록 3개)의 답입니다.

```
function App(){

  let [글제목, b] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
    <div className="App">
      <div className="black-nav">
        <div>개발 blog</div>
      </div>
      <div className="list">
        <h4>{ 글제목[0] }</h4>
        <p>2월 17일 발행</p>
      </div>
      <div className="list">
        <h4>{ 글제목[1] }</h4>
        <p>2월 17일 발행</p>
      </div>
      <div className="list">
        <h4>{ 글제목[2] }</h4>
        <p>2월 17일 발행</p>
      </div>
    </div>
  )
}
```

`useState()`를 3번 써서 state를 3개 만들어도 되지만, 귀찮으면 array 자료 하나로 묶을 수 있습니다. `[ ]` 대괄호 안에 문자들을 콤마로 구분해 넣으면 array가 됩니다.

array에서 원하는 항목 하나만 꺼내려면 `array이름[인덱스]`를 씁니다.

- `글제목[0]` → 0번째 자료
- `글제목[1]` → 1번째 자료

### 콘솔창의 warning은 무시해도 되나?

개발 중 노란색 **warning** 메시지가 뜰 때가 있습니다. 빨간 **error** 는 해결해야 하지만, warning은 대체로 무시해도 됩니다. 대부분 ESLint가 "변수 만들었는데 안 쓰네요" 같은 코딩 관습을 잔소리하는 것입니다.

초보 때는 일단 코드를 실행해보는 게 더 중요하므로, JS 파일 최상단에 아래 주석을 넣으면 ESLint를 잠시 끌 수 있습니다.

```
/*eslint-disable*/
```

### 좋아요 버튼 만들기

버튼을 누르면 좋아요 개수가 1씩 증가하는 기능을 만들어봅시다. 먼저 레이아웃입니다.

```
<h4> { 글제목[0] } <span>👍</span> 0 </h4>
```

좋아요 개수가 지금은 `0`으로 하드코딩되어 있는데, "자주 바뀔 데이터는 state로" 배웠으니 좋아요 개수를 state로 만들어봅시다.

#### state 만들기

```
function App(){

  let [따봉] = useState(0);
  return (
     <h4> { 글제목[0] } <span>👍</span> { 따봉 }</h4>
  )
}
```

`useState()`를 쓰고 왼쪽에 작명만 잘하면 끝입니다. 왼쪽에 2개를 작명할 수 있지만 귀찮으면 1개만 해도 됩니다.

#### onClick 사용법

HTML을 클릭했을 때 코드를 실행하는 법을 알아봅시다.

일반 HTML:

```
<div onclick="실행할 자바스크립트~~~">
```

JSX:

```
<div onClick={실행할함수}>
```

JSX에서는 다음 세 가지를 기억하세요.

1. `Click`의 C가 대문자입니다.
2. `{ }` 중괄호를 사용합니다.
3. 그냥 코드가 아니라 **함수** 를 넣어야 잘 동작합니다.

`onClick={ }` 안에는 미리 만든 함수명을 적거나, 함수를 그 자리에서 바로 만들어 넣으면 됩니다.

> **함수란?** 긴 코드를 짧은 단어 하나로 축약하는 문법입니다. `function 작명(){ 긴코드 }`로 만듭니다.

```
function App(){

  function 함수임(){
    console.log(1)
  }
  return (
     <div onClick={함수임}> 안녕하세요 </div>
  )
}
```

`<div>`를 클릭하면 함수 안의 코드가 실행됩니다. 함수 만들기가 귀찮으면 그 자리에서 직접 만들어도 됩니다.

```
<div onClick={ function(){ 실행할코드 } }>
<div onClick={ () => { 실행할코드 } }>
```

> **Q. 함수 만들 때 작명 안 해도 되나요?**
> A. 나중에 다시 안 쓸 거면 작명하지 않아도 됩니다. `() => {}`는 `function(){}`과 거의 같은 함수 만들기 문법입니다.

### (매우 중요) state 변경하는 법

좋아요 버튼을 누르면 `따봉` state를 +1 하고 싶습니다. 그냥 `변수 = 변수 + 1`을 쓰면 될 것 같지만, **이것은 변수가 아니라 state** 입니다.

```
function App(){

  let [ 따봉 ] = useState(0);
  return (
    <h4> { 글제목[0] } <span onClick={ ()=>{ 따봉 = 따봉 + 1 } } >👍</span> { 따봉 }</h4>
  )
}
```

state는 반드시 **state 변경함수** 로 변경해야 합니다. 안 그러면 HTML 재렌더링이 안 됩니다.

```
let [ 따봉, 따봉변경 ] = useState(0);
```

`useState()` 왼쪽의 두 번째 작명이 바로 state 변경함수입니다. 사용법은 다음과 같습니다.

```
따봉변경(새로운state)
```

state 변경함수는 `( )` 안에 넣은 값으로 기존 state를 **갈아치웁니다**.

- `따봉변경(1)` → `따봉`이 `1`로 변경
- `따봉변경(100)` → `따봉`이 `100`으로 변경

> `따봉변경( 따봉 = 따봉 + 1 )`처럼 쓰면 안 됩니다. 깔끔하게 새로운 state 값만 넣으세요.

#### 좋아요 누르면 +1 하기

```
function App(){

  let [ 따봉, 따봉변경 ] = useState(0);
  return (
      <h4> { 글제목[0] } <span onClick={ ()=>{ 따봉변경(따봉 + 1) } } >👍</span> { 따봉 }</h4>
  )
}
```

기존 `따봉` state에 1을 더한 값을 `따봉변경` 함수에 넣었습니다. 버튼을 누를 때마다 그 값으로 대체되어 1씩 증가합니다.

**오늘의 정리**

1. 클릭 시 무언가 실행하려면 `onClick={함수}` 형태로 이벤트 핸들러를 답니다.
2. state를 변경하려면 반드시 state 변경함수를 사용합니다. state 변경함수는 `( )` 안에 입력한 값으로 기존 state를 갈아치웁니다.

## array / object state 변경하는 법

먼저 글 수정 버튼을 만들어봅시다.

```
function App(){

  let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
      <button onClick={ ()=>{ ??? } }> 수정버튼 </button>
  )
}
```

버튼을 누르면 첫 글이 수정되게 하려면, `???` 자리에 무엇을 넣어야 할까요?

```
function App(){

  let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
    <button onClick={ ()=>{
      글제목변경(['여자코트 추천', '강남 우동맛집', '파이썬 독학'])
    } }> 수정버튼 </button>
  )
}
```

state 변경함수는 `( )` 안에 넣은 값으로 기존 state를 통째로 갈아치우므로, 바뀐 array 전체를 넣으면 됩니다.

> **Q. `글제목변경('여자코트 추천')`은 왜 안 되나요?**
> A. 기존 state를 통째로 갈아치우기 때문입니다. array 전체를 새 array로 바꿔야지, 글 하나만 넣으면 안 됩니다.

### 확장성 있게 다시 짜기

위 코드는 글이 100개라면 onClick 안이 매우 길어집니다. 그러니 기존 state를 복붙하지 말고, **기존 state의 첫 글만 살짝 바꿔서** 넘기는 식으로 만들어봅시다.

```
function App(){

  let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
    <button onClick={ ()=>{
      글제목[0] = '여자코트 추천';
      글제목변경(글제목)
    } }> 수정버튼 </button>
  )
}
```

array의 X번째 항목을 변경하려면 `array자료[X] = '바꿀값'`을 씁니다.

하지만 array/object 자료를 다룰 때는 **원본을 직접 조작하지 말고 기존 값을 보존** 하는 것이 좋은 관습입니다(원본이 나중에 필요할 수 있으므로). 그래서 복사본을 만들어 조작합니다.

```
function App(){

  let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
    <button onClick={ ()=>{
      let copy = 글제목;
      copy[0] = '여자코트 추천';
      글제목변경(copy)
    } }> 수정버튼 </button>
  )
}
```

그런데 이렇게 하면 수정 버튼을 눌러도 바뀌지 않습니다. 아래처럼 `[...글제목]`을 써야 제대로 동작합니다.

```
function App(){

  let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

  return (
    <button onClick={ ()=>{
      let copy = [...글제목];
      copy[0] = '여자코트 추천';
      글제목변경(copy)
    } }> 수정버튼 </button>
  )
}
```

왜 그런지 동작 원리를 알아봅시다.

### state 변경함수 동작 원리

state 변경함수는 먼저 `기존state === 신규state`를 검사합니다. **같으면 변경을 해주지 않습니다.**

`let copy = 글제목;`으로 만든 `copy`는 기존 state와 "같다"고 판단되어 변경이 일어나지 않은 것입니다. 안에 든 자료(`'남자코트 추천'` vs `'여자코트 추천'`)는 다른데 왜 같다고 할까요?

### array / object 동작 원리

![array/object 자료는 RAM에 저장되고 변수엔 화살표만 담긴다](/assets/20220426_112733.png)

**1. array/object를 만들면, 자료는 RAM에 저장되고 변수엔 "화살표"만 담깁니다.**

`let arr = [1,2,3]`을 만들면 `[1,2,3]`은 RAM(가상공간)에 저장되고, `arr` 변수에는 그 자료가 어디 있는지 가리키는 화살표만 담깁니다.

**2. array/object를 복사하면 화살표가 복사됩니다.**

```
let data1 = [1,2,3];
let data2 = data1;   //복사문법임
```

이때 `data1`과 `data2`는 각자 `[1,2,3]`을 따로 저장하는 게 아니라 **똑같은 값을 공유** 합니다. 화살표만 복사되어 둘 다 같은 자료를 가리키기 때문입니다. 그래서 한쪽을 바꾸면 다른 쪽도 같이 바뀝니다.

**3. 같은 화살표를 가진 변수끼리는 등호로 비교해도 같다고 나옵니다.**

```
let data1 = [1,2,3];
let data2 = data1;  //복사
data2[0] = 1000;  //data2 내부 변경
console.log(data2 === data1)   //true 나올듯
```

> 자세한 내용은 `javascript reference data type`으로 검색해 추가 학습할 수 있습니다.

그래서 아래 코드는 `copy`와 기존 `글제목`이 같은 화살표라서 state 변경이 안 됩니다.

```
let copy = 글제목;
copy[0] = '여자코트 추천';
글제목변경(copy)
```

반면 아래는 화살표가 달라지는 문법이라 잘 동작합니다.

```
let copy = [...글제목];
copy[0] = '여자코트 추천';
글제목변경(copy)
```

#### 점 3개(`...`)는 뭔가요?

**spread operator(전개 연산자)** 라는 문법입니다. array나 object 왼쪽에 붙이며, 의미는 "괄호를 벗겨주세요"입니다.

`...[1,2,3]`이라고 쓰면 그 자리에 `1,2,3`이 남습니다. 또 다른 용도는 array/object를 복사할 때입니다.

```
let data1 = [1,2,3];
let data2 = [...data1];
console.log(data1 === data2) //false 나올듯
```

`[...data1]`은 자료를 괄호 벗긴 뒤 새 array로 다시 만들어주므로, 화살표가 다른 **완전히 독립적인 복사본** 이 생성됩니다(`false`). object도 마찬가지입니다. 이런 독립적인 사본을 shallow copy 또는 deep copy라고 합니다.

#### 오늘의 정리

리액트에서 array/object state를 수정하려면 독립적인 복사본을 만들어 수정하는 것이 좋습니다.

- `[...기존state]`
- `{...기존state}`

이렇게 하면 독립적인 카피가 생성됩니다.

### 응용 문제: 가나다순 정렬 버튼

버튼을 누르면 글제목이 가나다순으로 정렬되는 기능을 만들어봅시다. HTML 정렬은 신경 쓸 필요 없고, 글제목 state만 정렬하면 됩니다(state가 바뀌면 HTML은 자동 재렌더링되니까요).

> **Q. array 정렬하는 법을 모르겠어요**
> A. 모르는 것은 구글에 찾아봐야 합니다.

```
function App(){

 let [글제목, 글제목변경] = useState( ['남자코트 추천', '강남 우동맛집', '파이썬 독학'] );

 return (
   <button onClick={ ()=>{
     let copy = [...글제목];
     copy.sort();
     글제목변경(copy)
   } }> 정렬버튼 </button>
 )
}
```

복사본을 만들어 `.sort()`로 정렬한 뒤 state 변경함수에 넣으면 됩니다.

> 따라 적는 것은 학습이 아니라 타자 연습일 뿐입니다. 직접 짜보세요.

## Component: 많은 div를 한 단어로 줄이기

모달창처럼 뜨는 상세페이지를 만들면서 **Component** 문법을 알아봅시다. 먼저 모달창 UI의 레이아웃부터 만듭니다.

```
<div className="modal">
  <h4>제목</h4>
  <p>날짜</p>
  <p>상세내용</p>
</div>
```

```
.modal{
  margin-top : 20px;
  padding : 20px;
  background : #eee;
  text-align : left;
}
```

(CSS는 CSS 파일에 넣습니다.)

### return 안에는 태그가 하나여야 한다

```
return(
  <div></div>
  <div></div>
)
```

HTML 코드를 짤 때 유의점이 있습니다. `return ( )` 안에 두 개의 태그를 나란히 적을 수 없습니다. `return ( )` 내부는 **하나의 태그로 시작해서 하나의 태그로 끝나야** 합니다.

```
return(
  <div>
    <div></div>
    <div></div>
  </div>
)
```

그래서 `<div>` 두 개를 나란히 쓰려면 하나의 `<div>`로 감싸야 합니다. 의미 없는 `<div>`가 싫으면 `<> </>`로 감싸도 되는데, 이를 **fragment** 문법이라고 합니다.

### Component 문법

리액트는 긴 HTML을 한 단어로 깔끔하게 치환하는 문법을 제공합니다. 이것을 **Component** 라고 합니다. 방금 만든 모달창을 한 단어로 줄여봅시다.

```
function App (){
  return (
    <div>
      (생략)
      <Modal></Modal>
    </div>
  )
}

function Modal(){
  return (
    <div className="modal">
      <h4>제목</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

줄이는 법은 다음과 같습니다.

1. `function`으로 함수를 하나 만들고 작명합니다.
2. 그 함수의 `return ()` 안에 축약할 HTML을 담습니다.
3. 원하는 곳에서 `<함수명></함수명>`을 쓰면 축약한 HTML이 등장합니다.

이렇게 축약한 HTML 덩어리를 **Component** 라고 부릅니다. `<div>` 뭉텅이보다 `<Modal>` 한 단어가 레이아웃을 파악하기 쉬워 나중에 관리하기 좋습니다.

#### Component 만들 때 주의점

1. Component는 보통 **영어 대문자** 로 작명합니다.
2. `return ()` 안엔 태그가 평행하게 여러 개 들어갈 수 없습니다.
3. `function App(){}` 내부에서 만들면 안 됩니다. `App`도 컴포넌트이므로, 컴포넌트 안에 컴포넌트를 만드는 셈이 됩니다.
4. `<컴포넌트></컴포넌트>` 또는 `<컴포넌트/>` 둘 다 가능합니다.

#### arrow function으로도 만들 수 있다

```
function Modal(){
  return ( <div></div> )
}

let Modal = () => {
  return ( <div></div>)
}
```

함수 만드는 문법은 하나가 아니므로 위처럼 써도 됩니다. `let` 대신 `const`를 쓰기도 합니다. 마음에 드는 것을 쓰세요.

#### 어떤 HTML을 Component로 만드는 게 좋을까

정해진 기준은 없지만, 관습적으로 다음과 같은 부분을 Component로 만듭니다.

- 사이트에 반복해서 출현하는 HTML 덩어리
- 내용이 자주 변경될 것 같은 HTML 부분
- 다른 페이지를 만들고 싶을 때 그 페이지의 HTML 전체
- 팀원과 협업 시 작업 분배 단위

함수를 쓰는 이유(긴 코드 축약, 재사용, 작은 기능으로 분리)와 컴포넌트의 용도는 똑같습니다.

#### Component의 단점

HTML을 깔끔하게 하려고 Component를 수백 개 만들면 그 자체로 관리가 힘듭니다. 또한 `function Modal` 안에서 부모의 `글제목` state를 그냥 `{글제목}`으로 쓰면 동작하지 않습니다. 자바스크립트에선 한 함수의 변수를 다른 함수가 마음대로 쓸 수 없기 때문입니다.

이를 해결하려면 **props** 라는 문법으로 state를 `<Modal>`까지 전해줘야 합니다. 그러니 초보처럼 모든 것을 Component로 쪼개지 말고, 꼭 필요한 곳만 쪼개는 것이 좋습니다.

## 동적인 UI 만드는 법 (모달창 만들기)

모달창 기능을 어떻게 만드는지 알아봅시다. 단순히 따라 치게 가르치면 평생 모달창밖에 못 만들게 되므로, **"동적인 UI를 알아서 만드는 3-step"** 을 익혀봅시다.

> **동적인 UI란?** 유저 조작 시 형태가 바뀌는 UI입니다. 모달창, 탭, 서브메뉴, 툴팁, 경고문 등.

### 동적인 UI 만드는 3-step

1. HTML/CSS로 미리 UI 디자인을 다 해놓는다.
2. UI의 현재 상태를 state로 저장한다.
3. state에 따라 UI가 어떻게 보일지 조건문 등으로 작성한다.

전에 만든 `<Modal>`이 글제목을 누르면 등장하는 기능을 만들어봅시다.

**step 1. HTML/CSS로 미리 디자인** — 저번에 만들어놨으니 패스합니다.

**step 2. UI의 현재 상태를 state로 저장**

```
let [modal, setModal] = useState(false);
```

> 영어로 작명할 때 state 변경함수는 앞에 `set`을 붙이는 게 관습입니다.

state에 넣을 자료는 현재 모달창 상태(열림/닫힘)만 표현할 수 있으면 됩니다.

```
let [modal, setModal] = useState('닫힘');
let [modal, setModal] = useState(0);
```

`'닫힘'/'열림'`, `0/1` 등 아무렇게나 표현해도 되지만, 여기서는 `true/false`(참/거짓을 담는 boolean)로 표현합니다.

**step 3. state에 따라 UI가 어떻게 보일지 작성**

```
function App (){

  let [modal, setModal] = useState(false);
  return (
    저 state가 true면 <Modal></Modal>
    false면 아무것도 보여주지마세요
  )
}
```

"이 경우엔 이렇게 해주세요"라고 짜려면 `if` 조건문을 씁니다.

### JSX에서 조건문 쓰는 법

조건문은 보통 `if / else`를 쓰지만, **JSX 중괄호 안에서는 `if/else`를 바로 쓸 수 없습니다.** 대신 **삼항연산자** 는 사용 가능합니다.

```
조건식 ? 조건식 참일 때 실행할 코드 : 조건식 거짓일 때 실행할 코드
```

예시:

```
3 > 1 ? console.log('맞음') : console.log('아님')
```

`3 > 1`이 참이므로 `'맞음'`이 출력됩니다. 이를 모달창에 적용합니다.

```
function App (){

  let [modal, setModal] = useState(false);
  return (
    <div className="app">
      (생략)
      {
        modal == true ? <Modal></Modal> : null
      }
    </div>
  )
}
```

> `null`은 아무 HTML도 남기지 않을 때 쓰는 자료입니다(텅 비었다는 뜻).

### 글제목 누르면 모달창 띄우기

state 기본값이 `false`라 지금은 모달창이 안 보입니다. 글제목을 클릭하면 모달창이 보이게 하려면, "글제목 누르면 모달 HTML을 보여주세요"가 아니라 **"글제목 누르면 state를 조절해주세요"** 로 짜면 됩니다(state가 바뀌면 HTML은 자동 재렌더링되니까요).

```
function App (){

  let [modal, setModal] = useState(false);
  return (
    <div>
      (생략)
      <button onClick={ ()=>{ setModal(true) } }> {글제목[0]} </button>
      {
         modal == true ? <Modal></Modal> : null
      }
    </div>
  )
}
```

글제목을 누르면 `modal` state가 `true`로 바뀌어 모달창이 보입니다.

![리액트 UI 만들기 = 스위치와 전등 비유](/assets/20220426_124346.png)

리액트의 UI 만들기는 **스위치와 전등** 에 비유할 수 있습니다.

1. 전등(`<Modal>`)을 예쁘게 달아놓고
2. 스위치(state)와 연결하고
3. 스위치를 on하면 불이 켜지고, off하면 꺼지도록 만들어둡니다.

나중에는 스위치(state) 조작만 하면 됩니다. 순수 자바스크립트는 HTML을 직접 조작했지만, 리액트에서는 HTML을 직접 조작하지 않으므로 사고방식을 좀 바꿔야 합니다.

> **오늘의 숙제**
> 글제목을 한 번 더 누르면 모달창이 닫히게 만들어보세요.

### 숙제 풀이: 토글 기능

글제목 클릭 시 다음처럼 동작하면 됩니다.

- state가 `true`면 `setModal(false)`
- state가 `false`면 `setModal(true)`

`if` 조건문이나 삼항연산자를 써도 되지만, 더 간단하게 `setModal(!modal)`을 쓸 수 있습니다.

```
function App (){

  let [modal, setModal] = useState(false);
  return (
    <div>
      (생략)
      <button onClick={ ()=>{ setModal(!modal) } }> {글제목[0]} </button>
      {
         modal == true ? <Modal></Modal> : null
      }
    </div>
  )
}
```

`!`(느낌표)는 우측 자료를 반대로 바꿔줍니다. `!true`는 `false`, `!false`는 `true`이므로 토글 기능이 완성됩니다.

## map: 많은 div를 반복문으로 줄이기

> **오늘의 숙제**
> 지금은 좋아요 버튼을 누르면 모든 글의 좋아요 개수가 동시에 증가합니다. 각각 개별적으로 기록하려면 어떻게 해야 할까요?

똑같은 HTML이 반복 출현하면 반복문으로 생성할 수 있습니다. 그런데 `for` 반복문은 JSX 중괄호 안에서 쓸 수 없어서 **`map()`** 을 대신 사용합니다.

### 자바스크립트 map 함수 사용법

모든 array 우측엔 `map()`을 붙일 수 있습니다.

**기능 1. array의 자료 개수만큼 안의 코드를 반복 실행**

```
var 어레이 = [2,3,4];
어레이.map(function(){
  console.log(1)
});
```

위 코드는 `console.log(1)`을 3번 실행합니다.

**기능 2. 콜백함수의 파라미터는 array 안의 자료를 하나씩 전달**

```
var 어레이 = [2,3,4];
어레이.map(function(a){
  console.log(a)
});
```

> 소괄호 안에 있는 함수를 **콜백함수** 라고 합니다.

위 코드는 `2, 3, 4`를 차례로 출력합니다.

**기능 3. `return` 오른쪽 값을 모아 새 array로 남긴다**

```
var 어레이 = [2,3,4];
var newArray = 어레이.map(function(a){
  return a * 10
});
console.log(newArray)
```

`return`한 값을 모아 array로 담고, `map()`을 쓴 자리에 남깁니다. `newArray`는 `[20, 30, 40]`이 출력됩니다.

### JSX 안에서 HTML 반복 생성하기

```
function App (){
  return (
    <div>
      {
        [1,2,3].map(function(){
          return ( <div>안녕</div> )
        })
      }
    </div>
  )
}
```

위 코드는 `<div>안녕</div>`을 3개 생성합니다.

**원리:** `[1, 2, 3]`에 map을 붙였으니 자료 개수(3)만큼 내부를 실행하고, 안에 `return <div>안녕</div>`이 있으므로 array에 3번 담깁니다.

```
[ <div>안녕</div>, <div>안녕</div>, <div>안녕</div> ]
```

이를 HTML로 미리보기 띄우면 아래처럼 보입니다.

```
<div>안녕</div>
<div>안녕</div>
<div>안녕</div>
```

### 글제목 3개를 map으로 축약

지금 프로젝트에서 `<div className="list">`가 3번 반복되고 있습니다. 이것도 map으로 축약할 수 있습니다(`글제목` state도 array라서 map을 붙일 수 있습니다).

```
function App (){
  return (
    <div>
      (생략)
      {
        글제목.map(function(){
          return (
          <div className="list">
            <h4>{ 글제목[0] }</h4>
            <p>2월 18일 발행</p>
          </div> )
        })
      }
    </div>
  )
}
```

**결론:** 비슷한 HTML이 여러 개 필요하면 `map()` 안에 담으면 됩니다.

### 반복마다 다른 제목 넣기

위 코드는 `<h4>{ 글제목[0] }</h4>`만 3번 반복되어 같은 제목만 나옵니다. 반복할 때마다 다른 제목(`글제목[0]`, `글제목[1]`, `글제목[2]`)을 넣으려면 map의 파라미터를 활용합니다.

```
function App (){
  return (
    <div>
      (생략)
      {
        글제목.map(function(a){
          return (
          <div className="list">
            <h4>{ a }</h4>
            <p>2월 18일 발행</p>
          </div> )
        })
      }
    </div>
  )
}
```

파라미터 `a`는 map이 반복될 때마다 array 안의 데이터 하나하나를 의미합니다. 반복할 때 차례로 `'남자 코트추천'`, `'강남 우동 맛집'` 등이 되어 `<h4>` 안에 들어갑니다.

#### 두 번째 파라미터(인덱스) 활용

```
function App (){
  return (
    <div>
      (생략)
      {
        글제목.map(function(a, i){
          return (
          <div className="list">
            <h4>{ 글제목[i] }</h4>
            <p>2월 18일 발행</p>
          </div> )
        })
      }
    </div>
  )
}
```

`map(function(a, i){ ... })`에서 파라미터는 2개까지 작명할 수 있습니다.

- 첫째 `a`: array 안에 있던 자료
- 둘째 `i`: 0부터 1씩 증가하는 정수 (`0, 1, 2 ...`)

### 반복문 안의 HTML에 onClick 달기

```
function App (){
  return (
    <div>
      (생략)
      {
        글제목.map(function(a, i){
          return (
          <div className="list">
            <h4 onClick={()=>{ 따봉변경(따봉+1) }}>{ 글제목[i] }</h4>
            <p>2월 18일 발행</p>
          </div> )
        })
      }
    </div>
  )
}
```

반복문 안 HTML에 `onClick`을 달아도 문제없습니다. 다만 지금은 아무 글제목을 클릭해도 따봉 개수가 다 똑같이 증가하는데, 이는 오늘 숙제로 해결합니다.

> **(참고) key 속성**
> map으로 반복 생성한 HTML엔 `key={i}` 속성을 추가해야 합니다. 그래야 리액트가 각 `<div>`를 구분할 수 있습니다. 없으면 warning이 뜹니다.
> ```
> <div className="list" key={i}>
> ```

> **(참고) 일반 for 반복문을 쓰고 싶다면**
> 1. HTML을 담을 array를 하나 만듭니다.
> 2. `for` 반복문을 돌립니다.
> 3. 반복할 때마다 array에 `<div>`를 하나씩 추가합니다.
> 4. 원하는 곳에서 `{array자료}`를 사용합니다.

```
function App (){

  var 어레이 = [];
  for (var i = 0; i < 3; i++) {
    어레이.push(<div>안녕</div>)
  }
  return (
    <div>
      { 어레이 }
    </div>
  )
}
```

위 코드도 `<div>안녕</div>`을 3개 생성합니다. `for`는 JSX 안에서 쓸 수 없어 바깥에서 처리하는 것입니다. 귀찮으면 map을 쓰세요.

### 숙제 풀이: 따봉 개수 개별 증가

지금은 따봉 state가 하나뿐이라 모든 글이 그것을 공유합니다. state를 3개 만들어 각 글에 넣을 수도 있습니다.

```
let [따봉, 따봉변경] = useState(0);
let [따봉1, 따봉변경1] = useState(0);
let [따봉2, 따봉변경2] = useState(0);
```

하지만 글이 10개면 10줄을 써야 하므로 귀찮습니다. array를 활용하면 한 변수에 자료 3개를 담을 수 있습니다.

```
let [따봉, 따봉변경] = useState([0,0,0]);
```

JSX에서 따봉을 보여줄 땐 `따봉[i]`로 보여줍니다.

```
{
   글제목.map(function(a, i){
      return (
        <div className="list">
          <h4>{ 글제목[i] }
            <span onClick={()=>{ ??? }}>👍</span> {따봉[i]}
          </h4>
          <p>2월 18일 발행</p>
        </div> )
  })
}
```

`onClick`에는 무엇을 넣을까요? 클릭 시 해당 인덱스의 따봉만 +1 하면 됩니다.

- 0번째 버튼 클릭 → `따봉[0] + 1`
- 1번째 버튼 클릭 → `따봉[1] + 1`
- 2번째 버튼 클릭 → `따봉[2] + 1`

`따봉`은 state이므로 state 변경함수를 써야 하고, array이므로 복사본부터 만들어 수정해야 합니다.

```
<h4>
  { 글제목[i] }
   <span onClick={()=>{
      let copy = [...따봉];
      copy[i] = copy[i] + 1;
      따봉변경(copy)
   }}>👍</span> {따봉[i]}
</h4>
```

state가 array일 때는 복사부터 한 뒤 수정합니다.

> **Q. 코드가 너무 어려워요**
> 남의 코드를 읽는 건 세상에서 가장 어려운 일이라 직접 해봐야 이해가 빠릅니다. 어려우면 map을 쓰지 않은 이전 코드부터 해보세요.

## 자식이 부모의 state를 쓰고 싶을 때는 props

> **오늘의 숙제**
> 모달창 안에 글 수정 버튼을 만들고, 누르면 첫 글제목이 '여자코트 추천'으로 바뀌는 기능을 만들어봅시다.

자식 컴포넌트가 부모 컴포넌트의 state를 쓰려면, 그냥 쓰면 안 되고 **props** 로 전송해서 써야 합니다.

### Modal 안에 글제목 state가 필요한데

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트 추천', '강남 우동맛집', '파이썬독학']);
  return (
    <div>
      <Modal></Modal>
    </div>
  )
}

function Modal(){
  return (
    <div className="modal">
      <h4>{ 글제목[0] }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

위 코드는 `글제목`이 정의되지 않았다는 에러가 납니다. `글제목` state는 `App()`에 있지 `Modal()`에 없기 때문입니다. 자바스크립트에선 다른 함수의 변수를 마음대로 가져다 쓸 수 없습니다.

![App 함수와 Modal 함수의 변수 스코프 분리](/assets/20220426_145233.png)

하지만 두 컴포넌트가 **부모/자식 관계** 라면 가능합니다(다른 컴포넌트 안에 있는 컴포넌트를 자식 컴포넌트라고 합니다).

![부모 컴포넌트의 state를 자식 컴포넌트로 전송](/assets/20220426_145250.png)

부모의 state를 자식으로 전송할 수 있으며, 이때 **props** 문법을 사용합니다.

### props로 부모 → 자식 state 전송하는 법

2개의 step을 따릅니다.

1. 자식 컴포넌트 사용하는 곳: `<자식컴포넌트 작명={state이름} />`
2. 자식 컴포넌트 만드는 function: `props` 파라미터 등록 후 `props.작명` 사용

예시로 `글제목` state를 `<Modal>`에 전송해봅시다.

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트 추천', '강남 우동맛집', '파이썬독학']);
  return (
    <div>
      <Modal 글제목={글제목}></Modal>
    </div>
  )
}

function Modal(props){
  return (
    <div className="modal">
      <h4>{ props.글제목[0] }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

props 전송 문법은 중요하니 외워둡시다.

> **(참고1)** props는 `<Modal 이런거={이런거} 저런거={저런거}>`처럼 무한히 전송할 수 있습니다.
>
> **(참고2)** state만 전송할 수 있는 건 아닙니다. 일반 변수, 함수도 전송할 수 있고(`<Modal 글제목={변수명}>`), 일반 문자열은 중괄호 없이도 됩니다(`<Modal 글제목="강남우동맛집">`).

![자식 → 부모 방향 전송은 불가능](/assets/20220426_145741.png)

자식 → 부모 방향 전송은 불가능합니다.

![옆 컴포넌트로의 전송도 불가능](/assets/20220426_145747.png)

옆에 있는 컴포넌트로의 전송도 불가능합니다.

### props는 함수 파라미터와 똑같다

함수에 파라미터를 넣는 이유는 "함수 하나로 다양한 기능을 쓰기 위해서"입니다. props도 똑같은 파라미터 문법입니다. 그래서 응용이 가능합니다.

배경색만 다른 모달창이 여러 개 필요하다고 합시다. `Modal2()`, `Modal3()`처럼 따로 만들어도 되지만, 내용이 같은데 색만 다르다면 같은 함수를 여러 개 만들 필요가 없습니다.

```
function Modal(props){
  return (
    <div className="modal" style=>
      <h4>{ props.글제목[0] }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

`props.color`처럼 구멍을 뚫어놓으면, 컴포넌트 사용 시 색을 바꿔 넘길 수 있습니다.

```
<Modal color={'skyblue'} />
```

위는 하늘색 모달창,

```
<Modal color={'orange'} />
```

위는 오렌지색 모달창이 생성됩니다. 비슷한 컴포넌트를 또 만들 필요가 없어집니다.

> **결론:** 함수 파라미터 문법을 잘 알면 그만큼 응용이 가능합니다. 즉, 자바스크립트를 잘해야 리액트를 잘합니다.

## props를 응용한 상세페이지 만들기

새로운 내용은 없고, 지금까지 배운 것으로 기능을 만들어봅시다.

지금 글을 누르면 모달창이 뜨는데 항상 0번 글제목만 보입니다.

> **Q. 지금 누른 글제목이 모달창에 뜨게 하려면?**
> 0번 글을 누르면 0번 제목이, 1번 글을 누르면 1번 제목이 모달창에 등장하게 만들어봅시다.

힌트: 모달창 안의 제목도 일종의 동적인 UI입니다. 동적인 UI 3-step을 떠올려봅시다.

1. HTML/CSS로 미리 디자인 — 이미 했으니 패스.
2. 현재 UI 상태를 state로 만들기.
3. state에 따라 UI가 어떻게 보일지 작성.

**step 2. state 만들기**

```
let [title, setTitle] = useState(0);
```

모달창 안에 보이는 글은 0번/1번/2번 중 하나이므로 숫자로 표현합니다.

**step 3. state에 따라 작성**

```
function App (){
  let [title, setTitle] = useState(0);
  (생략)
}

function Modal(props){
  return (
    <div className="modal">
      <h4>{ 만약에 title == 0이면 0번 글제목 보여주세요~ }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

"title이 0이면 `props.글제목[0]`, 1이면 `props.글제목[1]`"처럼 조건문을 써도 되지만, 더 간단하게 `props.글제목[title]`을 쓸 수 있습니다.

```
function App (){
  let [title, setTitle] = useState(0);
  (생략)
}

function Modal(props){
  return (
    <div className="modal">
      <h4>{ props.글제목[title] }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

그런데 에러가 납니다. `title` state는 부모(App)가 가진 state라서, 자식(Modal)이 쓰려면 props로 전송해야 합니다.

```
function App (){
  let [title, setTitle] = useState(0);
  (생략)
  {
    modal == true ? <Modal title={title} 글제목={글제목} /> : null
  }
}

function Modal(props){
  return (
    <div className="modal">
      <h4>{ props.글제목[props.title] }</h4>
      <p>날짜</p>
      <p>상세내용</p>
    </div>
  )
}
```

이제 스위치(`title`)를 0, 1, 2로 바꿔주면 모달창에 알맞은 글이 표시됩니다. 글에 `onClick`을 넣어 클릭 시 스위치를 바꿔주면 됩니다.

### 먼저 반복문 없는 버전으로 연습

```
function App (){
  let [title, setTitle] = useState(0);
  return (
    <div>
      <button onClick={()=>{ ? }}> 0번글 </button>
      <button onClick={()=>{ ? }}> 1번글 </button>
      <button onClick={()=>{ ? }}> 2번글 </button>
      <Modal 어쩌구/>
    </div>
  )
}
```

버튼 3개를 각각의 글제목이라 간주하고 코드를 짜봅시다. 0번 글을 누르면 `title`을 0으로 바꾸면 됩니다.

```
function App (){
  let [title, setTitle] = useState(0);
  return (
    <div>
      <button onClick={()=>{ setTitle(0) }}> 0번글 </button>
      <button onClick={()=>{ setTitle(1) }}> 1번글 </button>
      <button onClick={()=>{ setTitle(2) }}> 2번글 </button>
      <Modal 어쩌구/>
    </div>
  )
}
```

버튼을 누를 때마다 state가 바뀌고, 모달창 안의 제목도 자동으로 바뀝니다.

### map 반복문 버전에 적용

```
function App (){
  return (
    <div>
      {
        글제목.map(function(a, i){
          return (
          <div className="list">
            <h4 onClick={()=>{ setModal(true); setTitle(i); }}>{ 글제목[i] }</h4>
            <p>2월 18일 발행</p>
          </div> )
        })
      }
    </div>
  )
}
```

각 글제목을 누르면 `setTitle(i)`가 실행됩니다. map에서 `i`는 반복할 때마다 `0, 1, 2 ...`로 증가하는 정수이므로,

- 첫 글제목 클릭 → `setTitle(0)`
- 둘째 글제목 클릭 → `setTitle(1)`
- 셋째 글제목 클릭 → `setTitle(2)`

가 실행됩니다. 스위치가 조작되면 모달창 안의 제목도 알아서 바뀝니다.

## input 1: 사용자가 입력한 글 다루기

리액트에서 유저 입력 데이터를 다루는 법은 순수 자바스크립트와 거의 같습니다.

### input 태그 사용하기

유저 입력을 받는 박스는 다음 태그들로 만들 수 있습니다.

```
<input type="text"/>
<input type="range"/>
<input type="date"/>
<input type="number"/>
<textarea></textarea>
<select></select>
```

이외에도 다양한 인풋 박스가 있으니 필요할 때 찾아 쓰면 됩니다.

### input에 입력할 때 코드 실행하기

유저가 input에 입력할 때마다 코드를 실행하려면 `onChange` 또는 `onInput` 이벤트 핸들러를 답니다.

```
<input onChange={()=>{ 실행할코드 }}/>
```

`console.log(1)`을 넣어 테스트해볼 수 있습니다.

> **(참고) 다양한 이벤트 핸들러**
> - `onMouseOver={ }`: 요소에 마우스를 댔을 때 실행
> - `onScroll={ }`: 요소를 스크롤했을 때 실행
>
> 수십 개가 있으니 원하는 이벤트를 찾아 쓰면 됩니다.

### input에 입력한 값 가져오기

```
<input onChange={(e)=>{ console.log(e.target.value) }}/>
```

`e`라는 파라미터를 추가하고 `e.target.value`라고 적으면 현재 `<input>`에 입력된 값을 가져올 수 있습니다.

이벤트 핸들러 함수에 추가하는 `e`는 **이벤트 객체** 라고 부르며, 현재 이벤트와 관련한 유용한 기능을 제공합니다(작명은 자유).

- `e.target`: 현재 이벤트가 발생한 곳을 알려줌
- `e.preventDefault()`: 이벤트 기본 동작을 막음
- `e.stopPropagation()`: 이벤트 버블링을 막음 (좋아요 버튼 누를 때 모달창도 떠버리는 버그 해결 가능)

### 입력한 데이터 저장하기

사용자가 input에 입력한 데이터는 보통 state나 변수에 저장합니다. state를 사용해봅시다.

```
function App (){

  let [입력값, 입력값변경] = useState('');
  return (
    <input onChange={(e)=>{
      입력값변경(e.target.value)
      console.log(입력값)
    }} />
  )
}
```

state를 하나 만들고, `onChange`될 때마다 `e.target.value`를 state에 넣었습니다.

> 기본값을 뭘 넣을지 모르겠으면 따옴표 2개(`''`)를 칩니다. 빈 문자를 뜻합니다.

> **(참고) console.log 출력이 한 박자 늦는 이유**
> `a`를 입력하면 콘솔에 아무것도 안 뜨고, `aa`를 입력하면 `a`만 뜹니다. state 변경함수가 **비동기적(async)** 으로 처리되기 때문입니다. 자바스크립트는 늦게 처리되는 코드를 잠깐 제쳐두고 다음 줄을 먼저 실행하므로, `console.log(입력값)`이 먼저 실행됩니다. 실행 순서만 다를 뿐 state 변경은 잘 됩니다.

![input onChange로 state에 값 저장](/assets/20220426_174845.png)

> **오늘의 숙제**
> 1. input에 입력하고 발행 버튼을 누르면 블로그에 글이 하나 추가되는 기능을 만들어보세요.
> 2. 글마다 옆에 삭제 버튼을 만들고, 누르면 글이 없어지는 기능을 만들어보세요.
>
> **힌트:** HTML을 직접 만질 필요 없이 `글제목` state만 바꾸면 됩니다. array에 자료를 추가/삭제하는 문법은 모르면 구글에 찾아보세요.

## input 2: 블로그 글 발행 기능 만들기

새로운 내용은 없고 저번 숙제 2개를 풀어봅니다. 답은 언제나 하나가 아니니, 의도대로 동작한다면 넘어가도 됩니다.

### 숙제 1. 글 발행 기능

"버튼 누르면 유저가 입력한 글로 HTML을 생성"하는 건 순수 자바스크립트 방식입니다. 리액트는 스위치(state)만 만지므로, **"버튼 누르면 입력한 글을 `글제목` state에 넣어주세요"** 로 짜면 됩니다. `글제목.map()` 덕분에 state 개수에 맞게 HTML이 자동 생성되니까요.

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트추천', '강남우동맛집', '파이썬독학']);
  let [입력값, 입력값변경] = useState('');
  return (
    <div>
      <input onChange={ (e)=>{ 입력값변경(e.target.value) } } />
      <button onClick={()=>{
        ???
      }}>글발행</button>
    </div>
  )
}
```

`???` 자리에 무엇을 넣을까요?

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트추천', '강남우동맛집', '파이썬독학']);
  let [입력값, 입력값변경] = useState('');
  return (
    <div>
      <input onChange={ (e)=>{ 입력값변경(e.target.value) } } />
      <button onClick={()=>{
        let copy = [...글제목];
        copy.unshift(입력값);
        글제목변경(copy)
      }}>글발행</button>
    </div>
  )
}
```

1. 버튼 누르면 `글제목` state를 카피합니다(array state 조작은 우선 카피부터).
2. 카피한 것에 `unshift(입력값)`을 합니다(array 맨 앞에 자료 추가하는 문법).
3. state 변경함수를 사용합니다.

이제 input에 입력하고 발행 버튼을 누르면 글이 발행됩니다.

> **Q. state, props, component, map을 다 아는데 코드를 못 짜겠어요.**
> A. 리액트를 못하는 게 아니라 프로그래밍 기초가 부실한 것이니 안심하세요.

> **Q. 왜 새로고침하면 없어지나요?**
> A. 브라우저는 새로고침하면 HTML, JS 파일을 다시 읽고, state와 변수가 초기값으로 돌아갑니다. 실제 서비스라면 서버로 보내 DB에 영구 저장하겠지만, 여기서는 프론트엔드만 다루므로 이 정도로 충분합니다.

### 숙제 2. 글 삭제 기능

map 반복문 안에 버튼을 만든 뒤, "div 하나 삭제"가 아니라 **"state에서 글을 삭제"** 하는 리액트 방식으로 짭니다.

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트추천', '강남우동맛집', '파이썬독학']);
  let [입력값, 입력값변경] = useState('');
  return (
    <div>
    {
     글제목.map(function(a, i){
        return (
          <div className="list">
            <h4>{ 글제목[i] }</h4>
            <p>2월 18일 발행</p>
            <button onClick={()=>{ ? }}>삭제</button>
          </div>
        )
     })
    }
  </div>
  )
}
```

array에서 X번째 데이터를 삭제하려면 `array자료.splice(x, 1)`을 씁니다.

```
function App (){
  let [글제목, 글제목변경] = useState(['남자코트추천', '강남우동맛집', '파이썬독학']);
  let [입력값, 입력값변경] = useState('');
  return (
    <div>
    {
     글제목.map(function(a, i){
        return (
          <div className="list">
            <h4>{ 글제목[i] }</h4>
            <p>2월 18일 발행</p>
            <button onClick={()=>{
              let copy = [...글제목];
              copy.splice(i, 1);
              글제목변경(copy);
           }}>삭제</button>
          </div>
        )
     })
    }
  </div>
  )
}
```

1. 버튼 누르면 `글제목` state 사본을 만듭니다.
2. `splice(i, 1)`로 i번째 데이터를 삭제합니다.

그래서 각 삭제 버튼은 자기 인덱스의 글을 없앱니다.

- 0번째 삭제 버튼 → `copy.splice(0, 1)` → 0번 글 삭제
- 1번째 삭제 버튼 → `copy.splice(1, 1)` → 1번 글 삭제
- 2번째 삭제 버튼 → `copy.splice(2, 1)` → 2번 글 삭제

### 응용 과제

1. **빈 글 발행 막기** — 아무것도 입력 안 하고 발행 버튼 누르는 것을 막아 안전한 사이트를 만듭니다.
2. **따봉 개수 동기화** — 글을 추가하면 개별 따봉 기록도 하나 더 만들어줘야 합니다.
3. **날짜 데이터** — 글만 저장된 state에 날짜도 함께 저장하고, 발행 시 현재 날짜를 자동으로 넣어볼 수 있습니다.

## class를 이용한 옛날 React 문법

여러분은 최신 문법을 배우고 있지만, 실무에서는 옛날 문법으로 작성된 리액트 파일을 볼 때가 있으므로 이해는 할 수 있어야 합니다. 컴포넌트 만들기, state 만들고 변경하기를 예전에는 어떻게 했는지 알아봅시다.

### 옛날 문법으로 Component 만들기

```
function App(){
  return (
    <div>
      HTML 잔뜩있는 곳
      <Profile />
    </div>
  )
}


class Profile extends React.Component {
  constructor(){
    super();
  }

  render(){
    return (
      <div>프로필영역입니다</div>
    )
  }
}

```

class부터 시작해 최소 6줄을 써야 합니다.

1. class를 만들고 이름 짓습니다.
2. `React.Component`를 `extends`합니다.
3. `constructor(){}` 함수를 작성합니다.
4. `render(){}` 함수 안에 원하는 HTML을 적습니다.

그럼 `<Profile />`을 원하는 곳에 첨부해 사용할 수 있습니다.

> **문법 용어 정리**
> - **class**: 데이터/함수를 보관하는 덩어리.
> - **extends**: 오른쪽 덩어리의 성질을 물려받아 덩어리를 만든다는 뜻.
> - **React.Component**: 컴포넌트 성질을 가진 덩어리. 이를 extends해 class를 만들면 우리가 쓰던 컴포넌트가 됩니다.

왜 class를 쓰는가? class는 여러 데이터/함수를 한 곳에 보관할 때 쓰는 문법입니다. class를 만들어두면, 그 데이터를 복사해 쓸 수 있는 object나 또 다른 class(extends)를 쉽게 만들 수 있습니다.

리액트가 class를 쓰는 이유는,

1. 리액트 제작자들이 컴포넌트 관련 데이터/함수를 보관하려고 `React.Component` class를 만들어뒀고,
2. 그것을 extends로 복사하면 나만의 컴포넌트를 만들 수 있으며,
3. 그렇게 하면 내 컴포넌트가 리액트 관련 데이터/함수를 자유롭게 쓸 수 있기 때문입니다.

### 옛날 문법으로 state 만들고 쓰는 법

예전엔 `constructor(){}` 안에 모든 state를 보관했습니다.

```
class Profile extends React.Component {
  constructor(){
    super();
    this.state = { name : 'Kim', age : 30 }
  }

  render(){
    return (
      <div>
        <h3> 저는 { this.state.name } 입니다 </h3>
      </div>
    )
  }
}
```

1. state는 `constructor()` 안의 `this.state`라는 변수에 전부 보관합니다.
2. 꺼내 쓸 때는 `this.state.state명`을 씁니다.

> **문법 용어 정리 2**
> - **constructor()**: 변수와 함수가 가득한 class 덩어리를 만들 때 새 변수를 넣는 공간.
> - **super()**: extends한 `React.Component`의 변수들을 그대로 물려받겠다는 뜻. 반드시 먼저 써야 그 아래에서 state를 만들 수 있습니다.

### state 값 바꾸는 법

버튼을 누르면 이름을 'Park'으로 바꾸는 기능을 만들어봅시다. `onClick`은 똑같지만, state 변경 시에는 특정 함수를 써야 합니다.

```
class Profile extends React.Component {
  constructor(){
    super();
    this.state = { name : 'Kim', age : 30 }
  }

  render(){
    return (
      <div>
        <h3> 저는 { this.state.name } 입니다 </h3>
        <button onClick={ ()=>{ this.setState( {name : 'Park'} ) } }> 버튼 </button>
      </div>
    )
  }
}
```

1. 버튼 만들고 `onClick` 넣는 건 똑같습니다.
2. state 변경 시 `this.setState()` 내장함수를 꼭 씁니다. `( )` 안에는 바꿀 state 이름과 값만 적습니다.

> **주의:** 신문법 state 변경함수는 state를 통째로 바꿔치우지만, 예전 `setState()`는 **딱 필요한 부분만 수정** 하고 나머지는 건드리지 않습니다. 그래서 복사본을 만들고 수정하는 과정이 필요 없습니다.

### 커스텀 함수 만드는 법

최신 문법은 함수를 아무 데나 만들었지만, 예전 문법은 위치가 정해져 있습니다. 버튼 안의 `setState()` 코드를 함수로 빼봅시다.

```
class Profile extends React.Component {
  constructor(){
    super();
    this.state = { name : 'Kim', age : 30 }
  }

  changeName(){
    this.setState( {name : 'Park'} )
  }

  render(){
    return (
      <div>
        <h3> 저는 { this.state.name } 입니다 </h3>
        <button onClick={ this.changeName }> 버튼 </button>
      </div>
    )
  }
}
```

1. 버튼 안 코드가 길어서 함수로 뺐습니다.
2. 커스텀 함수는 `changeName` 자리에 만듭니다.
3. 함수 이름을 필요한 자리에 넣습니다(`this`를 잊지 마세요).

맞는 방법인데 **에러가 납니다.** `this.setState()`의 `this`가 이상해졌기 때문입니다. `changeName()`도 `function` 문법의 축약 버전인데, 자바스크립트에서 `function`을 쓰면 안의 `this`가 항상 새롭게 재정의됩니다. 그래서 `changeName` 안의 `this`도 재정의되어 의도와 다르게 동작합니다.

해결법은 두 가지입니다.

1. `this.changeName.bind(this)`로 사용하거나
2. 함수를 **arrow function** 으로 바꾼다.

```
class Profile extends React.Component {
  constructor(){
    super();
    this.state = { name : 'Kim', age : 30 }
  }

  changeName = () => {
    this.setState( {name : 'Park'} )
  }

  render(){
    return (
      <div>
        <h3> 저는 { this.state.name } 입니다 </h3>
        <button onClick={ this.changeName }> 버튼 </button>
      </div>
    )
  }
}
```

이렇게 하면 에러가 나지 않습니다.

### arrow function과 this 키워드의 관계

`() => {}`(arrow function)와 `function(){}`은 거의 같은 의미이지만 한 가지 차이가 있습니다.

- **arrow function**: 안의 `this`를 재정의하지 않고, 바깥의 `this` 값을 그대로 끌고 와 사용합니다.
- **function(){}**: `this` 값이 새롭게 변합니다.

그래서 arrow function은 내부의 `this` 값을 변화시키고 싶지 않을 때 사용합니다.

> `this`의 자세한 문법은 ES6 강의 무료 파트에서 학습할 수 있습니다.

### 오늘의 교훈: 신문법을 쓰자

리액트 공식 문서도 컴포넌트는 `function` 문법으로 만들 것을 권장합니다.

- 어려운 class 문법이나 `this` 키워드를 쓰지 않아도 되어 빠르고 쉽게 개발할 수 있습니다.
- function으로 만드나 class로 만드나 기능적 차이는 거의 없습니다.
- `useState` 같은 Hook을 자주 쓸 거라면 function이 훨씬 유용합니다.
- 다만 class 컴포넌트는 Lifecycle Hook을 조금 더 자세히 쓸 수 있습니다.
