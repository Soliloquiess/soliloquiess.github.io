## 학습 목표

- 복잡한 애플리케이션의 컴포넌트들을 효율적으로 관리하는 Vuex 라이브러리 소개
- Vuex 등장 배경인 Flux 패턴 소개
- Vuex의 주요 속성인 state, getters, mutations, actions 학습
- Vuex를 더 쉽게 코딩할 수 있는 Helper 기능 소개
- Vuex로 프로젝트를 구조화하는 방법과 모듈 구조화 방법 소개

----

## Flux란

MVC 패턴의 복잡한 데이터 흐름을 해결하기 위한 패턴이다.

![Flux 패턴 구성 요소](/assets/20210515_224620.png)

| 구성 요소 | 설명 |
| --- | --- |
| **action** | 화면에서 발생하는 이벤트 또는 사용자의 입력 |
| **dispatcher** | 데이터를 변경하는 방법(메서드) |
| **model** | 화면에 표시할 데이터 |
| **view** | 사용자에게 비춰지는 화면 |

### MVC 패턴의 문제점

![MVC 패턴의 양방향 데이터 흐름](/assets/20210515_225337.png)

- 기능 추가 및 변경에 따라 생기는 문제점을 예측할 수 없음
- 앱이 복잡해지면서 생기는 업데이트 루프

![복잡해진 MVC 데이터 흐름](/assets/20210515_225451.png)

### Flux 패턴의 단방향 데이터 흐름

- 데이터 흐름이 여러 갈래로 나뉘지 않고 단방향으로만 처리된다.

![Flux 단방향 데이터 흐름](/assets/20210515_225837.png)

## Vuex가 왜 필요할까

- 복잡한 애플리케이션에서 컴포넌트 개수가 많아지면 컴포넌트 간 데이터 전달이 힘들어진다.

![컴포넌트 간 데이터 전달의 어려움](/assets/20210515_230024.png)

### 이벤트 버스로 해결?

- 어디서 이벤트를 보냈는지, 혹은 받았는지 알기 어렵다.

----

## Vuex에서 해결할 수 있는 문제

1. MVC 패턴에서 발생하는 구조적 오류
2. 컴포넌트 간 데이터 전달 명시
3. 여러 개의 컴포넌트에서 같은 데이터를 업데이트할 때의 동기화 문제

## Vuex 컨셉

| 컨셉 | 설명 | 대응 |
| --- | --- | --- |
| **State** | 컴포넌트 간에 공유하는 데이터 | `data()` |
| **View** | 데이터를 표시하는 화면 | Template |
| **Action** | 사용자의 입력에 따라 데이터를 변경 | getMethods |

![Vuex 핵심 컨셉](/assets/20210515_232340.png)
