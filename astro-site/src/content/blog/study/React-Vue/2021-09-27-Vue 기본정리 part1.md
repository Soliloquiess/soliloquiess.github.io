## 개발 환경 세팅과 Vue 3 설치

문법 체험을 위해 HTML 파일에 라이브러리처럼 간단히 붙여 쓰는 방식은 사용하지 않고, 실제 웹앱 개발에 필요한 Vue 프로젝트를 처음부터 만드는 방식으로 진행합니다.

그러려면 **Vue CLI** 라는 라이브러리가 필요하니 그것부터 설치합니다. 요즘은 설치 과정이 맥이든 윈도우든 거의 동일합니다.

![Vue CLI 설치 안내 화면](/assets/20210927_214708.png)

### 설치 순서 요약

**1. Node.js 최신 버전 설치**

- 다운로드 페이지 기준 우측 최신 버전(Current)이 에러가 덜 나는 편입니다.
- 그래도 에러가 나면 왼쪽 LTS 버전으로 바꿔보세요. 제어판에서 삭제 후 재설치하면 됩니다.

**2. VS Code 에디터 설치**

- 구글에서 검색해 다운로드 후 설치합니다.

![VS Code 다운로드 화면](/assets/20210927_214823.png)

**3. 작업용 폴더를 만들고 에디터로 오픈**

- 아무 곳에나 작업 폴더를 만든 뒤, 에디터 상단 메뉴의 `Open Folder`로 그 폴더를 엽니다.
- 폴더를 제대로 열었다면 에디터 왼쪽 위에 방금 만든 폴더명이 표시됩니다.

![작업 폴더 오픈 화면](/assets/20210927_214858.png)

**4. 에디터 터미널에서 Vue CLI 설치**

- 터미널은 에디터 상단 `Terminal - New Terminal`로 엽니다.
- `npm install -g @vue/cli` 입력 후, Vue 3 버전을 방향키 + 엔터로 선택합니다.
- 안 되면 90% 확률로 Node.js 설치가 잘못된 경우입니다(해결책은 하단 참고).

npm 명령부터 에러가 난다면 yarn으로 대체할 수 있습니다.

- 윈도우: yarn 1.22 인스톨러를 구글에서 검색해 받아 설치하고, 컴퓨터를 재시작한 뒤 `yarn global add @vue/cli`
- 맥: 터미널을 열고 `npm install -g yarn`
- 주의: 윈도우는 맥처럼 `npm install -g yarn` 방식으로 설치하지 마세요.

![Vue CLI 설치 진행 화면](/assets/20210927_214930.png)

**5. 프로젝트 생성**

- 설치가 끝났으면 터미널에서 `vue create 프로젝트명`을 입력합니다.
- 프로젝트명은 자유롭게 지을 수 있으며, 하위 폴더로 프로젝트 폴더가 생성됩니다.

![vue create 실행 화면](/assets/20210927_215005.png)

**6. 생성된 프로젝트 폴더를 에디터로 다시 오픈 후 코딩 시작**

- 하위 폴더로 생성된 프로젝트 폴더(예: `vuedongsan`)를 다시 `Open Folder`로 엽니다.
- 폴더를 열지 않은 채로 코딩하면 동작하지 않을 수 있습니다.
- 여러 파일 중 `App.vue`에 코딩을 시작하면 됩니다.

**7. 미리보기**

- 터미널에서 `npm run serve`를 입력합니다.
- 위에서 프로젝트 폴더를 열어두지 않았으면 에러가 날 수 있습니다.

**8. VS Code 확장 기능 설치**

- 에디터 제일 왼쪽 네모 버튼(Extensions)에서 **Vetur**, **Vue 3 Snippets**, **HTML CSS Support** 세 개를 설치합니다.

### 설치 과정에서 자주 겪는 에러

설치 과정에서 일정 확률로 에러가 뜹니다. 대부분 에러 메시지를 구글에 검색하면 해결되지만, 자주 겪는 케이스를 정리합니다.

**설치가 10분 이상 걸려요**

- 인터넷이 느리면 그렇습니다. 안정적인 네트워크에서 진행하세요.

**npm, yarn 명령어 입력하자마자 에러가 납니다**

- 설치가 잘 되다가 중간에 에러가 나는 대부분의 경우는 Node.js가 최신 버전이 아닐 때입니다.
- ① Node.js 삭제 후 ② 다른 버전을 받아 다시 시도해보세요.

**`npm : command not found` 에러**

- 역시 대부분 Node.js가 잘못 설치된 경우입니다.
- Node.js 설치 시 설치 경로를 임의로 변경하지 마세요.
- 맥도 brew 등으로 설치하지 말고 공식 사이트에서 다운로드하세요.
- 리눅스는 Node.js 버전 업그레이드 명령어를 찾아 입력하면 됩니다.

**맥에서 permission / 권한 없음 에러**

- 직관적인 해결책은 `npm`이나 `yarn` 명령 앞에 `sudo`를 붙이는 것입니다.
- 예: `sudo npm install -g @vue/cli`
- 설치 중간에 맥북 비밀번호 입력이 필요할 수 있습니다. 다만 `sudo`는 임시방편입니다.

```
npm ERR! syscall access
npm ERR! Error: EACCES: permission denied, access '/usr/local/lib/node_modules'
```

위처럼 폴더 수정 권한이 없다는 에러라면, 해당 폴더(`/usr/local/lib/node_modules`)에 수정 권한을 주면 됩니다. 터미널에서 아래 둘 중 하나를 입력해보세요.

```
sudo chown -R 님맥북유저이름: 위에에러뜬경로
sudo chown -R $USER 위에에러뜬경로
```

- 맥북 유저 이름은 터미널에 `whoami`를 입력하면 나옵니다.
- 이것도 임시방편이며, 다른 경우가 있을 수 있으니 에러 메시지를 그대로 구글에 검색해보세요.

**윈도우 PowerShell에서 '보안 오류'(허가되지 않은 스크립트) 빨간 글씨**

- 윈도우 검색(돋보기) → PowerShell 검색 → 우클릭 → 관리자 권한으로 실행 후 아래를 대소문자 그대로 입력하세요.

```
Set-ExecutionPolicy Unrestricted
```

그러면 npm 명령이 정상 동작합니다.

![PowerShell 실행 정책 변경 화면](/assets/20210927_221339.png)

**윈도우 PowerShell에서 권한 없음 에러**

- 윈도우 검색에서 PowerShell을 우클릭 → 관리자 권한으로 실행한 뒤 `npm install -g @vue/cli` 또는 `yarn global add @vue/cli`를 입력합니다.
- `vue create` 시에도, 작업 폴더를 연 뒤 상단 메뉴에서 `파일 - PowerShell 열기 - 관리자 권한으로 PowerShell 열기`를 눌러 입력해보세요.

## HTML에 데이터를 넣는 Vue 데이터바인딩 문법

먼저 아래와 같은 데이터를 하나 저장해두세요.

```
products : ['역삼동원룸', '천호동원룸', '마포구원룸']
```

![상품 목록 레이아웃 예시](/assets/20210927_215320.png)

위 데이터와 문법으로 대충 이렇게 생긴 HTML 레이아웃을 만들어옵니다(가격은 임시로 아무거나 기입).

### 데이터바인딩을 왜 하는가

HTML에 자바스크립트 데이터를 꽂아 넣는 것을 **데이터바인딩**이라고 합니다. Vue에서 데이터바인딩하는 문법을 2개 알아볼 텐데, 그 전에 "왜 하는지"부터 이해하는 것이 좋습니다.

Vue 개발은 어려운 게 아닙니다. 평소 HTML/CSS로 웹페이지를 만들던 대로 코드를 작성하고, 필요한 순간순간 Vue 문법을 더하면 됩니다. 그래서 원룸을 파는 쇼핑몰을 만들어보며 배워봅니다. 일단 레이아웃부터입니다.

```
<template>
  <div>
    <h4>XX 원룸</h4>
    <p>XX 만원</p>
  </div>
  <div>
    <h4>XX 원룸</h4>
    <p>XX 만원</p>
  </div>
</template>
```

- HTML 코드는 `<template>` 안에 넣습니다.
- 자바스크립트 기능은 `<script>`, 스타일은 `<style>` 안에 넣습니다.

위 HTML을 추가해둡니다. 배울 문법이 하나 있어서 그렇습니다.

### 전통 방식 vs Vue 방식

자바스크립트로 자주 하는 작업이 변수나 데이터를 HTML에 꽂아 넣는 데이터바인딩입니다. 전통 방식은 이렇게 길게 한 줄을 써야 했습니다.

```
document.getElementById(어쩌구).innerHTML = 데이터;
```

반면 Vue는 ① 데이터를 어딘가에 보관해두고, ② 그것을 간단한 문법으로 HTML 중간중간에 쉽게 꽂아 넣을 수 있습니다.

data 보관함은 여기 있습니다.

```
<script>
export default {
  name : 'App',
  data(){
    return {
      price1 : 60
    }
  }
}

</script>
```

- `<script>` 태그 안에 `data(){ return { } }`를 열고, 데이터를 object 형식으로 저장합니다.
- 이것이 Vue의 data 보관함(변수 보관함)입니다. 중요한 데이터는 모두 여기에 보관하세요. 중괄호이니 object 형식에 맞춰서요.

이제 `price1`을 HTML 안에 꽂아 넣어 유저에게 보여주고 싶으면 중괄호 문법(`{{ price1 }}`)만 쓰면 됩니다.

```
<p>{{ price1 }} 만원</p>
```

저장하면 브라우저에 "60만원"이 잘 뜹니다.

### Q. 그냥 `<p>60만원</p>`으로 하드코딩하면 되는데 왜 데이터바인딩을 하나?

**이유 1. 수정이 편리합니다.** 쇼핑몰은 가격이 자주 변동되는데, 데이터로 저장해두면 JS로 조작하기 쉬워 나중에 수정이 편합니다.

**이유 2. 실시간 렌더링 기능을 쓸 수 있습니다.** Vue는 data가 변경되면 관련된 HTML에 실시간으로 반영됩니다. `price1`을 60에서 70으로 바꾸면

```
<p>{{ price1 }} 만원</p>
```

여기에도 변경 사항이 바로 적용됩니다. 따로 코드를 짤 필요 없이 자동으로 바뀝니다. 이런 사이트를 **웹앱**이라고 부릅니다.

그래서 웹앱을 만들고 싶으면 자주 바뀔 만한 데이터를 `data`란에 넣고 데이터바인딩해서 보여주세요.

### HTML 속성도 데이터바인딩이 가능합니다

```
style="" id="" class=""
```

이런 속성에도 저장해둔 data를 꽂아 넣을 수 있습니다.

```
<template>
  <div>
    <h4 :style="스타일">XX 원룸</h4>
    <p>XX 만원</p>
  </div>
  <div>
    <h4>XX 원룸</h4>
    <p>XX 만원</p>
  </div>
</template>

<script>
export default {
  name : 'App',
  data(){
    return {
      price1 : 60,
      스타일 : 'color:red'
    }
  }
}

</script>
```

이렇게 하면 `color:red`라는 데이터도 원하는 곳에 꽂아 넣을 수 있습니다. 속성 이름 앞에 콜론(`:`)을 붙이는 것이 핵심이며, 상상하는 모든 속성에 데이터를 꽂아 넣을 수 있습니다.

## v-for 반복문

> **5분 숙제:** `v-for` 문법을 이용해 지금 있는 상품 목록을 반복문으로 축약해보세요. 가격은 신경 쓰지 말고 상품 제목만 잘 보이면 됩니다.

### 지난 시간 숙제 복습

지난 시간에 데이터로 `products : [ '역삼동원룸', '천호동원룸', '마포구원룸' ]` 3개의 상품명을 저장하고, 이를 HTML에 데이터바인딩하라고 했습니다. 중괄호 문법으로 데이터를 꽂아 넣으면 됩니다.

```
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
</div>
<div>
  <h4>{{ products[1] }}</h4>
  <p>50만원</p>
</div>
<div>
  <h4>{{ products[2] }}</h4>
  <p>50만원</p>
</div>
```

HTML을 짜다 보면 비슷한 `<div>`가 수없이 반복됩니다. 이를 for 반복문처럼 축약하고 싶을 때 리액트나 Vue를 쓰면 가능합니다. `for`, `map`, `forEach` 같은 문법이 어려웠더라도 Vue 반복문은 훨씬 쉽습니다.

### 상단 메뉴 만들기

심심하니 상단 메뉴를 만들어봅니다. CSS는 vue 파일 하단 `<style>` 태그 안에 넣습니다.

```
<div class="menu">
  <a>Home</a>
  <a>Products</a>
  <a>About</a>
</div>
.menu {
  background : darkslateblue;
  padding : 15px;
  border-radius : 5px;
}
.menu a {
  color : white;
  padding : 10px;
}
```

### v-for HTML 반복문 (기본)

HTML에 반복되는 부분이 보이면 `v-for`로 축약할 수 있습니다. 위 a 태그 3개를 한 줄로 생성해봅니다.

```
<div class="menu">
  <a v-for="작명 in 3" :key="작명">Home</a>
</div>
```

1. 원하는 태그에 `v-for="작명 in 반복할횟수"`를 적습니다.
2. 그리고 `:key="작명"`도 추가합니다.

그러면 이 HTML 태그가 원하는 만큼 반복 생성됩니다. 위 예제는 Home이라는 a 태그가 3개 생성됩니다.

- **작명**은 변수 하나를 짓는 것이므로 자유롭게 정하면 됩니다.
- **`:key`** 속성은 반복문을 돌릴 때 꼭 필요합니다. 반복된 요소들을 각각 구분하기 위한 속성입니다.

### v-for HTML 반복문 (array / object 활용)

`v-for`에는 array, object 데이터를 넣을 수도 있습니다. array 자료를 데이터로 저장해봅니다.

```
data(){
  return {
    메뉴들 : ['Home', 'Shop', 'About']
  }
}

```

이제 `메뉴들`이라는 데이터로 반복문을 돌릴 수 있습니다.

```
<div class="menu">
  <a v-for="작명 in 메뉴들" :key="작명">Home</a>
</div>
```

사용법은 동일하며, 반복 횟수 자리에 array 자료를 넣을 수 있습니다. 이 경우

1. `메뉴들` 안의 자료 개수만큼 반복됩니다.
2. 작명한 변수는 반복될 때마다 `메뉴들` 안의 자료가 됩니다(1회 반복 시 `'Home'`, 2회 반복 시 `'Shop'`).

```
<div class="menu">
  <a v-for="작명 in 메뉴들" :key="작명">{{ 작명 }}</a>
</div>
```

작명한 변수를 실제로 데이터바인딩해보면, a 태그가 3번 생성되며 각각 Home, Shop, About이 출력됩니다. array/object 안의 자료를 하나씩 HTML로 만들어 보여주고 싶을 때 `v-for`가 유용합니다.

### v-for 변수 2개 (인덱스)

`v-for` 안에서는 변수를 2개까지 작명할 수 있습니다.

```
<div class="menu">
  <a v-for="(작명,i) in 메뉴들" :key="i">{{ 작명 }}</a>
</div>
```

소괄호를 열면 작명을 두 개까지 허용합니다.

- 첫째 변수: array 안의 데이터
- 둘째 변수: `0, 1, 2 …`로 1씩 증가하는 정수(반복 횟수)

둘째 변수는 보통 관습적으로 `:key` 안에 넣습니다.

결론적으로 HTML을 복붙하기 힘들거나, 데이터 개수만큼 HTML을 자동 생성하고 싶을 때 `v-for`를 사용하세요.

## Vue 이벤트 핸들러로 클릭 감지하기 (허위 매물 신고 버튼)

> **5분 숙제:** 모든 상품에 신고 버튼과 기능을 만들어오세요. 상품마다 신고 수를 따로 집계해야 하므로, 신고 수 3개를 각각 저장할 공간이 미리 필요합니다.

### 지난 시간 숙제 복습

하드코딩으로 만들었던 상품 목록 div 3개를 `v-for`로 축약해오라고 했습니다.

```
<div v-for="(a,i) in products" :key="i">
  <h4>{{ products[i] }}</h4>
  <p>50만원</p>
</div>
```

반복문을 돌리면서 `products[i]`를 상품명으로 출력합니다. `a`라고 써도 동일합니다.

### @click 이벤트 핸들러

어떤 HTML 요소를 클릭했을 때 동작을 일으키고 싶으면, 자바스크립트에서는

```
<div onclick=""> 이 안에 자바스크립트를 집어넣습니다.
```

Vue에서는 `@click=""`을 사용합니다. 부동산 사이트에 허위 매물 신고 버튼과 기능을 만들어보며 배워봅니다.

### 신고 버튼과 신고 수 만들기

반복문이 복잡해 보이니 다시 원래대로 복구합니다.

```
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
</div>
<div>
  <h4>{{ products[1] }}</h4>
  <p>50만원</p>
</div>
<div>
  <h4>{{ products[2] }}</h4>
  <p>50만원</p>
</div>

```

여기에 버튼을 추가합니다.

```
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
  <button>허위매물신고</button>
  <span>신고수 : 0</span>
</div>
```

버튼을 누르면 신고 수가 1 증가하는 기능을 만들 것입니다. 먼저 신고 수를 기록할 데이터가 필요합니다.

```
data(){
  return {
    신고수 : 0,
  }
}
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
  <button>허위매물신고</button>
  <span>신고수 : {신고수}</span>
</div>
```

① 하단에 데이터를 만들고 ② HTML에 꽂아 넣었습니다. 버튼을 누르면 `신고수`를 1 증가시키고 HTML도 재렌더링되어 원하는 기능이 완성됩니다.

### 버튼을 누르면 기능 실행하기

자바스크립트는 `onclick=""`을 썼지만 Vue에서는 `@click=""`을 사용하며, 안에 자바스크립트를 자유롭게 입력할 수 있습니다.

```
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
  <button @click="신고수++">허위매물신고</button>
  <span>신고수 : {신고수}</span>
</div>
```

버튼을 누를 때마다 `신고수`가 +1 됩니다. `신고수++`, `신고수+=1` 등 자유롭게 작성할 수 있습니다.

`@click` 외의 이벤트 핸들러도 만들 수 있습니다.

- `@mouseover`: 마우스를 댔을 때 실행
- `@input`: 인풋에 값을 입력했을 때 실행
- 그 외 알고 있는 이벤트명을 자유롭게 기입하면 됩니다.

### 코드가 길면 함수(methods)를 만들어 씁니다

긴 코드를 축약하는 게 함수입니다. `@click` 안에 들어갈 내용이 길면 함수를 만들어 넣으세요. 함수를 만드는 자리는 `methods : {}` 항목입니다.

```
data(){
  return {
    신고수 : 0,
  },
  methods : {
    increase(){
      this.신고수 += 1
    }
  }
}
```

- `methods` 항목 안에 `함수이름(){}` 형태로 함수를 계속 만들 수 있습니다.
- **꼭 기억할 점:** 여기서 데이터를 가져다 쓰려면 `this.데이터이름`이라고 써야 합니다. `this`는 위에 있는 데이터와 함수를 담은 큰 object라고 생각하면 됩니다.

HTML 부분에서 함수를 호출하면 `this.신고수+=1`이 실행됩니다.

```
<div>
  <h4>{{ products[0] }}</h4>
  <p>50만원</p>
  <button @click="increase()">허위매물신고</button>
  <span>신고수 : {신고수}</span>
</div>
```

- `함수()` 형태로 호출해도 되고, `@click="increase"`처럼 함수 이름만 써도 됩니다.
- 함수는 한글로 작명하면 잘 안 될 수 있습니다.

## v-if와 모달창 만들기 (동적 UI 만드는 법)

> **5분 숙제:** 모달창 닫기 버튼과 기능을 만들어오세요.

### 지난 시간 숙제 복습 (신고 버튼 3개)

버튼과 신고 수 표시를 3개 만들어두고, 각각 신고 수가 다르게 동작해야 합니다.

```
<button>허위매물신고</button>
<span>신고수 : 0</span>

<button>허위매물신고</button>
<span>신고수 : 0</span>

<button>허위매물신고</button>
<span>신고수 : 0</span>
```

상품이 3개이므로 신고 수를 기록할 공간도 3개 필요합니다. array 자료로 만들었습니다.

```
data(){
 return {
   신고수 : [0,0,0]
 }
}
```

그리고 다음처럼 처리하면 기능 개발 끝입니다.

- 0번째 상품에는 `신고수[0]` 표기, 0번째 버튼 누르면 `신고수[0]++`
- 1번째 상품에는 `신고수[1]` 표기, 1번째 버튼 누르면 `신고수[1]++`
- …

![상품별 신고 수 동작 화면](/assets/20220104_134934.png)

이런 식으로 하면 성공합니다. 다만 그대로 따라 하지 말고 직접 해보는 것이 가장 중요합니다.

### 모달창이란

오늘은 상품을 누르면 뜨는 상세 페이지를 모달창 스타일로 만들어봅니다(실제 페이지 이동은 아님).

![모달창 UI 예시](/assets/20220104_134845.png)

모달창은 배경이 어둡고 안에 흰 창이 있는 UI입니다. 상품명을 클릭하면 이걸 띄울 것입니다.

### 동적 UI 만드는 2-Step

모달창만 만들 수 있는 사람이 되면 안 되므로, Vue로 동적 UI를 만드는 일반적인 Step을 정리합니다.

1. 현재 HTML UI의 상태를 데이터로 저장해둠 (보이는지/안 보이는지 등)
2. 그 상태에 따라 HTML UI를 보여줄지 말지를 Vue 문법으로 작성함

이것만 기억하면 모달창 외에 탭, 서브메뉴 등 클릭으로 동작하는 UI를 스스로 만들 수 있습니다.

### 잠깐, 이미지 넣는 법

이미지를 `src` 폴더 안 아무 곳에 넣고 아래처럼 적으면 이미지가 나옵니다.

```
<img src="./경로">
```

- `public` 폴더에 넣을 거면 `/경로`라고 적습니다.
- `public` 폴더에 넣으면 발행 시 이미지 이름이 변하지 않습니다(`src`는 임의로 바꿔줌).
- 이미지가 크면 `width`, `margin-top` 등으로 스타일링하세요.

### 모달창 만들기

갑자기 등장하는 UI는 미리 만들어놓고 원할 때 띄우는 방식입니다. 그러니 모달창부터 HTML/CSS로 만듭니다.

```
<div class="black-bg">
 <div class="white-bg">
   <h4>상세페이지</h4>
   <p>상세페이지내용임</p>
 </div>
</div>
body {
 margin : 0;
}
div {
 box-sizing: border-box;
}
.black-bg {
 width: 100%; height:100%;
 background: rgba(0,0,0,0.5);
 position: fixed; padding: 20px;
}
.white-bg {
 width: 100%; background: white;
 border-radius: 8px;
 padding: 20px;
}
```

이 HTML은 위쪽에 적는 것이 좋습니다.

### 2-Step 적용

**Step 1. 현재 모달창의 상태를 데이터로 저장**

모달창은 보이는/안 보이는 상태만 존재하므로 데이터로 표현합니다.

```
data(){
 return {
   모달창열렸니 : true,
 }
}
```

`true/false`로 나타냈지만 `0`/`1`로 표현해도 무방합니다. 미리 채워놓는 값은 기본값입니다.

**Step 2. 상태에 따라 보여줄지 말지를 v-if로 작성**

HTML 태그에 `v-if="조건식"`을 쓰면 조건식이 참일 때만 보여줍니다.

```
<div class="black-bg" v-if="모달창열렸니 == true">
 <div class="white-bg">
   <h4>상세페이지</h4>
   <p>상세페이지내용임</p>
 </div>
</div>
```

이렇게 하면 `모달창열렸니`가 true일 때만 div를 보여줍니다.

이제 `모달창열렸니` 데이터를 조작하면 모달창을 켜고 끌 수 있습니다. UI를 만들 때는 일종의 스위치를 미리 만들어둔다고 생각하면 됩니다.

> **Q.** 상품 제목을 눌렀을 때 모달창을 열려면(`모달창열렸니` 스위치를 켜려면) 코드를 어떻게 짜야 할까요? 앞 강의에서 배운 내용으로 직접 해보세요.

> **오늘의 상식:** `data(){}` 부분을 리액트 등에서는 **state**라고 부릅니다. 정보를 저장하는 곳이자 UI의 현재 상태를 저장하는 곳이며, 상태는 영어로 state입니다.

## 실제 데이터로 상품 목록 만들기

> **5분 숙제:** 반복문으로 상품 목록을 6개 만들어오세요. 실제 상품 데이터가 잘 반영되어 있어야 합니다.

### 실제 상품 데이터 준비

오늘은 실제 원룸 데이터를 HTML에 꽂아 보여줍니다. 보통은 서버에서 데이터를 받아오지만, 우리는 가짜 데이터를 직접 준비합니다.

데이터를 그대로 `data()`에 붙여넣으면 길고 복잡해지니, **import / export** 문법으로 다른 js 파일에 보관한 뒤 가져옵니다.

### export default / import 문법

어떤 js 파일에서 만든 변수나 자료를 다른 js 파일에서 쓰고 싶으면 export와 import를 씁니다. 다른 파일에서 export 해야 import 할 수 있습니다(수출을 해야 수입할 수 있음).

예를 들어 `oneroom.js`의 변수를 `App.vue`에서 쓰고 싶은 경우:

```
(oneroom.js)
var apple = 10;
export default apple
(App.vue)
import 어쩌구 from './oneroom.js파일경로'
```

- `export default` 옆에 내보낼 변수/자료를 적습니다.
- 가져다 쓰려면 `import 작명 from 경로`를 적습니다. 작명한 변수를 출력하면 export 했던 `10`이 나옵니다.
- `export default`는 파일 맨 마지막에 딱 한 번만 사용 가능합니다.
- import 시 작명은 자유롭게 가능합니다.

#### export {} / import 문법

export 할 게 많으면 `export {}` 문법을 사용합니다.

```
(oneroom.js)
var apple = 10;
var apple2 = 100;
export { apple, apple2 }
(App.vue)
import { apple, apple2 } from './oneroom.js파일경로'
```

- `export`는 원하는 만큼 사용 가능합니다.
- import 시에는 작명이 불가능하고, export 했던 변수명을 그대로 적어야 합니다.

### 원룸 데이터를 import 해서 저장하기

긴 데이터를 `oneroom.js`에 저장해두고, `App.vue`의 데이터로 가져옵니다.

```
(oneroom.js)
export default 위에있던원룸데이터
(App.vue)
import data from './oneroom.js파일경로'

data(){
  return {
    원룸들 : data
  }
}
```

`원룸들`이라는 이름으로 저장했습니다. 이 데이터는 `[ {}, {}, {}, {}, {}, {} ]` 형태이며, `{}` 하나마다 하나의 상품 데이터(상품명, 가격 등)가 들어 있습니다.

첫째 상품에 데이터바인딩하려면:

```
<div>
  <img src="{{ 원룸들[0].image }}">
  <h4>{{ 원룸들[0].title }}</h4>
  <p>{{ 원룸들[0].price }}</p>
</div>
```

이렇게 하면 첫째 상품의 제목이 데이터바인딩됩니다. 중괄호/대괄호가 무엇인지는 강의를 참고하세요.

## 모달창에 상세 페이지 만들기

> **5분 숙제:** 오늘 만든 모달창 안의 내용을 꾸며보세요. 상품명, 상품 설명, 가격, 이미지 정도만 기입하면 됩니다.

### 지난 시간 숙제 복습

```
<div v-for="(a, i) in 원룸들" :key="i">
  <img :src="a.image" class="room-img">
  <h4 @click="모달창열렸니 = true">{{ a.title }}</h4>
  <p>{{ a.price }}</p>
</div>
```

`원룸들`을 반복문 돌리면

1. `원룸들` 안의 자료 개수만큼 반복됩니다(`<div>`가 6회 생성).
2. 첫째 변수 `a`는 안에 있던 자료(하나의 `{}`)가 됩니다.

제목을 누르면 모달창을 여는 코드도 작성해두었습니다.

### 모달창 안에 실제 상세 내용 출력하기

모달창에 실제 상품의 상세 내용을 출력하려고 합니다. 첫째 상품을 누르면 첫째 상품의 정보를, 둘째 상품을 누르면 둘째 상품의 정보를 모달창에 출력하려면, 이전의 2-Step을 그대로 적용하면 됩니다.

```
<div class="black-bg" v-if="모달창열렸니 == true">
  <div class="white-bg">
    <h4>{{ 원룸들[0].title }}</h4>
    <p>상세페이지내용임</p>
  </div>
</div>
```

이렇게 쓰면 `원룸들` 데이터 중 0번째 제목이 모달창에 들어갑니다. 그런데 항상 0번이 아니라, 사용자가 방금 누른 상품 번호가 들어가야 합니다.

```
<div class="black-bg" v-if="모달창열렸니 == true">
  <div class="white-bg">
    <h4>{{ 원룸들[누른거].title }}</h4>
    <p>상세페이지내용임</p>
  </div>
</div>
```

1번 상품을 누르면 1이, 2번 상품을 누르면 2가 되는 변수가 필요합니다. 그래서 `누른거`라는 데이터를 신설했습니다.

```
data(){
  return {
    누른거 : 0
  }
}
```

여기에 "사용자가 누른 상품 번호"를 저장합니다. 이게 1로 바뀌면 1번 상품 제목이, 2로 바뀌면 2번 상품 제목이 모달창에 보입니다.

이제 사용자가 1번 상품 제목을 누르면 `누른거`를 1로 바꾸면 됩니다.

```
<div v-for="(a, i) in 원룸들" :key="i">
  <img :src="a.image" class="room-img">
  <h4 @click="모달창열렸니 = true; 누른거 = i">{{ a.title }}</h4>
  <p>{{ a.price }}</p>
</div>
```

각 상품명을 누르면 `누른거`가 바뀌도록 만들었습니다. X번 상품 제목을 누르면 `누른거`가 X로 바뀝니다.

**총정리**

1. X번 상품을 누르면 `누른거` data가 X로 바뀜
2. 그러면 모달창 안의 `{{ 원룸들[누른거].title }}`도 실시간 재렌더링됨(관련 data가 바뀌었으니까)
3. 그러면 모달창 제목이 `{{ 원룸들[누른거].title }}`로 나옴

이 또한 동적 UI 2-Step과 같습니다. 상세 페이지도 일종의 UI이므로 ① 상태를 미리 저장하고 ② 그 상태에 따라 HTML이 어떻게 보일지 작성하면 스위치가 완성됩니다.

### v-else 문법

`v-if`는 조건식이 참일 때만 보여줍니다. `v-else`도 있습니다.

```
<div v-if="1 == 2">
  안녕하세요
</div>
<div v-else>
  안녕하세요2
</div>
```

`v-if`의 조건식이 참이 아닐 경우 `v-else`를 보여줍니다. 조건에 따라 이거 또는 저거를 보여주고 싶을 때 사용합니다.

```
<div v-if="1 == 2">
  안녕하세요
</div>
<div v-else-if="1 == 3">
  안녕하세요2
</div>
```

자바스크립트 `else if`처럼, 조건식을 연달아 검사하고 싶을 때 `v-else-if`를 사용합니다.

## 컴포넌트(Component) 만들어 쓰기

### 지난 시간 숙제 복습 (모달창 꾸미기)

```
<div class="black-bg" v-if="모달창열렸니 == true">
  <div class="white-bg">
    <img :src="원룸들[누른거].image" style="width:100%">
    <h4>{{ 원룸들[누른거].title }}</h4>
    <p>{{ 원룸들[누른거].content }}</p>
    <p>{{ 원룸들[누른거].price }} 원</p>
    <button @click="모달창열렸니 = false">닫기</button>
  </div>
</div>
```

### 컴포넌트란

HTML을 짜다 보면 `<div>` 수십 개가 나옵니다. 이게 싫으면 컴포넌트를 만들어 씁니다. **컴포넌트는 원하는 HTML 덩어리를 한 글자로 축약해주는 문법**입니다.

예를 들어 할인 배너를 만들어봅니다.

```
<div class="discount">
  <h4>지금 결제하면 20% 할인</h4>
</div>
```

이 코드가 너무 길고 복잡하다고 가정하면, 긴 HTML을 짧게 축약하는 컴포넌트 문법을 쓰면 됩니다.

### 컴포넌트 만드는 법

`어쩌구.vue` 파일을 아무 곳에 만든 뒤 축약할 HTML을 붙여넣습니다. `.vue` 파일은 항상 다음 형식으로 만듭니다(예: `Discount.vue`).

```
<template>
  축약할 HTML~~
</template>

<script>
export default {
  name : '작명',

}
</script>
<style>
  넣을 스타일
</style>
```

이 vue 파일 하나를 컴포넌트라고 부릅니다. `name : 어쩌구`로 작명하면 나중에 디버깅할 때 쓰입니다.

### 컴포넌트 쓰는 법

원하는 곳에서 `.vue` 파일을 ① import 하고 ② 등록하고 ③ 씁니다.

```
import Discount from './Discount.vue 경로'

export default {
  data() {

  },
  components : {
    Discount,
  }
}
```

1. import 하고
2. `components` 항목 안에 등록합니다. 이미 `components`가 있으면 그것을 쓰고 없으면 만듭니다.
3. 그러면 `<template>` 안에서 자유롭게 사용할 수 있습니다.

```
<Discount>
<Discount/>
```

### Q. 굳이 왜 쓰나?

- `<div>`가 여러 개 있는 게 보기 싫으면 씁니다.
- 다른 곳에서도 같은 HTML 재사용이 쉬워집니다.
- 나중에 라우터로 페이지를 구분하고 싶으면 하나의 페이지를 하나의 컴포넌트로 만드는 게 좋습니다.

### 모달창도 컴포넌트화하기

전에 만든 모달창 코드는 매우 길고 복잡하니, `Modal.vue`를 만들고 직접 축약해보세요.

다만 이렇게 쓰면 문제가 있습니다.

> 모달창이 안 열리는데요?

데이터바인딩에 문제가 생겨서 그렇습니다. 해결은 다음 강의에서 합니다.

컴포넌트를 무작정 많이 만들면 코드가 오히려 복잡해질 수 있으니, 만들 이유가 있을 때 만드세요.

## props로 부모 데이터를 자식에게 전해주기

> **15분 숙제:** 예전에 만든 상품 목록 6개를 `<Card/>` 컴포넌트로 바꿔오세요. 컴포넌트로 바꾸면 모달창 여는 `@click` 코드가 고장날 수 있는데, 그 문제는 직접 해결해보세요.

### 모달창이 안 열리는 이유

지난 시간 `<Modal/>` 컴포넌트를 만들었지만 모달창이 잘 안 뜹니다. `Modal.vue` 코드를 보면

```
<div class="black-bg" v-if="모달창열렸니 == true">
  <div class="white-bg">
    <img :src="원룸들[누른거].image" style="width:100%">
    <h4>{{ 원룸들[누른거].title }}</h4>

(생략)
```

`원룸들`, `누른거` 변수를 데이터바인딩했는데, 이 데이터가 `Modal.vue` 안에 없어서 문제가 생긴 것입니다(데이터바인딩하려면 데이터가 그 파일에 있어야 함).

그렇다고 데이터를 `Modal.vue`에 복사하면 안 됩니다. 사본을 만들면 업데이트 시 모든 복사본에 반영하기 어렵기 때문입니다. 옳은 방법은 `App.vue`의 데이터를 **props**로 전송해서 쓰는 것입니다.

### 용어 정리: 부모와 자식

`App.vue` 안에 `Modal.vue`를 넣어 쓰고 있다면

- `Modal` = 하위(자식) 컴포넌트
- `App` = 상위(부모) 컴포넌트

![부모-자식 컴포넌트 구조](/assets/20220104_172208.png)

### props 3-Step

부모 → 자식으로 props 문법을 통해 전송해야 자식이 데이터를 사용할 수 있습니다.

1. 보내고
2. 등록하고
3. 쓰면 됩니다 (3번은 생략해도 에러 안 남)

**① 보내기 (App.vue)**

```
<Modal :원룸들="원룸들" />
```

데이터바인딩 문법으로 데이터를 Modal로 보냅니다. `<Modal :작명="하단의데이터이름" />` 형태이며, 여기서는 이름을 통일했습니다.

> 콜론 `:`의 역할은 2개입니다. 데이터바인딩 또는 props 전송.

**② 등록하기 (Modal.vue)**

```
<script>
  export default {
    name : 'Modal',
    props : {
      원룸들 : Array,
    }
  }
</script>
```

자식 컴포넌트는 데이터를 받으면 `props : {}`를 열고 `{ 데이터이름 : 자료형 }`을 적습니다. 자료형엔 `Array`, `Object`, `String`, `Number` 등을 적습니다.

**③ 쓰기**

props로 등록한 것은 HTML 안에서 데이터바인딩으로 자유롭게 사용할 수 있습니다.

`원룸들`, `누른거`, `모달창열렸니` 데이터를 `App.vue`에서 `Modal.vue`로 보내면 모달창이 동작합니다. 단, 닫기 버튼은 주석 처리하세요. props를 수정하면 에러가 납니다.

> **props는 받아서 사용만 하는 read-only 데이터**로 취급해야 하며 수정하면 안 됩니다.

### Q. 애초에 Modal.vue에 데이터를 저장하면 안 되나?

해당 데이터를 `Modal.vue` 안에서만 쓸 거라고 자신하면 거기 만들어도 됩니다. 하지만 데이터를 만들 때 원칙이 있습니다.

> **데이터를 사용하는 컴포넌트들 중 최상위 컴포넌트에 데이터를 만들어야 함**

데이터는 위로 전송하는 게 복잡하고 추적이 어렵기 때문입니다. 귀찮으면 그냥 모두 `App.vue`에 저장하는 것도 나쁘지 않습니다.

> **오늘의 교훈:** 컴포넌트로 많이 쪼개면 props 보내는 게 귀찮으니, 만들기 전에 "만들면 이득이 있을까"를 생각하세요.

## props 나머지 내용

props로 데이터를 전송할 때, 굳이 하단의 데이터가 아니더라도 문자, 숫자, array, object 자료를 직접 넣을 수 있습니다.

### props 보내는 여러 방법

```
<Discount :데이터이름="[1,2,3]" />
<Discount :데이터이름="{ age:20 }" />
<Discount :데이터이름="100" />
<Discount 데이터이름="안녕하쇼" />
```

각각 array, object, 숫자, 문자 보내기입니다. **콜론을 안 붙이면 문자로 전달됩니다.**

object 자료를 각각의 속성으로 나눠 보내고 싶은 경우:

```
오브젝트 : {name : 'kim', age : 20}
```

```
<Discount :데이터이름="오브젝트.name" :데이터이름="오브젝트.age"  />
```

이게 번거로우면 다음 특별한 문법을 사용해도 됩니다.

```
<Discount v-bind="오브젝트명" />
```

다만 이러지 말고 그냥 object 자료를 통째로 보내는 게 낫습니다.

### 지난 시간 숙제: 상품 목록을 `<Card>` 컴포넌트화하기

**① Card.vue 만들기**

```
<template>
  <div>
    <img :src="a.image" class="room-img">
    <h4>{{ a.title }}</h4>
    <p>{{ a.price }} 원</p>
   </div>
</template>
```

(밑은 생략)

`v-for`까지 넣어 상품 6개를 전부 `<Card>`로 만들 수도 있지만, 여기서는 상품 한 개만 `<Card>`로 축약했습니다(취향입니다). 모달창 여는 기능은 나중에 할 거라 잠깐 지웠습니다.

그리고 `App.vue`에서 import, 등록, 사용합니다.

```
<Card/>
import Card from './Card.vue'

export default {
  components : {
    Card
  }
}
```

아무것도 안 나오는 이유는 `Card.vue` 안의 `a.title`에서 `a`가 비어 있기 때문입니다. `a`는 반복문을 돌릴 때 나오는 하나하나의 상품 데이터였습니다. 이를 props로 전송하면 상품명, 가격이 제대로 나옵니다.

**② a에 필요한 정보를 props로 보내기 (App.vue)**

`원룸들 : [ {}, {}, {}, {}, {}, {} ]` 안의 하나의 `{}` 데이터를 그대로 props로 보냅니다.

```
<Card :원룸="원룸들[0]" />
```

`원룸`이라는 이름으로 첫째 상품 데이터를 보냈습니다.

**Card.vue에서 등록 후 사용**

```
<template>
  <div>
    <img :src="원룸.image" class="room-img">
    <h4>{{ 원룸.title }}</h4>
    <p>{{ 원룸.price }} 원</p>
   </div>
</template>


<script>
export default {
  props : {
    원룸 : Object
  }
}
</script>
```

`원룸`이라는 이름으로 props를 보냈으니 데이터바인딩 시 `원룸.title` 등으로 쓰면 됩니다. 이제 Card가 첫째 상품으로 잘 보입니다. 나머지 5개 상품은 직접 만들어보세요(`<Card>`에 반복문을 돌려도 됩니다).

## custom event로 자식이 부모 데이터 바꾸기

> **5분 숙제:** 모달창 닫기 버튼과 기능을 다시 만들어오세요. 데이터를 어떻게 변경해야 모달창이 닫히는지 생각해봅시다.

### 자식은 부모 데이터를 직접 바꿀 수 없다

자식 컴포넌트가 부모의 데이터를 바꾸고 싶을 때가 있지만, 다른 파일에 있으므로 직접 바꿀 수 없습니다. props로 전해준 데이터도 변경하면 안 됩니다(부작용 방지를 위해 Vue가 변경을 막아둠).

![custom event 개념도](/assets/20220104_215222.png)

대신 자식은 부모에게 "데이터 좀 바꿔줘"라고 **메시지**를 보냅니다. 부모는 그 메시지를 수신하면 데이터를 변경하는 코드를 짜둡니다. 이것이 **custom event** 문법입니다.

**custom event 요약**

1. 자식: `$emit(작명, 전달할자료)`로 부모에게 메시지를 보냅니다(자료 전달은 선택).
2. 부모: `@작명="데이터변경하는JS코드"`로 메시지를 수신해 데이터를 변경합니다.

### 제목 누르면 모달창 여는 기능 고치기

`Card.vue` 안에 상품 제목이 있고, 제목을 누르면 모달창을 열고 싶습니다.

```
<template>
  <div>
    <img :src="a.image" class="room-img">
    <h4 @click="모달창열렸니 = true">{{ a.title }}</h4>
    <p>{{ a.price }} 원</p>
   </div>
</template>
```

이렇게 하면 안 됩니다. `모달창열렸니`는 부모(App.vue)의 데이터이기 때문입니다. 자식은 부모에게 부탁만 할 수 있으며, 그 방법이 `$emit('작명', 전달할자료)`입니다. `$emit`은 Vue가 제공하는 '부모에게 부탁하는' 함수입니다(전문 용어로 '커스텀 이벤트 전송').

```
<template>
  <div>
    <img :src="a.image" class="room-img">
    <h4 @click="$emit('openModal')">{{ a.title }}</h4>
    <p>{{ a.price }} 원</p>
   </div>
</template>
```

제목을 누르면 `openModal`이라는 이름으로 커스텀 이벤트를 전송합니다.

부모(App.vue)는 부탁을 받으면 데이터를 수정합니다.

```
<Card @openModal="모달창열렸니 = true"  />
```

이제 상품 제목을 누르면 모달창이 열립니다. 다만 다른 상품을 눌러도 같은 모달창만 열리는 문제가 있습니다.

### custom event로 부모에게 자료 보내기

예전엔 `@click="모달창열렸니 = true; 누른거 = i"`처럼 몇 번 상품을 눌렀는지도 함께 처리해야 했습니다. 그래서 누른 상품 번호도 전달해야 합니다.

```
<template>
  <div>
    <img :src="a.image" class="room-img">
    <h4 @click="$emit('openModal', 원룸.id)">{{ a.title }}</h4>
    <p>{{ a.price }} 원</p>
   </div>
</template>
```

`$emit()`의 둘째 파라미터에 원하는 자료를 넣으면 부모까지 전달됩니다. `원룸.id`를 넣었습니다(0번 상품은 id가 0, 1번 상품은 id가 1).

부모는 `$event` 변수에 그 자료가 담깁니다.

```
<Card @openModal="모달창열렸니 = true; 누른거 = $event"  />
```

이렇게 하면 예전과 똑같은 기능이 됩니다.

**요약**

- 자식: `$emit(작명, 전달할자료)`로 부탁
- 부모: `<자식컴포넌트 @작명="부탁받으면할일">`로 수신
- 하단에서 함수로 만들어 쓸 때는 `this.$emit()`이라고 해야 합니다.

## 사용자의 input 받기 (v-model)

### 지난 시간 숙제 복습 (모달창 닫기 버튼)

```
<button @click="모달창열렸니 = false">닫기버튼</button>
```

`모달창열렸니`는 `App.vue`의 데이터라 `Modal.vue`에서 직접 못 바꿉니다. 부모 데이터를 수정하려면 커스텀 이벤트로 메시지를 줍니다.

```
<button @click="$emit('closeModal')">닫기버튼</button>
```

```
<Modal @closeModal="모달창열렸니 = false">
```

`Modal.vue`는 메시지를 보내고, `App.vue`는 수신하는 코드를 작성합니다.

### input 입력값을 data로 저장하기

사용자 입력으로 여러 UI 기능(상품 수량 변경, 총금액 계산 등)을 만들 수 있습니다. 그러려면 input 입력 내용을 `data(){}`에 저장하는 법을 알아야 합니다. `@input`, `@change` 이벤트 핸들러를 써도 되고, 더 짧은 `v-model`을 써도 됩니다.

`Modal.vue`에 input을 만들고, 입력한 수량에 맞는 최종 가격을 모달창에 보여주려고 합니다.

```
<template>
  (생략)
  <input @input="month = $event.target.value">
</template>
<script>
export default {
  data(){
    return {
      month : 0
    }
  }
}
</script>

```

1. `@input`은 `@click`과 같은 이벤트 핸들러로, 사용자가 input에 입력할 때 동작합니다(유사품 `@change`).
2. `$event`는 Vue가 제공하는 특별한 변수로 event object를 뜻합니다. JS의 `addEventListener('click', function(e){})`에서의 `e`와 같은 의미입니다. `$event.target.value`로 input 입력값을 가져옵니다.
3. 그걸 `month` data에 저장합니다.

> **Q.** 사용자가 수량을 10이라고 입력하면 최종 가격도 ×10 하려면 코드를 어떻게 짜야 할까요? 직접 해보세요.

### v-model 문법

다음 코드를

```
<input @input="month = $event.target.value">
```

이렇게 바꿔도 똑같이 동작합니다.

```
<input v-model="month">
```

`v-model`은 "여기 입력된 값을 data로 바로 저장해주세요"라는 문법입니다. 따옴표 안에 하단의 data 이름만 적으면 됩니다.

- (참고) `<input>` 외에 `textarea`, `select`, `input type="checkbox"` 등에도 적용 가능합니다.
- (참고2) 사용자가 input에 적은 값은 무조건 문자입니다(`123`도 `'123'`으로 저장). `v-model.number="month"`처럼 directive를 쓰면 숫자가 들어올 때 숫자로 저장합니다. 다만 숫자 변환만 해줄 뿐 `'ㄱㄴㄷ'` 문자 입력은 막지 못합니다(막는 건 다음 시간에).

## watcher로 데이터 감시하기

> **5분 숙제:** `month`에 숫자가 아니라 문자가 들어오면 "숫자만 입력하라"는 `alert()`을 띄우고 `month`의 초기값을 1로 다시 설정해봅시다.

사용자가 input에 입력한 데이터는 무조건 문자입니다. `v-model.number="데이터이름"`을 쓰면 숫자 자료로 변환해 저장하지만, `'abc'` 입력을 막을 순 없습니다. 이걸 막고 싶으면 **watcher**를 씁니다.

`watch : {}` 항목은 어떤 데이터를 계속 감시하는 역할을 합니다. 감시란 특정 데이터가 변경될 때마다 실행되는 코드를 적을 수 있다는 뜻입니다.

### watch 사용법

```
export default {
  data(){
    return {
      month : 1
    }
  },
  watch : {
    month(){
      //month가 변경될 때 실행할 코드
    }
  }
}
```

- `watch` 항목에 감시자를 만듭니다.
- 함수명을 감시하고 싶은 데이터명으로 작명합니다. `month()`로 지으면 `month` 감시자가 됩니다.
- 함수 안에 `month`가 변할 때마다 실행할 코드를 적습니다.
- `month(a)`처럼 파라미터를 작명하면, 그 파라미터는 `month`가 변경될 값을 의미합니다(두 개까지 작명 가능: 첫째는 변경될 값, 둘째는 변경 전 값).

### Q. month가 12보다 크게 변하면 경고문을 띄우려면?

자바스크립트의 `alert('안녕')`으로 경고문을 띄울 수 있습니다.

```
export default {
  data(){
    return {
      month : 1
    }
  },
  watch : {
    month(a){
      if (a > 12) {
         alert('13이상 입력하지마')
      }
    }
  }
}
```

이메일 형식, 숫자/문자 여부, 비밀번호 대문자 포함 여부 등 검사할 경우가 많습니다. 정규식을 쓰거나, 체크할 게 많으면 Vue용 form validation 라이브러리를 사용하기도 합니다.

### 오늘의 숙제 해설

자바스크립트로 자료가 숫자인지 판단하는 방법:

- `isNaN()`: 숫자를 넣으면 false, 글자를 넣으면 true(`'123'` 같은 숫자처럼 보이는 글자는 false).
- `typeof`: 문자면 `'string'`, 숫자면 `'number'`.

input 데이터는 무조건 문자라 `typeof`는 항상 `'string'`이 나올 가능성이 큽니다. 그래서 `isNaN()`이 더 유용합니다(`v-model.number`를 쓰면 또 다를 수 있음).

`month` 감시자 안에서 `isNaN()`에 넣어보고 true면 alert를 띄웁니다.

```
watch : {
    month(a){
      if (isNaN(a) == true){
        alert('문자입력하지마라');
        this.month = 1;
      }
    },
 },
```

문자를 입력한 경우 `month` 초기값도 1로 설정했습니다. 다만 스페이스바 입력 같은 경우는 거를 수 없으니 직접 해결해보세요.

## Vue에서 UI 애니메이션 주는 법 2가지

UI 등장/퇴장 시 애니메이션을 주고 싶을 때, ① 순수 CSS 애니메이션을 쓰거나 ② Vue가 제공하는 `<transition>` 태그를 쓸 수 있습니다.

### 1. 순수 CSS로 애니메이션 주기

1. 애니메이션 시작 전 class를 디자인
2. 애니메이션 동작 후 class를 디자인
3. 원할 때(예: 모달창 열릴 때) 동작 후 class를 부착
4. 시작 전 class에 `transition` 속성을 주면 부드러운 애니메이션 완성

조건부로 class를 부착하고 싶을 때:

```
<div :class="{ 클래스명 : true }"> </div>
```

오른쪽이 true(또는 truthy)일 때만 부착됩니다. 버튼을 누르면 저걸 true로 바꾸면 됩니다(true 대신 데이터명을 넣어도 됨).

### 2. Vue의 `<transition>` 태그 이용하기

1. 애니메이션을 주고 싶은 UI를 감쌉니다.

```
<transition name="작명"></transition>
```

2. CSS로 다음 스타일을 줍니다.

```
.작명-enter-from { 애니메이션 동작 전 상태 }
.작명-enter-active { 애니메이션 동작 중 상태, 대부분 transition 이런거 }
.작명-enter-to { 애니메이션 동작 후 상태 }
```

UI 등장 시 애니메이션이 동작합니다. 퇴장 애니메이션을 주려면 스타일에서 `enter`를 `leave`로 바꾸면 됩니다.

## 상품 정렬 기능과 데이터 원본 보존

> **혼자 해볼 숙제:** 상품명 가나다순 정렬은? 가격 역순 정렬은? 50만원 이하의 상품만 보여주는 필터 기능은? 답은 없으니 직접 해보세요.

버튼을 누르면 상품이 가격순으로 정렬되는 기능을 만들어봅니다. `원룸들` 데이터(`[ {}, {}, {}, {}, {}, {} ]`)를 가격순으로 정렬만 하면 됩니다. HTML이 어떻게 보일지는 신경 쓸 필요 없습니다. 데이터바인딩을 잘 해두었으면 데이터가 바뀔 때 HTML도 자동으로 반영됩니다.

### 정렬 버튼 만들기

```
<button @click="priceSort()" >가격순정렬</button>
methods : {
  priceSort(){
    this.원룸들.sort()
  }
}
```

자바스크립트 `.sort()` 함수 설명:

```
var array = [2,5,1];
array.sort();
```

이렇게 쓰면 array가 가나다순(문자 기준)으로 정렬됩니다.

```
var array = [2,5,1];
array.sort(function(a,b){
  return a - b
});
```

이렇게 쓰면 숫자 오름차순으로 정렬됩니다. `a`, `b`는 array 안의 하나하나의 자료를 뜻하며, `return` 오른쪽 값이 음수면 `a`를 왼쪽, 양수면 `a`를 오른쪽으로 보냅니다. 내림차순으로 하려면 어떻게 짜야 할지 생각해보세요.

### 프로젝트에 적용하기

```
<button @click="priceSort()" >가격순정렬</button>
methods : {
  priceSort(){
    this.원룸들.sort(function(a,b){
      return a - b
    })
  }
}
```

`원룸들`은 `[ {}, {}, {}, {}, {}, {} ]` 형태로, `{}` 안에 이름, 가격 등이 들어 있습니다. 그래서 `return a - b`라고만 쓰면 안 됩니다(`{} - {}`는 음수/양수가 안 나옴). `return` 오른쪽이 음수나 양수가 나오도록 가격끼리 비교하는 코드를 작성하세요.

### 순서 원래대로 되돌리기 (원본 보존)

```
<button @click="sortBack()" >되돌리기</button>
methods : {
  sortBack(){
    ???
  }
}
```

원래 순서로 돌아오게 하려면, 사본을 만들어 원본을 보존하는 방법이 좋습니다. `원룸들` data를 똑같이 두 개 만듭니다.

```
data(){
  return {
    원룸들오리지널 : data,
    원룸들 : data
  }
}
methods : {
  sortBack(){
    this.원룸들 = this.원룸들오리지널
  }
}
```

오리지널 원본을 만들어두고, 되돌리기 버튼을 누르면 원본을 가져와 집어넣는 식인데, **실은 안 됩니다.**

array나 object에 `=` 등호로 뭔가를 넣으면 양쪽이 값을 공유하라는 뜻이기 때문입니다(reference data type 참고). 그래서 위처럼 `sortBack()`을 디자인하면 안 됩니다.

```
data(){
  return {
    원룸들오리지널 : [...data],
    원룸들 : [...data]
  }
}
methods : {
  sortBack(){
    this.원룸들 = [...this.원룸들오리지널]
  }
}
```

array를 복사하거나 값 공유가 일어나지 않게 넣고 싶으면 `[...array자료]`를 사용합니다. 이렇게 하면 안전하게 원본 array를 집어넣을 수 있어 되돌리기 버튼이 잘 동작합니다.

### spread operator (점 3개 `...`) 정리

자료의 사본을 만들 때 가장 쉬운 방법은 등호입니다.

```
var a = '안녕하세요';
var b = a;
```

등호는 오른쪽 값을 대입하라는 뜻이므로 `a`, `b` 둘 다 "안녕하세요"가 됩니다.

그런데 array, object를 등호로 복사하면 문제가 생깁니다.

```
var a = [1,2,3];
var b = a;
```

이건 사본을 만들라는 뜻이 아니라 `a`와 `b`가 `[1,2,3]`을 공유하라는 뜻입니다. 이제 `b`를 수정하면 `a`에도 반영됩니다. object도 마찬가지입니다.

그래서 `this.원룸들 = this.원룸들오리지널`은 원본을 넣으라는 뜻이 아니라 둘이 값을 공유하라는 뜻이 됩니다. 이 상태에서 `원룸들.sort()`를 하면 `원룸들오리지널`도 정렬되어 원본을 보존할 수 없습니다.

값 공유가 일어나지 않는 독립적인 복사본을 만들려면:

```
var a = [1,2,3];
var b = [...a];
```

`...`는 **spread operator**로, array/object 앞에 붙여 괄호를 제거하는 문법입니다. `...[1,2,3]`은 `1,2,3`만 남깁니다. array를 해체했다가 다시 array로 만들면 서로 값을 공유하지 않는 별개의 사본이 됩니다. 왜 그냥 복사하면 값 공유가 일어나는지는 reference data type을 찾아보세요.

## Vue의 라이프사이클(Lifecycle)

> **숙제 1.** 메인페이지 방문하자마자 30%라고 적힌 할인 문구가 1초마다 1%씩 감소하려면? (`setInterval()`이 필요할 수 있음)
>
> **숙제 2.** 모달창 내 input에 2를 기입했을 때 `alert()`을 띄우려면? watcher 말고 오늘 배운 lifecycle hook을 이용합시다. (힌트: 데이터가 변경되어 컴포넌트가 재렌더링되는 것을 update라고 부릅니다.)

![Vue Lifecycle 다이어그램](/assets/20220105_013913.png)

Vue 공식 문서의 **Lifecycle**은 컴포넌트가 어떤 step으로 생성/삭제/업데이트되는지 설명합니다. 이를 알아야 **Lifecycle hook**을 쓸 수 있습니다.

### 컴포넌트의 생명주기

1. 컴포넌트를 보여줄 때 `create → mount` 단계로 생성됩니다. `create`는 데이터 생성, `mount`는 `index.html`에 장착이라고 생각하면 됩니다.
2. 데이터가 바뀌어 재렌더링될 때는 `update` 단계를 거칩니다.
3. 다른 페이지로 이동하는 등 컴포넌트가 삭제될 때는 `unmount` 단계를 거칩니다.

이 단계 중간중간에 코드를 실행시키고 싶을 때(mount 전에 ajax로 데이터 가져오기, update 전에 데이터 검증 등) Lifecycle hook을 골라 씁니다.

```
beforeCreate()
created()
beforeMount()
mounted()
beforeUpdate()
updated()
beforeUnmount()
unmounted()
```

함수명만 봐도 기능을 알 수 있습니다. `beforeUpdate()`는 컴포넌트가 update되기 전에, `mounted()`는 mount된 후에 코드를 실행시키고 싶을 때 씁니다.

```
data(){
  return {

  }
},
mounted(){
  어쩌구~
}
```

컴포넌트 파일 하단에 이렇게 쓰면 mount 후 `어쩌구~` 코드를 실행합니다. 특히 서버에서 데이터를 가져오는 코드는 보통 `mounted()`나 `created()`에 작성합니다.

### 숙제 1 해설: 1초마다 1%씩 감소

`App.vue`에 할인 문구를 p 태그로 만듭니다. 30% 수치를 데이터로 만듭니다(자주 변하는 데이터이므로).

```
(App.vue)
<p>지금 결제하면 {{ amount }}% 할인</p>

(App.vue 하단 데이터만드는 곳)
data(){
 return {
    amount : 30,
  }
}
```

페이지 로드 시 30을 1초마다 1씩 감소시킵니다. `setInterval()`을 `mounted()` 안에 넣습니다.

```

(App.vue 하단)

mounted(){
  setInterval(()=>{
      this.amount--;
  }, 1000);
}

```

**직접 해볼 응용**

- 숫자가 0 밑으로 안 떨어지게 하려면? (0이 되면 타이머 삭제)
- Discount 컴포넌트를 이용하려면?

### 숙제 2 해설: input에 2 입력 시 alert

모달창 input에 입력하면 HTML이 재렌더링됩니다. 이를 라이프사이클 용어로 Modal 컴포넌트가 `update`된다고 표현합니다(일부분만 바뀌어 보여도 Modal 전체가 재렌더링됨).

update 전에 코드를 실행하기 위해 `Modal.vue`에 라이프사이클 훅을 겁니다.

```
beforeUpdate(){
    if (this.month == 2){
      alert('2개월은 너무 적음.. 안팝니다')
    }
 }
```

업데이트 전에 `month`를 검사하고, 그 값이 2면 원하는 동작을 실행합니다.

**직접 해볼 응용**

- 2를 입력하면 이걸 3이나 4로 강제로 바꿔버리는 건 어떨까요?
