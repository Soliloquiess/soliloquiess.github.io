### 지난 시간 숙제 & style 속성 데이터바인딩

> **지난 시간 숙제:** 인스타그램 포스팅 데이터 3개를 `App.vue`에 보관한 뒤 `Post.vue`에 데이터바인딩해서 3개의 게시물이 잘 보이게 하기. props 문법을 잘 쓰면 됩니다.

### 풀이 흐름

1. `postdata.js`로 긴 데이터를 보관
2. 그걸 `App.vue`에 데이터로 집어넣음
3. 게시물 데이터를 `App.vue` → `Container.vue`로 props 전송
4. 게시물 데이터를 `Container.vue` → `Post.vue`로 props 전송
5. `Post.vue`에서 데이터바인딩

별거 아니고 props 보내는 문법을 몇 번 쓰면 됩니다. 답안을 따라 적기만 했다면, 지우고 다시 직접 해보는 것을 추천합니다.

### 다음 시간으로 넘어가기 전에

`Container.vue` 안에 `<Post/>`가 3개 있을 텐데

```
<Post/>
<Post/>
<Post/>
```

이 3개를 반복문으로 짧게 줄여보세요.

### 이미지를 꽂아넣으려면

`Post.vue`에 프로필 이미지, 포스팅 이미지 두 개를 `background-image : url()`로 꽂아 넣어야 합니다(CSS 레이아웃을 그렇게 짰기 때문). `Post.vue`를 보면

```
<div class="profile"></div>
<div class="post-body"></div>
```

각각 프로필 이미지, 포스팅 이미지를 배경으로 넣습니다. 즉 `style="background-image: url(이미지경로)"`를 넣으면 됩니다. 그 안에 변수/데이터를 넣으려면 콜론을 붙여(`:style="자바스크립트~~"`) 데이터바인딩하면 됩니다.

### style 속성 데이터바인딩하는 법

`:style=""` 안에 자바스크립트 데이터를 넣을 일이 많습니다. 그냥 콜론을 붙여 데이터바인딩하면 되지만, 코드가 길어지면 보기 싫습니다. 그래서 편의를 위한 여러 방법을 지원하는데, 가장 자주 쓰는 게 **object**입니다.

```
<div :style="{ fontSize : '20px', marginTop : '10px' }">
<div :style="{ color : 'red' }">
```

`style=""`에 object 형태로 여러 CSS 속성을 넣을 수 있습니다. object 자료이므로 CSS 속성명에 대시(`-`)를 쓸 수 없어, 대문자로 바꿔 붙여 씁니다(전문 용어로 **CamelCase**). 데이터 상태에 따라 스타일 변화가 필요할 때(애니메이션 등)도 유용합니다.

```
<div class="post-body" :style="{ backgroundImage : 'url(이미지경로ㄷㄷ)' }"></div>
```

실제 `Post.vue`에 이미지를 넣을 때 이렇게 해도 됩니다. 이미지 경로 데이터를 문자 중간에 변수처럼 넣으려면, 자바스크립트에서 문자 중간에 변수를 넣는 두 방법 중 하나를 씁니다.

```
'문자' + 변수 + '문자'
`문자 ${변수} 문자`
```

### 서버로 ajax 요청하는 더보기 버튼

> **15분 숙제:** 버튼을 한 번 더 누르면 두 번째 게시물을 가져와 보여주도록 코드를 짜오세요. `https://codingapple1.github.io/vue/more1.json`으로 GET 요청하면 둘째 게시물을 줍니다.

강의에서 사용할 URL: `https://codingapple1.github.io/vue/more0.json` — 여기로 GET 요청하면 더보기용 게시물을 하나 줍니다.

### 서버란

서버는 그냥 요청하면 데이터를 주는 프로그램일 뿐입니다(네이버 웹툰 서버 = 웹툰 달라고 하면 주는 프로그램). 서버에 데이터를 요청하려면 **GET 요청 / POST 요청**을 씁니다.

- **GET**: 데이터를 가져올 때 사용. 서버가 요구하는 URL을 적습니다.
- **POST**: 서버로 데이터를 보낼 때 사용. 역시 서버가 요구하는 URL을 적습니다.

더보기 버튼을 누르면 서버로 추가 게시물을 요청할 텐데, 임시 서버를 위 URL로 만들어두었습니다.

### Ajax 요청은 axios로

`fetch`라는 자바스크립트 기본 함수를 써도 되지만 호환성 때문에 **axios**를 자주 쓰게 됩니다. 터미널에서 `npm install axios`(yarn이면 `yarn add axios`)를 입력합니다(미리보기 띄우던 터미널은 끄고 설치).

```
import axios from 'axios';

axios.get('서버URL').then( 결과 => {
  GET요청 성공시 실행할 코드~~
  console.log(결과);
})
```

원하는 URL로 GET 요청을 합니다. `.then()` 안에 콜백 함수를 추가하고, 그 안에 성공 시 실행할 코드를 적습니다. 가져온 데이터는 `결과` 파라미터에 담겨 있습니다.

```
import axios from 'axios';

axios.get('서버URL').then( 결과 => {
  GET요청 성공시 실행할 코드~~
}).catch( ()=>{
  실패시 실행할 코드
})
```

요청 실패 시 실행할 코드는 `.catch` 안에 적습니다. URL이 틀리거나 서버가 다운되면 실패할 수 있습니다.

```
import axios from 'axios';

axios.post('서버URL', '보낼데이터').then( 결과 => {
  POST요청 성공시 실행할 코드~~
}).catch( ()=>{
  실패시 실행할 코드
})
```

POST 요청은 서버로 원하는 데이터(문자, object 등)를 전송할 수 있습니다.

### 데이터 중심 개발

GET 요청으로 게시물을 하나 더 가져온 뒤, 그것으로 `<Post>`를 하나 더 생성합니다. `<Post>`는 게시물 데이터 개수만큼 반복문을 돌려 생성하도록 해놨습니다. 따라서 여러분이 짤 코드는 "`<Post>` HTML 하나 추가"가 아니라 **"게시물 데이터를 하나 증가"**입니다. Vue는 항상 데이터 중심으로 개발합니다.

> **숙제:** 버튼을 한 번 더 누르면 두 번째 게시물(`more1.json`)을 가져와 보여주도록 코드를 짜오세요.

### 숙제 풀이

버튼을 한 번 누르면 `more0.json`, 두 번 누르면 `more1.json`을 요청하도록 짰습니다. 버튼 누른 횟수를 기록할 데이터가 필요합니다.

```
data(){
    return {
      게시물 : postdata,
      더보기 : 0,
    }
 },
 ```

그다음 `more()` 함수:

```
more(){
      axios.get(`https://codingapple1.github.io/vue/more${this.더보기}.json`)
      .then( 결과 => {
        this.게시물.push(결과.data);
        this.더보기++;
      })
 }
 ```

`더보기`가 0이면 `more0.json`, 1이면 `more1.json`으로 GET 요청하고, 성공 시 `더보기`를 ++ 했습니다.

### 탭 만들기 & 사진 업로드 페이지

### 탭 UI 만드는 법

```
<div>내용0</div>
<div>내용1</div>
<div>내용2</div>
<button>버튼0</button>
<button>버튼1</button>
<button>버튼2</button>
```

탭은 버튼1을 누르면 내용1만 보이는 UI입니다. 예전에 배운 동적 UI 만드는 법을 그대로 씁니다.

1. 현재 탭의 상태를 data로 저장 (지금 몇 번째 내용이 보여야 하는지)
2. data에 따라 HTML이 어떻게 보일지 작성

```
data(){
  return {
    step : 0
  }
}
```

탭의 상태를 `0, 1, 2`로 표현할 수 있으니 숫자로 저장했습니다.

```
<div v-if="step == 0">내용0</div>
<div v-if="step == 1">내용1</div>
<div v-if="step == 2">내용2</div>
<button>버튼0</button>
<button>버튼1</button>
<button>버튼2</button>
```

`step`이 0이면 0번 내용만 보입니다. 이제 버튼0을 누르면 `step`을 0으로, 버튼1을 누르면 `step`을 1로 변경하면 탭이 동작합니다.

### 서버 없이 업로드한 이미지 다루기 (잡기술)

> **5분 숙제:** 업로드한 이미지를 step 1, step 2 화면의 파란 박스란에 보여주려면 코드를 어떻게 짜야 할까요? `background-image : url()`로 넣되, 유저가 업로드한 이미지를 넣어야 합니다.

오늘은 Vue 문법이 아니라 업로드한 이미지 다루는 법을 알아봅니다. 웹개발을 잘하려면 Vue 문법뿐 아니라 웹브라우저 기능을 많이 알아야 합니다. Vue는 브라우저 다루는 걸 도와주는 도구일 뿐입니다.

옛날엔 업로드한 이미지를 바로 보여주기 어려워, 서버로 보내 저장하고 그 URL을 받아 `<img src="">`에 넣어 보여줬습니다. 하지만 IE11 이상에선 브라우저 내에서 직접 이미지 조작이 가능합니다.

1. `FileReader()`: 이미지를 글자로 변환(저장/전송 가능)
2. `URL.createObjectURL()`: 이미지 URL을 잠깐 만들어줌(새로고침하면 사라짐, 가상 호스팅 같은 개념)

여기서는 2번을 사용합니다.

### 업로드 버튼 누르면 이미지를 URL로 만들기

```
<input @change="upload()" type="file" id="file" />
```

```
methods : {
 upload(e){
   let 파일 = e.target.files;
   let url = URL.createObjectURL(파일[0]);
   console.log(url);
   this.step++
 }
}
```

코드 3줄 설명:

1. `e.target.files`: 업로드한 파일을 리스트로 알려줍니다(파일 하나만 올려도 `[]`에 담김).
2. `URL.createObjectURL()`: 업로드한 파일을 담으면 가상 URL을 생성합니다. `<img src="">`에 넣으면 이미지가 보입니다.
3. 업로드하면 다음 페이지로 넘어가도록 `this.step++`.

### 글 발행 기능 만들기

> **5분 숙제:** `App.vue`의 발행 버튼을 누르면, 업로드한 이미지와 작성한 글이 적용된 새 게시물이 메인페이지에 뜨도록 만들어봅시다.

지난 시간 이미지 띄우는 숙제는, 이미지 URL을 `App.vue`의 데이터로 저장하고 props로 `Container.vue`에 보내 적절한 곳에 데이터바인딩하는 식으로 했습니다.

### Next 버튼 기능 개발

상단 Next 버튼을 누르면 다음 step으로 넘어가고, 메인페이지(step이 0)에선 안 보여야 합니다. `@click`과 `v-if`를 함께 답니다.

```
<li v-if="step == 1" @click="step++">Next</li>
```

### 발행 버튼 기능 개발

발행을 누르면 업로드한 이미지와 작성한 글이 `<Post/>` 하나로 메인페이지에 떠야 합니다. `<Post/>` HTML 생성은 Vue가 알아서 합니다(게시물 개수만큼 반복문을 돌려놨으니). 그러니 발행 버튼을 누르면 **게시물 데이터를 3개에서 4개로 늘리면** 됩니다.

발행 버튼은 Next 버튼 옆에 만들고, `step == 2`일 때만 보이게 합니다.

```
<li v-if="step == 1" @click="step++">Next</li>
<li v-if="step == 2" @click="publish()">발행</li>
```

이제 `publish()` 함수에서 게시물 array에 내 게시물을 하나 추가합니다.

### 게시물 발행 기능

```
methods : {
  publish(){
      var 내게시물 = {
        name: "Kim Hyun",
        userImage: "https://placeimg.com/100/100/arch",
        postImage: "https://placeimg.com/640/480/arch",
        likes: 36,
        date: "May 15",
        liked: false,
        content: "내가입력한글",
        filter: "perpetua"
      };
      this.게시물.unshift(내게시물);
      this.step = 0;
    },
}
```

게시물 data에 들어있는 `{}`와 똑같은 자료를 만들어 `this.게시물`에 추가합니다. `unshift()`는 array에 데이터를 하나 추가하는 JS 문법입니다. 업로드 후 `step`을 0으로 이동시킵니다. 지금은 임시 게시물이 발행되니, 실제 입력한 이미지와 글이 발행되도록 숙제로 해보세요.

### 5분 숙제 답안

임시 게시물 대신 실제 유저가 입력한 글과 이미지를 `var 내게시물 = {}`에 넣으면 됩니다.

**유저가 입력한 글 전송 (Container.vue → App.vue)**

유저가 입력한 글은 `Container.vue`의 `<textarea>`에 있습니다. 커스텀 이벤트로 `App.vue`에 전송했습니다.

`App.vue`는 이 데이터를 저장할 공간을 만들고, 이벤트를 수신하면 저장합니다.

```

data(){
  return {
    작성한글 : '',
  }
}
```

이제 유저가 `@input` 할 때마다 `App.vue`의 `작성한글`에 데이터가 저장됩니다.

### props 대신 slot 쓰기

> **오늘의 상식:** Vue에선 `<div>`에 `class=""`를 여러 개 사용할 수 있습니다.
> ```
> <div :class="데이터" class="filter-item"> </div>
> ```
> 이렇게 작성해도 잘 먹습니다.

### slot 문법

자식이 부모 데이터를 사용하려면 props로 전송해야 합니다(전송 → 등록 → 사용). 이게 귀찮으면 간편한 대용 문법이 **slot**입니다.

1. 자식: `<slot></slot>` 태그로 데이터가 꽂힐 곳을 정해둡니다.
2. 부모: `<자식컴포넌트>데이터</자식컴포넌트>`처럼 자식 컴포넌트 사이에 데이터를 작성해 보냅니다.

그러면 부모의 데이터가 자식의 `<slot>` 자리에 자동으로 꽂힙니다. props 등록이 필요 없습니다.

> 유의점: props는 `src`, `style` 속성 등에서도 쓸 수 있지만, slot은 HTML 태그이므로 HTML 태그처럼만 쓸 수 있습니다.

#### Named Slots

부모가 전해줄 데이터가 많아 `<slot>`을 여러 개 쓰고 싶을 때, `name` 속성으로 구분합니다.

1. 자식 컴포넌트:

```
<slot name="a"></slot>
```

2. 부모:

```
<자식컴포넌트>
  <template v-slot:a>데이터</template>
</자식컴포넌트>
```

`v-slot:` 옆에 이름을 적으면 특정 slot에 데이터를 보낼 수 있습니다(예제는 `a` slot에 전송). 이름은 자유 작명 가능합니다.

#### Slot Props

부모가 slot으로 데이터를 전해줄 때, 가끔 자식의 데이터를 사용하고 싶을 때 씁니다.

1. 자식 컴포넌트:

```
<slot :데이터="데이터"></slot>
```

자식이 가진 데이터를 부모로 보낼 수 있습니다.

2. 부모:

```
<자식컴포넌트>
  <template v-slot:default="작명"> </template>
</자식컴포넌트>
```

`v-slot:` 옆에 `default`라고 적으면 자식이 보낸 데이터를 수신해 slot 사용 시 활용할 수 있습니다(실은 그럴 일이 별로 없음).

### 멀리 있는 컴포넌트 간 데이터 전송: mitt

> **숙제:** 필터를 고르면 ① 그 필터가 업로드한 이미지에도 적용되고 ② 글을 발행했을 때도 선택한 필터가 적용되어 보여야 합니다.

#### mitt 라이브러리

멀리 떨어진 컴포넌트에 데이터를 전송할 일이 있습니다. 강의에선 `FilterBox.vue`에서 `App.vue`까지(상위-상위 컴포넌트) 전송하고 싶은데, 그럴 경우 코드가 매우 길어집니다. 이럴 때 **mitt** 외부 라이브러리를 쓰면 쉽게 전송됩니다.

Vue 2에서는 EventBus를 썼지만 이제 못 쓰므로 mitt를 따로 설치합니다(`npm install mitt` 또는 `yarn add mitt`).

`main.js`를 이렇게 수정합니다.

```
import { createApp } from 'vue'
import App from './App.vue'

import mitt from 'mitt'
let emitter = mitt();
let app = createApp(App);
app.config.globalProperties.emitter = emitter;

app.mount('#app')
```

`app.config.globalProperties`는 모든 컴포넌트에서 사용할 수 있는 변수를 등록하는 object(글로벌 변수 보관함)입니다. 자주 쓰는 axios도 여기 보관하면 import 없이 `this.axios.get`으로 쓸 수 있어 편리합니다.

#### mitt 사용법

데이터를 보내고 싶은 곳에서:

```
this.emitter.emit('이벤트명작명', '데이터')
```

데이터를 수신하고 싶은 곳에서:

```
this.emitter.on('이벤트명작명', (a)=>{
  데이터수신시 실행할 코드
  a는 출력해보면 데이터나옴
})
```

보통 수신 코드는 `mounted()` 안에 적습니다. 자바스크립트 이벤트 리스너와 사용법이 비슷합니다.

### Vuex 1: 사용하는 이유

> **지난 시간 숙제:** 필터를 고르면 ① 그 필터가 업로드한 이미지에 적용되고 ② 발행 시에도 선택한 필터가 적용되어 보이게 하기.

**풀이**

1. `FilterBox` 안 div를 클릭하면 mitt로 이벤트를 발사(필터명을 넣어서)
2. `Container`, `App`은 이벤트 수신 시 데이터로 저장

그다음 그 데이터를 알맞은 위치(Container의 큰 이미지 부분, App의 `내게시물` 변수)에 꽂았습니다.

#### Vuex 쓰는 이유

Vuex는 필수 라이브러리가 아닙니다. 다음 경우에 씁니다.

1. **props와 custom event로 데이터를 주고받기 힘들 때.** Vuex를 설치하면 js 파일 하나에 모든 데이터를 저장할 수 있고, 모든 컴포넌트가 그 데이터를 직접 꺼내 쓰고 수정할 수 있습니다. props 없이 모든 컴포넌트가 데이터에 직접 접근 가능합니다.
2. **Vue 파일과 데이터가 너무 많을 때.** Vuex는 상태(데이터) 관리 라이브러리입니다. `name` 데이터를 컴포넌트 100개에서 쓰는데 버그가 나면 100곳을 다 뒤져야 하지만, Vuex는 데이터와 수정 방법을 한 곳에서 관리하므로 디버깅이 쉽습니다. 큰 프로젝트엔 필수지만, 지금 프로젝트에선 배보다 배꼽이 클 수 있습니다.

설치는 `https://next.vuex.vuejs.org/`에서 명령어를 확인하거나, 프로젝트 생성 시 vuex를 선택해도 됩니다.

#### Vuex 셋팅

`src` 안 아무 곳에 `store.js`를 만들고 다음 코드를 작성합니다.

```
import { createStore } from 'vuex'

const store = createStore({
  state(){
    return {

    }
  },
})

export default store
```

그다음 `store.js`를 `main.js`에 등록해야 합니다(등록 안 하면 큰일 남).

```
import store from './store.js'
app.use(store).mount('#app')
```

이제 `store.js`에 저장한 데이터를 모든 컴포넌트가 가져다 쓸 수 있습니다. 데이터는 `store.js`에 저장하고, 출력은 vue 파일에서 `{{ $store.state.어쩌구 }}`로 합니다. 함수나 mounted 등에서는 `this.$store.state.어쩌구`로 씁니다.

### Vuex 2: state 데이터 바꾸는 법

> **숙제:** 인스타그램 사진을 누르면 좋아요가 +1, 다시 누르면 -1 되게 만들어보세요. 좋아요 개수를 `store.js`에 저장한 후 `Post.vue`에 꽂아 넣고, 좋아요/취소 기능을 만듭니다. 사진 3개 중 일단 하나만 만들어봅시다.

### state는 store.js만 수정하게

vue 파일에서 state를 직접 수정하면 안 됩니다(해도 되지만 그러면 Vuex 쓸 이유가 없음). `name : 'kim'` state를 100개 vue 파일에서 수정하는데 버그가 나면 100개를 다 뒤져야 합니다. 이를 방지하기 위해

> **state 수정은 store.js만 할 수 있게 코드를 짜는 게 좋습니다.**

그러면 state 버그 발생 시 `store.js`만 뒤지면 됩니다. 이를 'state 관리가 용이하다'고 하며, 그래서 Vuex를 상태 관리 라이브러리라고 부릅니다.

#### mutations로 데이터 수정 방법 정의

`mutations`라는 object 항목을 만들어 state 수정 방법을 함수로 정의하고, 이 함수를 부르는 방식으로 사용합니다.

예를 들어 버튼을 누르면 `age` state에 1을 더하고 싶으면, 먼저 `age`의 수정 방법을 만듭니다.

```
const store = createStore({
  state () {
    return {
      name : 'kim',
      age : 20,
    }
  },
  mutations :{
    한살더하기(state){
      state.age++
    }
  },
}
```

`한살더하기()`를 만들었습니다. 파라미터 하나를 추가하면 위의 state를 뜻하므로, 그걸 가지고 마음대로 수정하면 됩니다.

vue 파일에서 버튼을 누르면 `store.js`에게 `한살더하기()` 실행을 부탁합니다(정중히).

```
<button @click="$store.commit('한살더하기')">버튼</button>
```

vue 파일은 `$store.commit(함수명)` 형식에 맞춰야 부탁할 수 있습니다. 그러면 버튼을 누를 때 `store.js`가 `한살더하기()`를 실행해 `age`에 1이 더해집니다. 이것이 Vuex의 state 수정 방법입니다.

Vuex를 쓰면 state 수정이 귀찮아지지만, 나중에 버그 찾으려고 Vue 파일 100개를 뒤지는 것보단 낫습니다.

### 좋아요 & 좋아요 취소 기능 (숙제 해설)

> **지난 시간 숙제:** 사진을 누르면 좋아요 +1, 다시 누르면 -1. 좋아요 개수를 `store.js`에 저장 후 `Post.vue`에 꽂아 넣고 좋아요/취소 기능 만들기.

좋아요를 눌렀는지 여부를 어딘가에 저장해둬야 동작을 판단할 수 있습니다.

1. 좋아요를 눌렀는지를 state로 저장
2. 사진 클릭 시, 누른 상태면 좋아요 -1
3. 안 눌렀으면 좋아요 +1

좋아요를 +1 수정할 때는 당연히 `mutations` 함수를 만들고, vue 파일은 그걸 부탁하는 식으로 사용합니다.

**(응용) 모든 사진에 좋아요 기능을 개발하려면?**

지금은 사진 하나만 동작합니다. 모든 사진에 "이 사진 좋아요 눌렀냐"는 데이터를 저장해둬야 합니다. 사진이 3개면 `[false, false, false]` 같은 array를 써도 됩니다. 가장 좋은 방법은 게시물 정보 전부를 Vuex로 옮겨 거기에 기록하는 것입니다(데이터를 찢지 말고 한 곳에 모으는 게 좋음).

### Vuex 3: actions 항목

Vuex에선 state를 수정할 때 `mutations` 함수를 만들어 쓰라고 배웠는데, 가끔 서버에서 데이터를 가져와 수정하고 싶을 때가 있습니다. 그럴 때의 ajax 요청은 `mutations`에 직접 적지 않고 **actions**에 적어야 합니다.

왜냐하면 `mutations`에 ajax처럼 오래 걸리는 코드를 넣으면 나중에 길게 코드 짤 때 힘들어지기 때문입니다.

예를 들어 `name`을 먼저 수정하고 그다음 `age`를 수정하고 싶다고 합시다.

```
name수정하는함수()
age수정하는함수()
```

그런데 `name수정하는함수()` 안에 3초 넘게 걸리는 ajax 요청이 있다면

```
name수정하는함수()  //3초걸림
age수정하는함수()
```

`age` 수정이 먼저 될 수도 있습니다. 자바스크립트는 ajax처럼 오래 걸리는(정확히는 비동기 처리를 지원하는) 코드를 잠깐 제껴두고 다음 줄부터 실행하려는 경향이 있기 때문입니다. 그러면 의도치 않은 버그가 생기고 코드가 길어질 수 있습니다.

그래서 state를 수정하는 `mutations` 함수는 ajax 없이 순수하게 state 변경만 하는 함수로 만들고, ajax는 `actions`에 맡깁니다.

#### actions 만드는 법

`mutations`처럼 `actions` 항목을 만든 후 함수 형태로 만듭니다.

```
actions : {
  데이터가져오기(){
    axios.get('').then(()=>{
      성공시 실행할 코드
    })
  }
}
```

`App.vue` 등에서 `$store.dispatch('데이터가져오기')`라고 사용하면 데이터를 가져옵니다. actions 함수를 부탁할 땐 `$store.dispatch`라고 정중히 부탁합니다.

데이터를 가져온 직후 그것으로 state도 변경하고 싶으면:

```
actions : {
  데이터가져오기(context){
    axios.get('').then(()=>{
      context.commit('mutations함수명')
    })
  }
}
```

state 변경은 무조건 `mutations` 함수를 만들어 써야 하고, `mutations` 함수는 예외 없이 `commit()`으로 써야 합니다. 그걸 쓰려면 함수에 `context` 파라미터(자유 작명 가능)를 추가하면 됩니다. 그 파라미터는 `$store` 변수 같은 거라고 생각해도 좋습니다.

복잡하지만 익숙해지면 코드 관리가 좋다는 걸 느낄 겁니다. 프로젝트가 커지면 Vuex는 필수이므로, Vuex로 인스타그램이나 부동산 사이트를 다시 만들어보세요. 강의를 듣는 것보다 처음부터 직접 만들어보는 연습이 중요합니다.

### Vuex 4: mapState

state를 vue 파일에서 꺼낼 때 `$store.state.name`처럼 썼는데, 이게 길고 귀찮으면 **mapState** 함수를 써보세요. 그 전에 **computed** 항목을 알아야 합니다.

### 잠깐, computed

함수는 `methods : {}` 안에 만들라고 했지만, `computed : {}` 안에 만들 수도 있습니다.

```
computed : {
  now2(){
    return new Date()
  }
},
methods : {
  now(){
    return new Date()
  }
}
```

똑같은 기능의 함수를 2개 만들어봤는데 차이는

- `methods` 함수: 함수를 부를 때마다 안의 코드가 실행됨
- `computed` 함수: 함수를 불러도 안의 코드가 실행 안 됨. 컴포넌트 로드 시 한 번 실행되고 그 값을 계속 저장해서 씀

`computed`는 일종의 계산 결과 저장 공간입니다.

- `computed` 함수는 `return`을 꼭 써야 합니다.
- `computed` 함수를 쓸 때는 소괄호 없이 함수명만 씁니다.

### computed로 state 꺼내기

`$store.state.name`을 매번 쓰면 코드가 길고 더러워집니다(`$store.state.name.name2[0].name3` 같은 식). 그럴 땐 자주 꺼내 쓰는 state를 `computed`에 넣어두면 짧게 쓸 수 있습니다.

```
computed : {
  name(){
    return this.$store.state.name
  }
}
```

`<script>` 태그 안에서 `$store`를 쓰려면 `this.$store`입니다. 그러면 위 HTML에서 `{{ name }}`처럼 쉽게 state를 꺼낼 수 있어 HTML이 깔끔해집니다.

### mapState로 computed 짧게

꺼낼 state가 100개면 computed도 100개 만들어야 하니, 그럴 땐 **mapState**를 씁니다. 알아서 computed에 state를 등록해줍니다.

```
import {mapState} from 'vuex'

computed : {
  ...mapState(['state이름1', 'state이름2'])
}
```

위에서 import 해와야 쓸 수 있습니다.

```
import {mapState} from 'vuex'

computed : {
  ...mapState({ 작명 : 'state이름1'})
}
```

object 자료형을 넣으면 state를 가져올 때 이름 변경도 가능합니다.

```
import {mapState, mapMutations} from 'vuex'

computed : {
  ...mapState(['state이름1', 'state이름2']),
  ...mapMutations([ '좋아요', 'setMore' ])
}
```

`mapMutations`를 쓰면 mutations 함수도 쉽게 가져다 쓸 수 있습니다. 이제 `$store.commit('좋아요')`가 아니라 `좋아요()`로 쓸 수 있습니다. `mapActions`도 있으니 코드가 더러워지면 써보세요.

### Progressive Web App(PWA) & 셋팅

우리가 만든 웹사이트도 앱으로 발행할 수 있습니다. **PWA**를 사용하면 웹사이트를 모바일 앱처럼 쓸 수 있습니다. 실상은 "웹페이지 홈화면에 바로가기 추가"를 고상하게 표현한 것이지만, 브라우저 주소창이 제거된 상태로 보여주므로 일반 유저는 앱이라고 생각합니다.

### PWA 발행

라이브러리 설치가 필요합니다.

```
vue add pwa
```

설치만 하면 `npm run build` 시 PWA가 알아서 셋팅되어 나오고, 배포하면 바로 PWA로 사용 가능합니다.

### PWA 구성 요소

브라우저가 사이트를 PWA로 인식하면 "설치하시겠습니까?" 팝업을 띄웁니다. PWA로 인식하려면 두 파일이 필요합니다.

- **manifest.json**: 앱의 이름, 아이콘, 테마 색상 등을 설정하는 파일
- **service-worker.js**: 어떤 html/css/js 파일을 캐싱할지 명시하는 파일(캐싱한 파일은 오프라인에서도 이용 가능)

두 파일은 `npm run build` 시 자동 생성되므로 직접 만들 필요는 없습니다.

### PWA 설정

세부 설정을 바꾸려면 `manifest.json`을 직접 건드리지 않습니다(build 때마다 새로 생성되므로). 대신 프로젝트 폴더 안에 `vue.config.js`를 만들고 아래 코드를 넣습니다.

```
module.exports = {
  pwa: {
    name: '님 앱이름',
    themeColor: '#4DBA87',
    msTileColor: '#000000',
    workboxOptions: {
      exclude: [/\.map$/, /manifest\.json$/, 'index.html']
    }
  }
}
```

구글의 **workbox** 라이브러리 사용법대로 설정을 채웁니다. 앱 이름, 색상을 커스터마이징할 수 있고, `exclude`에는 캐싱하기 싫은 파일을 입력합니다(정규식도 잘 먹음).

### 모바일 앱으로 발행

실제 구글 플레이스토어에 등록 가능한 모바일 앱으로 발행하고 싶으면, Vue 문법으로 iOS/Android 개발을 돕는 툴들이 있습니다(`https://v3.vuejs.org/guide/mobile.html#introduction`).

다만 신규 서비스 출시는 모바일 앱보다 웹이 나을 수 있습니다.

- 앱 설치 유도는 웹 방문 유도 대비 마케팅 비용이 3배 이상
- 인앱결제 수수료 30%
- 앱 개발자 등록 비용
- 버전 업데이트 시 플랫폼 2개를 관리해야 함

### Composition API란

`.vue` 파일이 길어지면, data 100개 / methods 200개 / computed 300개가 흩어져 있어 특정 데이터 관련 기능을 찾으려면 여러 곳을 이동해야 하는 단점이 있습니다(`age` 다루는 곳이 150번줄, 380번줄, 670번줄에 흩어짐). 이게 싫으면 Vue 3부터 제공하는 **Composition API**를 씁니다. computed, methods, watch, data()로 파일을 쪼갤 필요 없이 관련 기능을 한 곳에서 쭉 짤 수 있습니다(문법은 약간 더 귀찮을 수 있음).

### 마이페이지 만들기

팔로워 목록을 보여주는 페이지를 만들며 배웁니다. `MyPage.vue`를 만들고:

```
<template>
<div style="padding : 10px">
  <h4>팔로워</h4>
  <input placeholder="?" />
  <div class="post-header">
    <div class="profile"></div>
    <span class="profile-name">사용자명</span>
  </div>
</div>
</template>
```

`Container.vue`에는:

```
<div v-if="step == 3">
  <MyPage/>
</div>
```

이렇게 컴포넌트를 사용하려면 import 하고 components에 등록합니다.

`App.vue`는:

```
data(){
  return {
    step : 3
  }
}
```

`step`을 3으로 강제 변경합니다.

### Composition API로 데이터 만드는 법

팔로워 이름을 저장할 array 데이터를 하나 만듭니다. 이전엔 `data(){}`에 넣었지만 약간 다릅니다. 기본적으로 `setup()` hook을 만들고 항상 그 안에 코드를 짭니다.

```
import { ref } from 'vue'
export default {
  name : 'mypage',
  setup(){
    let follower = ref([]);

    return { follower }
  },
}
```

`follower : []` 데이터를 만든 것과 같은 효과입니다.

- `ref()` 함수를 import 해와서 `var 데이터이름 = ref(데이터)`에 항상 데이터를 저장합니다. 안 그러면 데이터 변경 시 실시간 재렌더링이 안 됩니다.
- 마지막에 `return { 데이터이름 }`으로 뱉어줘야 HTML란에서 `{{ follower }}`로 사용할 수 있습니다(안 그러면 데이터바인딩 불가).

### Ajax 요청 & 데이터 변경

Composition API라고 ajax 요청법이 다르지 않습니다(axios 그대로 사용). 다만 데이터 수정법이 살짝 다릅니다.

```
import { ref } from 'vue'
import axios from 'axios'

export default {
  name : 'mypage',
  setup(){
    let follower = ref([]);

    axios.get('/follower.json').then((a)=>{
      follower.value = a.data
    })

    return { follower }
  },
}
```

get 요청으로 json 파일을 가져오면 `a.data`에 `[{},{},{},{},{}]` 데이터가 딸려옵니다. 그걸 `follower`에 저장할 때 `follower = a.data`가 아니라 **`follower.value = a.data`**라고 써야 합니다.

굳이 설명하면, `ref()`로 데이터를 만드는 것은 object 자료형으로 데이터를 감싸는 행위이므로 `.value`로 안의 데이터를 수정해야 합니다. 이해할 필요는 없고 그렇게 쓰라고 되어 있으니 그렇게 씁니다.

### Lifecycle hook 사용법

컴포넌트가 부착/업데이트될 때 뭔가 실행하고 싶으면 `created()`, `mounted()`, `beforeUpdate()` 등을 쓸 수 있다고 했습니다. Composition API에서는 함수명이 약간 다릅니다.

```
import { ref, onMounted } from 'vue'
import axios from 'axios'

export default {
 name : 'mypage',
 setup(){
   let follower = ref([]);

   onMounted(()=>{
     axios.get('/follower.json').then((a)=>{
       follower.value = a.data
     })
   })

   return { follower }
 },
}
```

1. `onMounted()` 같은 훅을 import 해옵니다.
2. `onMounted(()=>{ 마운트 후 실행할 코드 })`로 작성합니다.

여기서 lifecycle 함수명은 원래 이름 앞에 `on`만 붙이면 됩니다(`beforeUpdate()` → `onBeforeUpdate()`).

(참고) `created()`는 `onCreated()`로 쓰지 않습니다(그런 함수 없음). `setup()` 공간 자체가 `created()`와 매우 유사하기 때문입니다.

> **오늘의 교훈:** 귀찮으면 Options API를 사용합니다. 이전까지 우리가 쓴 문법이 Options API입니다.

### ref 말고 reactive

데이터를 만들 때 `ref`를 쓰라고 했는데, `reactive`라는 함수도 똑같이 사용 가능합니다(import 필요).

```
import { ref, reactive } from 'vue'

export default {
  setup(){
    let follower = ref([]);
    let test = reactive({name : 'kim'})

    return { follower }
  },
}
```

`ref`든 `reactive`든 같은 역할을 하는데

- `reactive`: object, array 같은 reference data type을 주로 담음
- `ref`: 숫자, 문자 같은 primitive data type을 담음

다만 구분하기 귀찮아서 전부 `ref`를 씁니다. `ref` 함수를 까보면 "그냥 reactive 해주세요"밖에 없습니다(둘 다 같은 함수).

#### Composition API에서 props 사용법

`setup()` 함수 안에서는 `this.props`처럼 props를 가져다 쓸 수 없습니다. 그래서 이렇게 씁니다.

```
import { ref, toRefs } from 'vue'

export default {
  setup(props){
    let follower = ref([]);
    let { 프롭스명 } = toRefs(props)  
    console.log(프롭스명.value)
    return { follower }
  },
}
```

`setup()`에는 파라미터를 두 개까지 넣을 수 있는데, 첫째는 자동으로 props, 둘째는 context가 됩니다(둘째는 잘 안 쓰임). 첫째 파라미터를 출력하면 props가 다 담겨 있습니다.

그걸 그냥 쓰면 안 되고 `ref` 안에 담아 써야 합니다(안 담으면 props가 바뀌어도 실시간 반영 안 됨). 담을 때는 `toRefs`를 쓰는데, 그냥 ref를 여러 번 해주는 함수입니다(import 필요). 등호 왼쪽에 `{props이름, props이름2 ~~}`를 적으면 됩니다. 그럼 `props이름.value`로 props를 꺼낼 수 있습니다.

> props는 자식 컴포넌트가 수정하면 안 됩니다.

##### watch 사용법

`setup()` 안에서 watch로 데이터 변화를 감시하려면:

```
import { ref, watch } from 'vue'

export default {
  setup(props){
    let follower = ref([]);
    watch( 데이터명, ()=>{ 데이터 변화시 실행할 코드 } )
    return { follower }
  },
}
```

`watch` 함수를 import 해와서 이렇게 씁니다. props도 감시 가능합니다.

##### computed 사용법

```
import { ref, computed } from 'vue'

export default {
  setup(props){
    let follower = ref([]);
    let 어쩌구 = computed( ()=>{ return 10 } )
    console.log(어쩌구.value)
    return { follower }
  },
}
```

computed는 데이터 연산 결과를 잠깐 저장하는 곳입니다. import 후 소괄호 안에 연산 결과를 `return`으로 뱉는 함수를 넣고, 그걸 변수에 담아 씁니다. computed도 일종의 데이터라서 `.value`로 찍어야 제대로 출력됩니다.

##### methods 사용법

일반 함수를 만들고 싶을 땐 `methods`에 넣었지만, Composition API에서는

```
import { ref } from 'vue'

export default {
  setup(props){
    let follower = ref([]);

    function hello(){

    }
    return { follower, hello }
  },
}
```

아무 데나 함수를 만들면 그게 methods입니다. 그걸 `return` 안에 적으면 HTML란에서도 사용 가능합니다.

##### Vuex store 사용법

`setup()`에서 `$store.state.name`을 쓰려면:

```
import { ref } from 'vue'
import { useStore } from 'vuex'

export default {
  setup(props){
    let follower = ref([]);
    let store = useStore();
    console.log(store.state.name)
    return { follower }
  },
}
```

1. `useStore`를 vuex에서 import 해옵니다.
2. `useStore()`가 예전의 `$store`와 같은 의미입니다. 이제 state 뽑기, commit, dispatch를 자유롭게 합니다.

(참고) `mapState`는 setup 안에서 못 씁니다. Vuex 5가 출시되면 문법이 약간 달라질 겁니다.

### 검색 기능 숙제 해설

> **숙제:** 팔로워 검색 기능 만들기. (힌트: `array.filter()`를 쓰면 array에서 원하는 것만 남길 수 있음)

흐름은 ① input에 검색어를 입력하면 ② `follower`(`[{},{},{},{},{}]`)에서 검색어가 들어 있는 항목만 `[{},{}]`로 남깁니다. `follower` state가 변하면 HTML은 v-for로 자동 변경됩니다.

**1번을 자바스크립트로 번역**

```
<input placeholder="?" @input="search($event.target.value)" />
setup(){
  let follower = ref([]);
  function search(검색어){

  }

  return {follower, search}
}
```

검색어를 입력할 때마다 `search()`를 실행하고, `$event.target.value`(입력한 검색어)를 파라미터로 넘깁니다.

**filter() 함수 설명**

`filter()`는 array에 붙이는 JS 기본 함수로, 원하는 자료만 남길 때 씁니다.

```
filter((a)=>{ return 조건식 })
```

`a`는 array 안의 자료가 되고, 조건식이 참일 때만 그 자료를 남깁니다.

```
[1,2,3].filter((a)=>{ return a == 2 })
```

이렇게 작성하면 `[2]`만 남습니다.

**2번을 자바스크립트로 번역**

```
setup(){
  let follower = ref([]);
  function search(검색어){
    let newFollower = follower.value.filter((a)=>{
        return a.name.indexOf(검색어) != -1
    });
  follower.value = [...newFollower]
  }

  return {follower, search}
}
```

1. `follower.value`에 `filter()`를 붙이고, 조건식은 `a.name`에 검색어가 들어 있냐로 작성
2. 결과를 `let newFollower =`로 변수에 담음(JS filter 사용법)
3. 그걸 `follower.value`에 집어넣음(안전하게 카피본 사용)

![검색 기능 동작 화면](/assets/20220425_172058.png)

검색은 잘 되는데 문제가 있습니다.

> **Q.** 뭔가 검색하고 백스페이스로 지우면 팔로워가 영구적으로 사라져 있는데요? 검색을 계속하면 팔로워가 텅 빕니다.

버그는 원인을 파악해야 해결됩니다. `follower`가 의심되면 콘솔 출력이나 Vue devtools를 켜보세요. 이 문제는 예전 부동산 사이트의 sort/되돌리기 기능 때와 유사합니다.

1. `s`를 검색하면 `follower`에 `s`가 들어 있는 2명만 `[{},{}]`로 남습니다.
2. `s`를 삭제하면 원래 5명을 다 보여줘야 하지만, 이미 `follower`는 `[{},{}]`만 남아 있습니다.

`follower`를 영구적으로 변형시켜버려 복구를 못 하는 것입니다. 그래서 예전처럼 원본을 보존하면 됩니다.

```
setup(){
    let follower = ref([]);
    let followerOriginal = ref([]);

    onMounted(()=>{
      axios.get('/follower.json').then((a)=>{
        follower.value = a.data;
        followerOriginal.value = [...a.data];
      })
    });

    function search(검색어){
      let newFollower = followerOriginal.value.filter((a)=>{
        return a.name.indexOf(검색어) != -1
      });
      follower.value = [...newFollower]
    }
    return {follower, search}
  },
  ```

1. `followerOriginal`이라는 원본 보존용 state를 만듭니다.
2. `followerOriginal`에 filter를 주고, 그 결과를 `follower.value`에 넣습니다.

이제 검색어를 입력할 때마다 `followerOriginal`에서 filter를 수행하고 그 결과를 `follower`에 반영합니다. `s`를 입력해도 `follower`만 변형되고 `followerOriginal`은 그대로 남으며, `s`를 지우면 `followerOriginal`의 모든 항목을 `follower`에 넣어줍니다.

이해가 안 된다면 자바스크립트 문법과 자료형이 익숙하지 않아서일 뿐입니다. 그래서 자바스크립트 연습 겸 코드를 혼자 짜보라는 것입니다.

**응용:** 대소문자 포함 검색, 자음 검색은 어떻게 할 것인지도 생각해보세요.
