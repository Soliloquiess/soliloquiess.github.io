## Blog 프로젝트 생성과 Bootstrap 4, 5 사용법

> **5분 숙제**
> Bootstrap을 이용하든 뭘 하든 `App.vue`를 블로그 메인페이지스럽게 꾸며오십시오. 형식은 따로 없으니 자유롭게 합시다.

오늘부터는 `vue-router`를 배우기 위해 blog 레이아웃을 간단히 만들어 봅시다.

기존에 만들던 쇼핑몰의 복잡한 코드에 router까지 덮어씌우면, 개념 이해보다는 복잡한 코드를 따라치다 끝나기 때문에, 깔끔한 마음으로 새 프로젝트를 만들어 봅니다.

### 새 프로젝트 만들기

새 프로젝트 만드는 법을 까먹었다면, 처음에 만들었던 작업 폴더를 열고 터미널에서 `vue create 프로젝트명`을 입력하면 된다.

- 예: `vue create blog`

### Bootstrap 라이브러리

이번 프로젝트에서는 **Bootstrap** 라이브러리를 이용해 본다.

- 버튼, 메뉴, 카드, 리스트 같은 UI를 예제 코드 복붙식으로 쉽게 만들 수 있다.
- 한마디로 CSS 짜기 귀찮을 때 쓰는 것이다.

설치 방법은 두 가지가 있다.

1. 손수 css, js 파일을 복붙하는 방법
2. npm을 이용하는 방법

#### 1. css, js 파일 복붙으로 설치

1. Bootstrap을 구글에 검색해서 나오는 공식 사이트에 들어간다.
2. 우측 상단에서 버전(4.x / 5.x)을 선택한다.
3. **Getting started** 메뉴로 들어간다.
4. css, js 파일을 자신의 `index.html`에 그대로 붙여넣으면 끝이다.

#### 2. npm으로 설치

설치 전 주의사항:

- 먼저 `blog` 폴더를 에디터에서 열어야 한다. (폴더를 열지 않고 설치하면 큰일난다.)
- 터미널에서 `npm run serve`가 돌고 있으면 프로세스를 종료한다. (`Ctrl + C` / `Command + C`, 혹은 터미널 휴지통 버튼) 종료하지 않고 설치하면 잘 안 되는 경우가 있다.

그리고 터미널에 다음 명령어를 입력한다. (둘 중 하나 선택)

```
npm install bootstrap (5버전 설치시)
또는
npm install bootstrap@4 jquery popper.js (4버전 설치시)
```

- (참고1) 안정적인 yarn을 써보고 싶다면 `npm install` 대신 `yarn add`를 쓴다.
- (참고2) 정확한 설치 명령어는 바뀔 수 있으니 `getbootstrap.com`의 다운로드 메뉴를 참고하자.

그리고 `main.js` 파일에 다음 코드를 추가해야 설치가 완료된다.

```
import 'bootstrap'
import 'bootstrap/dist/css/bootstrap.min.css'
```

`main.js`에서 맨 마지막 줄보다 위에만 넣으면 된다.

#### 3. BootstrapVue

`BootstrapVue`를 설치해도 되는데, 아직은 Vue 2버전하고만 호환된다. 별 차이는 없다.

### Bootstrap 사용법

1. `getbootstrap.com`에 들어가 원하는 UI(버튼, navbar, 카드 등)를 검색한다.
2. 나오는 HTML 예제 코드를 복사 붙여넣기 한다.
3. 그러면 UI 디자인 끝이다.

CSS 짜는 시간을 크게 단축시켜 주므로 자주 이용한다. 처음부터 직접 짜는 것보다 완성품을 복붙하고 수정하는 게 언제나 빠르다.

비슷한 대용품으로는 material design 느낌의 **Vuetify** 같은 라이브러리도 유명하다. 다만 지금은 그런 디자인 유행이 지났다.

`App.vue`에서 이 컴포넌트를 `<List/>` 형태로 보여주려면 코드를 어떻게 짜야 하는지 생각해 봅시다.

## vue-router 설치와 기본 라우팅

> **5분 숙제**
> `Detail.vue`를 만들어 놓고(내용은 아무렇게나 상세페이지스럽게), `/detail`로 접속하면 `Detail.vue`를 보여주는 코드를 짜오십시오.

페이지를 여러 개 만들고 싶을 때 **라우터**를 사용한다. `vue-router` 라이브러리를 설치하면 라우터를 이용할 수 있다.

그러면 다음과 같은 코드를 짤 수 있다.

- "누군가 `codingapple.com/list`로 접속하면 `List.vue`를 보여주세요"
- "누군가 `codingapple.com/detail`로 접속하면 `Detail.vue`를 보여주세요"

### 라우터 셋팅 방법

**1. vue-router 4버전을 설치한다.**

```
npm install vue-router@4
```

- 터미널에 입력하면 된다. (`yarn add`를 써도 된다.)
- `npm run serve`가 돌고 있으면 끄고 입력하자. 안 그러면 에러가 난다.

**2. 라우터 파일을 만든다.**

`src` 폴더 안 아무 데나 `router.js` 파일을 만들고 아래 코드를 복사 붙여넣는다.

```
import { createWebHistory, createRouter } from "vue-router";

const routes = [
  {
    path: "/경로",
    component: import해온 컴포넌트,
  }
];

const router = createRouter({
  history: createWebHistory(),
  routes,
});

export default router;
```

- 이건 vue-router 만든 사람이 정한 사용법일 뿐이라 굳이 이해할 필요는 없다.
- 이제 `path`, `component` 부분만 수정하면 나만의 페이지들을 만들 수 있다.

**3. main.js에도 라우터를 쓴다고 알려준다.**

```
import router from './router'
createApp(App).use(router).mount('#app')
```

`main.js`를 잘 살펴본 후 이런 모양으로 코드를 수정하면 된다.

**4. 원하는 곳에 `<router-view>`를 추가한다.**

`App.vue`의 HTML 내부 아무 데나 `<router-view></router-view>` 태그를 추가하면, 라우터로 구분된 페이지가 그 자리에 보인다.

**5. 라우트를 만든다.**

예를 들어 누군가 `/list`로 접속하면 `List.vue`를 보여주고 싶으면 `router.js`를 이렇게 수정한다.

```
import List from './components/List.vue';

const routes = [
  {
    path: '/list',
    component: List,
  }
];
```

그러면 `/list`로 접속하면 `List.vue`가 보인다.

- 안 보인다면 props도 잘 전송했는지 확인한다. `List.vue`는 블로그 글이라는 데이터를 props로 전송해 주어야 잘 보인다.
- props 전송 예: `<router-view :블로그글="블로그글"></router-view>`

**6. 라우트를 하나 더 만들고 싶으면**

```

(router.js)

import List from './components/List.vue';

const routes = [
  {
    path: '/list',
    component: List,
  },
  {
    path: '/경로',
    component: 위에서 import 해온 컴포넌트
  }
];
```

이런 식으로 오브젝트 자료만 추가해 주면 된다.

**7. 페이지 이동 링크를 만들고 싶으면**

`/list` 페이지로 이동하는 버튼을 만들고 싶으면 원하는 곳에 다음을 추가한다.

```
<router-link to="/list">이동하기</router-link>
```

- `a` 태그와 비슷하다.
- `to=""` 안에 원하는 경로를 마음대로 설정할 수 있다.

## 상세페이지 200만개 만들기 (URL 파라미터)

저번 시간 숙제로 `/detail`로 접속하면 `Detail.vue`를 보여주라고 했다. `Detail.vue`를 글의 상세페이지로 만들고 싶어서 낸 숙제다.

그런데 문제가 있다. 지금 블로그에 글이 3개니까 상세페이지가 3개 필요하다. 그럼 URL을 어떻게 만들어야 효율적일까?

개발 중 모르는 점이 생기면 남의 사이트 벤치마크가 최고다. 네이버 블로그 같은 사이트들을 보면:

![URL이 /글번호 형태로 디자인된 예시](/assets/20220105_172242.png)

- URL이 `/어쩌구/글번호` 형태로 디자인되어 있다.
- `/어쩌구/0`을 입력하면 0번 글, `/어쩌구/1`을 입력하면 1번 글을 보여준다.

### 라우트를 3개 만들면?

우리도 똑같이 만들어 보자. route를 3개 만들면 될까?

```
(router.js)
const routes = [
  {
    path: '/detail/0',
    component: Detail,
  },
  {
    path: '/detail/1',
    component: Detail,
  },
  ~~ 계속
];
```

이런 식으로 계속 나누면 되겠지만, 상세페이지가 100만 개 필요하면 저것도 100만 개를 작성해야 한다. 그러면 늙어 죽기 때문에, 이를 해결할 **파라미터 문법**을 배워 본다.

### URL 파라미터 문법

```
(router.js)
const routes = [
  {
    path: '/detail/:id',
    component: Detail,
  },
];
```

- 주소창에 `/detail/` 뒤에 무엇을 작성하든 **항상 `Detail.vue`를 보여주세요**라는 뜻이다.
- 콜론 기호(`:`)를 붙이고 뒤에 아무 이름이나 작명하면 된다.

다만 문제는, `/detail/0`과 `/detail/1`로 접속해도 각각 다른 글이 아니라 똑같은 내용만 보인다는 점이다. 이를 해결하려면 하나만 더 알아보자.

## 심심할까봐 소개하는 Nested routes & push 함수

> **알아서 해볼 거**
> `/list` 페이지에서 글 제목을 누르면 각각 맞는 detail 페이지로 이동하게 코드를 짜보십시오. (예: 1번째 글 제목을 누르면 `/detail/1`로 이동)

오늘은 **nested routes(중첩 라우트)** 를 사용해 볼 것이다.

- `/detail/0/author`로 접속하면 detail 페이지 내에 작가 소개란을 보여주고,
- `/detail/0/comment`로 접속하면 detail 페이지 내에 댓글란을 보여주도록 한다.

특정 페이지 내에서 또 라우트를 나누는 경우인데, 이걸 nested routes라고 한다.

- 모달창 식 UI로 만들어도 똑같이 구현 가능하지만, 이렇게 다른 URL로 나눠 두면 **뒤로가기 / 앞으로가기 버튼이 동작**한다는 장점이 있다.

### 구현 방법

`router.js`에 다음과 같이 작성한다.

```
const routes = [
  {
    path : '/detail/:id',
    component : Detail,
    children : [
      { path : 'author', component : Author },
      { path : 'comment', component : Comment },
    ]
  }
]
```

- `children` 항목을 개설하면 route를 추가할 수 있다. 그러면 `/detail/:id/author`, `/detail/:id/comment` 경로로 접속했을 때 보여줄 컴포넌트를 작성할 수 있다.
- (당연히 `Author`, `Comment` 컴포넌트는 미리 만들어 import해서 넣어야 한다.)
- 마무리로 `Detail.vue`에 `<router-view></router-view>` 태그를 추가하면, `/detail/:id/author`로 접속했을 때 `<router-view>` 자리에 `Author` 컴포넌트가 보인다.
- 다만 간단한 UI라면 이렇게 만들지 말고 탭 기능 식으로 만드는 게 나을 수 있다.

### 라우터 개발 팁

- 라우터 관련 문법 에러는 터미널에 표기되지 않고, **크롬 개발자도구 콘솔창에 작게 warning으로 표기**된다. (warning이라서 사이트가 멈추지는 않는다.)
- 그러니 라우터 관련 에러는 콘솔창에서 확인하고 수정하자.

### push 함수로 페이지 이동

페이지 이동을 간지나게 하고 싶다면:

```
$router.push('/detail/0')
```

- 위 코드가 실행되면 `/detail/0`으로 이동한다.
- HTML 안에 넣거나, 기능 개발하는 곳에 넣을 수 있다.
- `router-link` 대신 쓸 수 있는데, 변수도 집어넣을 수 있다는 장점이 있다.

```
$router.go(-1)
```

- 위 코드는 **뒤로가기** 기능을 실행한다.
- `1`이라고 쓰면 앞으로 가고, `-2`라고 쓰면 두 번 뒤로 간다.
- `$router` 변수를 잘 쓰면 브라우저 방문 기록을 들춰보거나 마음대로 조작할 수 있다.

자세한 내용은 [Vue 라우터 4버전 공식 사이트](https://next.router.vuejs.org/)를 참고하자. 이런 건 외워봤자 다음날 다 까먹기 때문에, 자잘한 내용은 필요할 때 찾아 쓰는 게 효율적이다.

## 라우터 나머지 기능들 (hash mode, guards)

### 1. Hash mode vs HTML5 mode

우리가 맨 처음 셋팅해 놓은 코드는 이렇다.

```
import { createRouter, createWebHistory } from 'vue-router'

const router = [];
const router = createRouter({
  history: createWebHistory(),
  routes,
})
```

`history: createWebHistory()` 설정을 추가해 놨는데, 이게 문제가 되는 경우 다르게 설정할 수 있다.

```
import { createRouter, createWebHashHistory } from 'vue-router'

const router = [];
const router = createRouter({
  history: createWebHashHistory(),
  routes,
})
```

`history: createWebHashHistory()`로 설정하는 것을 **Hash mode**라고 한다.

- 이걸 선택하면 URL이 전부 `#`이 붙은 채로 시작한다.
- 예: `codingapple.com/#/detail`

**HTML5 mode를 선택한 경우**

- 누군가 `/detail`이라고 URL에 입력하면, 이는 "Vue router로 `/detail`을 보여주세요"가 아니라 **"서버에 `/detail` 페이지를 요청해주세요"** 라는 뜻이다.
- 그래서 Vue가 라우팅하기 전에 서버가 `/detail` 페이지를 보여주려 할 수 있고, 서버에 기능을 개발 안 했으니 404 페이지가 뜰 수 있다.
- 이를 방지하려면 서버에 "누가 `/어쩌구`로 접속하면 그냥 Vue에게 라우팅을 맡겨주세요"라고 미리 기능을 개발해야 한다.

**Hash mode를 선택한 경우**

- URL에 `#`이 붙은 채로 시작한다. 예: `codingapple.com/#/`가 메인페이지.
- `#`을 붙이는 이유는, URL에서 `#` 뒤의 내용은 절대 서버로 전달되지 않기 때문이다.
- 따라서 서버가 라우팅을 채가는 일을 방지하고, Vue router에게 온전히 라우팅을 맡길 수 있다.
- 서버가 없으면 `#`이 붙는 hash 라우터로 사이트를 만드는 것도 좋은 선택이다.

### 2. Navigation guards

특정 URL로 접속할 때 뭔가 코드를 실행하고 싶은 경우가 있다.

- 예: 마이페이지를 로그인한 사람만 보여주고 싶을 때, 라우팅하기 전에 "너 로그인했냐" 같은 코드를 실행해야 한다.
- 그럴 때 **navigation guard**를 쓰면 된다. 내비게이션 전에 실행할 수 있는 코드, 즉 hook 같은 것이라고 보면 된다.

`beforeEnter` 항목을 만들고 함수를 적어 주면 된다. 그 함수는 경우에 따라 다른 route를 반환하는 함수가 된다.

```
const routes = [
  {
    path: "/hello",
    component: HelloWorld,
    beforeEnter: ()=>{
      if (로그인했냐 == false) {
        return '/login'
      }
    }
  }
];
```

- `/hello`로 들어가기 전에 "로그인 여부 변수가 false면 `/login` 페이지로 보내기" 기능을 개발한 것이다.
- 실제 서비스라면 로그인한 사용자 정보는 보통 쿠키나 로컬스토리지에 저장되어 있으니, 그게 있는지 검사하면 된다.
- 다만 자바스크립트 변수·데이터는 콘솔창 입력으로 언제나 위조가 가능하므로, **서버도 마이페이지 내용을 보내기 전에 로그인 검증을 해야 한다.**

```
const routes = [
  {
    path: "/hello",
    component: HelloWorld,
    beforeEnter: (to, from)=>{
      return to.fullPath
    }
  }
];
```

- 파라미터는 두세 개 작명이 가능하다. 첫 파라미터는 **목적지 페이지(to)**, 둘째 파라미터는 **출발 페이지(from)** 이다.
- 이것들은 `$route` 변수와 똑같이 이용할 수 있다. (`to.fullPath`는 전체 경로, `to.params.id`는 id 파라미터)

반환값 규칙:

- `return false` → 라우팅 중단
- `return`이 없으면 → 원래 route인 `/hello`로 잘 이동시켜 준다.

### 3. 여러 개의 route에 같은 navigation guard 추가하기

`router` 변수에 `.beforeEach()`를 쓰면 된다. 내부엔 함수가 들어가는데 사용법은 아까와 똑같다.

```
const router = createRouter({ 어쩌구 })
router.beforeEach((to, from) => {
  //페이지 변경 전에 실행할 코드
})
```

- 라우팅 **전에** 실행하고 싶으면: `beforeEach()` 또는 `beforeResolve()`
- 라우팅 **후에** 실행하고 싶으면: `afterEach()`

### 4. Vue 컴포넌트 안에서도 navigation guard 사용 가능

vue 파일 안에서도 페이지 이동 시 코드를 실행할 수 있다. `created()`, `mounted()`와 비슷하게 활용 가능하다.

```
beforeRouteEnter(){}
beforeRouteUpdate(){}
  ```

- lifecycle hook 쓰는 위치에 적으면 된다.
- 파라미터는 두 개 입력 가능하며, 각각 목적지 `to`, 출발지 `from`을 의미한다.
- 특정 페이지로 접속했을 때 ajax 요청을 할 일이 있으면 여기에 쓰면 된다.

더 자세한 내용은 [vue router 4버전 페이지](https://next.router.vuejs.org/)를 참고하자.

## 만든 Vue 사이트 배포하기

만든 Vue 사이트를 build & GitHub Pages로 배포하는 방법이다. 간단한 내용이라 글로 빠르게 진행한다.

### 왜 build를 해야 하나

- Vue 사이트를 남들도 이용할 수 있게 배포하려면, 작업하던 `App.vue` 파일을 그대로 올리는 게 아니라 **build용 파일을 생성한 후 그걸 올려야** 한다.
- 웹브라우저는 HTML, CSS, JS 세 언어만 해석할 수 있고 `.vue` 파일은 읽을 줄 모르기 때문이다.
- build를 하면 브라우저 친화적인 HTML / CSS / JS 파일로 바꿔 준다. 그걸 웹에 올려야 사용자가 사이트를 볼 수 있다.

**Q. 웹서버를 가지고 있을 경우 배포는 어떻게 하나?**

- Vue는 HTML을 이쁘게 만들어 주는 툴일 뿐이다.
- 빌드를 하면 HTML 파일이 나오는데, 그걸 서버에서 전송하면 된다.
- 예: "누가 우리 사이트 메인페이지로 접속하면 빌드해 둔 HTML 파일을 보내주세요"라고 서버에 API를 짜면 끝이다.

### 1. Vue 프로젝트 build 하는 법

Vue 프로젝트에서 터미널을 켠 후 다음 중 하나를 입력한다.

```
npm run build
yarn build
```

- 개발 도중 심각한 에러가 있으면 build가 되지 않는다. 터미널 에러 / 콘솔창 심각한 에러가 있으면 먼저 수정하자.
- 그러면 프로젝트 폴더 내에 `dist` 폴더가 생성된다.

![build 후 생성된 dist 폴더](/assets/20220105_172620.png)

- `dist` 안에 `index.html`, css 파일, js 파일이 전부 담겨 있다.
- build하면 파일 사이즈를 크게 압축해 주고, 파일명도 무작위로 생성해 준다.
- 여기 안에 있는 내용을 모두 서버에 올리면 된다. 서버가 없으면 GitHub Pages를 쓰자.

### 2. 무료 호스팅 GitHub Pages에 올리기

GitHub Pages는 HTML/CSS/JS 파일을 무료로 호스팅해 주는 사이트다.

1. `github.com`에 들어가 로그인한다.
2. 우측 상단 `+` 버튼을 눌러 **New Repository**를 누른다.

![New Repository 메뉴 위치](/assets/20220105_172626.png)

3. 다음과 같이 입력한다.

![Repository 생성 입력 화면](/assets/20220105_172634.png)

- **Repository name**은 반드시 왼쪽에 뜨는 `여러분아이디.github.io` 형태로 입력해야 호스팅받을 수 있다. (임의로 설정하면 호스팅이 안 된다.)
- README 파일 생성도 체크한 뒤 생성하면 된다.

### 3. Repository에 파일 올리기

Repository 생성이 끝나면 자동으로 해당 repository로 들어가진다.

![파일 드래그 앤 드롭 화면](/assets/20220105_172640.png)

- `dist` 폴더 **안의 내용물**을 전부 드래그 앤 드롭한다. (`dist` 폴더 자체를 올리는 게 아니다.)
- 드래그 앤 드롭하고 초록 버튼까지 누르면 배포 끝이다.
- 10분 정도 후에 `https://여러분아이디.github.io`로 들어가면 사이트가 보인다.

### 4. "내아이디.github.io/서브경로"에 발행하고 싶은 경우

GitHub 문제이거나 뭔가 이상할 때도 이 방법을 써보자. 서브경로에 발행하는 방법이다.

**(1)** 아까 만든 것은 지우지 말고 repository를 하나 더 만든다. (아무 이름, 예: `vue`) README 파일도 체크한다.

![서브경로용 repository 생성](/assets/20220105_172647.png)

**(2)** 뷰 프로젝트 파일을 열어 `vue.config.js` 파일을 만들고 다음과 같이 작성한다.

![vue.config.js의 publicPath 설정](/assets/20220105_172655.png)

- `publicPath` 공간에 `/내가만든repository이름`을 적어 준다.
- 원래 Vue 프로젝트를 `apple.com/vue` 같은 서브경로에 배포하고 싶으면 `/vue`라고 서브경로를 적어야 한다.

**(3)** `npm run build`를 다시 한다.

**(4)** `dist` 안의 프로젝트 파일을 아까 만든 `vue` repository에 업로드한다. (드래그)

**(5)** repository 설정으로 들어가 `main`, `/` 경로로 GitHub Pages 발행하겠다고 저장한다.

![GitHub Pages 발행 설정](/assets/20220105_172704.png)

**(6)** 이제 `내아이디.github.io/vue`로 접속하면 Vue 프로젝트가 나온다. 안 보이면 `내아이디.github.io/vue/index.html`로 접속해 보자.

### 자주 묻는 질문(FAQ)

**Q1. 왜 사이트 주소로 접속했는데 404 페이지가 뜨죠?**

- 10분 더 기다려 본다.
- `Ctrl + Shift + R`로 새로고침해 본다.
- `님이발행한주소/index.html`까지 URL에 작성해 본다.
- `내아이디.github.io` repository를 만들 때 아이디를 잘못 적었을 수 있다. 대소문자까지 정확히 적어야 한다. (틀렸으면 새로 만들면 된다.)

**Q2. 왜 특정 URL을 입력하면 404 페이지가 뜨죠?**

- `어쩌구.github.io/detail/1` 같은 세부 페이지 URL을 주소창에 직접 입력하면 404가 날 수 있다.
- 주소창에 입력하는 건 서버에게 "`/detail/1` 페이지 주세요"라고 요청하는 행위이기 때문이다.
- 원래는 서버에서 "누가 `어쩌구.github.io/어쩌구`로 접속하면 Vue 메인페이지로 보내주세요"라고 개발해야 하는데, GitHub은 서버를 만질 수 없고 HTML 파일만 보여주는 곳이다.
- 그래서 사이트 메뉴에 페이지 이동 버튼을 잘 만들어 두거나, URL에 `#`이 붙는 **hash mode**를 라우터 코드에 쓰면 해결될 수 있다.

**Q3. 업데이트 사항이 생기면 배포를 또 어떻게 하죠?**

- build를 다시 하고 그 파일을 그대로 다시 업로드하면 된다.
- build할 때마다 CSS, JS 파일명이 `dist` 폴더 내에 무작위로 다시 생성되므로, 새로 배포할 때마다 방문자들은 새로운 CSS, JS 파일을 이용하게 된다.

**Q4. build할 때 압축시키지 말고 남기고 싶은 파일은?**

- 코드 짤 때 `./경로`로 첨부한 이미지, js 파일들은 전부 압축되고 이름이 변한다.
- 이름이 변하지 않게 하려면 `public` 폴더 안에 넣고 build해 본다. 그러면 build 후에도 그대로 루트 경로에 파일이 남아 있다.
- (개발 시 그런 파일을 이용하고 싶으면 `public` 폴더에 보관하고 `./`가 아닌 `/` 경로로 import한다.)

**Q5. 서버에 올렸는데 왜 이상한 페이지가 나오거나 일부 img, css 파일이 로드 안 되죠?**

- 기존 `내아이디.github.io`가 이미 있는 경우, 다시 올려버리면 GitHub Pages가 가끔 이상해진다.
- 새로 서브경로 repository를 만들어 올려 본다. (서브폴더 배포는 위의 설정을 따른다.)
- 그 전에 build할 때 에러가 없었는지 확인한다.
- 페이지가 안 나오면 크롬 개발자도구를 열어 `index.html`이 쓰고 있는 css, js, img 파일들의 경로가 제대로 되어 있는지 체크해 본다.
