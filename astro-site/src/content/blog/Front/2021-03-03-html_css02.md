## CSS 기본 구조

![CSS 선택자 예시](/assets/20210303_093450.png)

선택자는 HTML 안에서 스타일을 적용할 대상을 지정한다.

![CSS 선언 블록 예시](/assets/20210303_093533.png)

아래 예시에는 디자인 속성이 2가지 들어 있다.

```
.css(선택자). {margin:30px; color: #000;}

```

- `{}` 안은 **선언 블록**으로, 각각의 스타일을 선언한다(여기서는 2개).
- `margin`은 **속성**, `30px`은 **값**이다.

---

## CSS 적용 방식

### 1) 내부(Internal) 형식 - `<style>` 태그 사용

```
	<!DOCTYPE html>

	<html>
	<head>
		<meta charset="UTF-8">
		<title>Insert title here</title>
		<style type = "text/css">
		h1{
			color:#00cc00;	/*글자색*/
			font-family: consolas;	/*글자체*/
			font-size:50pt;	/*글자크기*/
		}	/*여기가 하나의 스타일*/
		h2{
			color:pink;
			font-family: 궁서체;
			font-size:20pt;
		}/*여기가 하나의 스타일*/
		</style>
	</head>
	<body>
		<h1> css test </h1>
		<h2> 내부 파일 형식</h2>
</body>
</html>

```

### 2) 외부(External) 형식 - 별도 CSS 파일 사용

`mystyle.css` 파일을 만들어 분리한다.

```
@charset "UTF-8";

h1{
	color:magenta;
}

ul{
	color:rgb(255,153,0);
	list-style-type:square;
}

```

![외부 CSS 적용 결과](/assets/20210303_101707.png)

- 외부에 있는 스타일이 내부 스타일에 의해 덮어쓰기가 되어버린다.
- 가끔 필요에 의해 링크 태그를 내부 스타일보다 뒤에 적는 경우가 있는데, 이럴 경우 스타일이 다르게 나올 수 있다.

`<link>` 태그 대신 `@import`로도 외부 파일을 불러올 수 있으며 결과는 동일하다.

```
<!-- <link rel = "stylesheet" type = "text/css" href = "./mystyle.css"> -->



<style type = "text/css">

@import url("./mystile.css"); /*외부파일형식*/

```

### 3) 인라인(Inline) 형식 - 태그 안에 직접 작성

```
	<h2 style = "color:pink;"> 세계 3대 미항 </h2>	<!-- inline형식 -->
```

태그 안에 바로 스타일을 넣는 방식을 **인라인 방식**이라 한다. 다만 인라인 방식은 **재사용성이 제일 떨어진다.**

## 구글 폰트(웹 폰트) 사용

![구글 폰트 목록](/assets/20210303_102920.png)

폰트 목록에서 `1 style`은 세부 스타일이 1개라는 뜻이다. 여기서 Akaya를 선택한다.

![Akaya 폰트 선택](/assets/20210303_103021.png)

![폰트 가져오기 클릭](/assets/20210303_103051.png)

그러면 앞에서 본 것처럼 2가지 방식(`<link>`, `@import`)이 제공된다.

![폰트 코드 복사](/assets/20210303_104033.png)

하나 더 찾아서 추가해 본다.

```
<!DOCTYPE html>
<html>
<head>
	<meta charset="UTF-8">
	<title>Insert title here</title>
	<link rel="preconnect" href="https://fonts.gstatic.com">
	<link href="https://fonts.googleapis.com/css2?family=Akaya+Telivigala&display=swap" rel="stylesheet">
	<link rel="preconnect" href="https://fonts.gstatic.com">
	<link href="https://fonts.googleapis.com/css2?family=Oi&display=swap" rel="stylesheet">

	<!-- 웹 폰트에서 가져옴(구글폰트) -->
	<style type = "text/css">
	 	p{
	 		font-family: 'Akaya Telivigala', cursive;	/* 글자체 따옴표로 묶은건 공백이 있어서 */
	 		color :magenta;
	 		font-size:30pt;
	 	}
	 	h2{
	 		font-family: 'Oi', cursive;
	 		color:green;
	 		font-size:20px;
	 	}
	 /* 	만약 처음 글자체 못 얻어오면 두번째 글자체 적용시키라는 뜻.(아까 h1,h2마냥 consolas가 없으면 궁서체 적용처럼) */
	</style>
</head>
<body>
	<h2>browser basic font</h2>
	<p>this is web font <p>
</body>
</html>
```

- 글자체에 공백이 있으면 따옴표로 묶는다.
- 콤마로 여러 글자체를 적으면, 첫 글자체를 못 얻어올 때 두 번째 글자체를 적용한다(앞의 `consolas` 없으면 궁서체 적용과 같은 원리).

## 클래스(class)와 아이디(id) 선택자

선택자로 태그 이름을 적지 않고, `.`(클래스)이나 `#`(아이디)으로 시작해 이름을 정하고 속성을 준다.

```
<!DOCTYPE html>
<html>
<head>
	<meta charset="UTF-8">
	<title>Insert title here</title>
	<!-- 	class/id:  동일한 스타일을 세분화 시켜 적용할 때 사용 -->
	<style type = "text/css">
		.blue{
			color:blue;
		}

		.green{
			color:green;
		}

		.red{
			color:red;
		}

		#pink{	/* #: 아이디 */
			color:pink;
		}

	</style>
</head>
<body>
<body>
	<h1 class = "blue">this is body the first h1 tag.</h1>
	<h1 class = "green">this is body the second h1 tag.</h1>
	<h1 class = "red">this is body the third h1 tag.</h1>
	<h1 class = "green">this is body the third h1 tag.</h1>
	<h1 id = "pink">this is body the third h1 tag.</h1>
	<p class = "blue"> hhhjsdf</p>

</body>
</body>
</html>
```

- `id`는 CSS가 적용되긴 하지만, 같은 `id`가 붙은 태그가 2개가 되면 문제가 생긴다.
- 나중에 자바스크립트로 아이디가 `pink`인 대상을 찾아 쓰려 할 때, 중복이면 원하는 대상을 고를 수 없게 된다.
- `id`는 CSS에 적용할 때도 `#`을 붙여서 사용한다.

```

h1 id = "pink"

```

`id`는 사실 식별자로 쓰려고 붙인 것이지 CSS 용도로 붙인 것은 아니다.

> **class/id : 동일한 스타일을 세분화시켜 적용할 때 사용.** 아무 클래스나 만들어 넣을 수 있다.

## 박스 모델(Box Model)

박스 모델은 HTML 태그를 둘러싼 사각형 영역을 의미한다.

- 콘텐츠 안쪽 여백 = **패딩(padding)**
- 그 바깥 경계선 = **보더(border)**
- 보더 바깥 여백 = **마진(margin)**

즉, border 기준으로 안쪽은 패딩, 바깥쪽은 마진이다.

![박스 모델 구조](/assets/20210303_112559.png)

보더를 기준으로 안쪽에 들어가는 부분이 패딩이다.

```
<!DOCTYPE html>

<html>
<head>
<meta charset="UTF-8">
<title>Insert title here</title>
<style type = "text/css">
	p{
		border-color:red;
		border-style:solid;
		border-width:2px; /* 두께 */
}
</style>
</head>
<body>
	<!--  box model : html tag를 둘러싼 주변의 사각형 -->
	<p>this is p tag.</p>
	<p>this is p tag.</p>
</body>
</html>


```

![보더 적용 결과](/assets/20210303_112935.png)

![페이지 검사 화면](/assets/20210303_113429_3cawuew77.png)

페이지 검사(개발자 도구)를 통해 각 패딩, 마진, 보더의 상세 정보를 확인할 수 있다.

![박스 모델 상세](/assets/20210303_113528.png)

- **마진은 공유해서 쓴다(margin collapse).** 마진을 30px과 50px로 줬다면, 더 큰 50px을 기준으로 띄운다.

![마진 병합](/assets/20210303_113948.png)

상하좌우 패딩을 다르게 주고 싶다면 다음처럼 적는다.

```
padding :20px 30px 40px 50px;	/*보더기준 안 쪽 여백 사방 지정 가능*/

```

```
<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>Insert title here</title>
<style type = "text/css">
	p{
		/*  border-color:red;
		border-style:solid;
		border-width:2px; */
		border : 2px blue dotted;
		padding :20px 30px 40px 50px;	/*보더기준 안 쪽 여백 사방 지정 가능*/
		margin :30px
	}
</style>
</head>
<body>
	<!--  box model : html tag를 둘러싼 주변의 사각형 -->
	<p>this is p tag.</p>
	<p>this is p tag.</p>
</body>
</html>

```

### border 형식 테스트

`solid`, `double`, `dotted`, `groove`, `ridge`, `inset`, `outset`, `dashed` 등 다양한 border 스타일을 테스트해 본다.

```
<!DOCTYPE html>
<html>
<head>
	<meta charset="EUC-KR">
	<title>Insert title here</title>
	<style type="text/css">

		.solid{
			border:  10px solid red ;

		}
		.double{
			border:  10px double blue;
		}

		.dotted{
			border:  10px dotted blue;
		}
		.groove{
			border:  10px groove blue;
		}
		.ridge{
			border:  10px ridge blue;
		}
		.inset{
			border:  10px inset blue;
		}
		.outset{
			border:  10px outset blue;
		}
		.dashed{
			border:  10px dashed blue;
		}

	</style>
</head>
<body>
	<p class="solid">solid border</p>
	<p class="double">double border</p>
	<p class="dotted">dotted border</p>
	<p class="groove">groove border</p>
	<p class="ridge">ridge border</p>
	<p class="inset">inset border</p>
	<p class="outset">outset border</p>
	<p class="dashed">dashed border</p>

</body>
</html>

```

## CSS 길이 단위

**상대적인 단위**

- `em` : 현재 사용 중인 폰트 크기를 1로 함
- `ex` : 문자 'x'의 크기를 1로 함
- `px` : 모니터의 화소("픽셀")에 대한 상대적인 크기

**절대적인 단위**

- `in` : inches, 1in = 2.54cm
- `cm` : centimeters
- `mm` : millimeters
- `pt` : points, 1pt = 1/72in
- `pc` : picas, 1pc = 12pt

## 블록 요소와 인라인 요소

엘리먼트(element)가 태그(tag)보다 더 큰 요소다.

### 1) 블록(block) 요소

- `h1`, `h2`, `p` 등 줄바꿈이 되는 태그
- `div`도 블록 요소이며, `span`은 기능은 `div`와 같지만 줄바꿈이 안 된다.

### 2) 인라인(inline) 요소

- 줄바꿈이 되지 않는 태그
- `span`, `a` 등이 있다.

각 블록 요소와 인라인 요소는 CSS `display` 속성으로 서로 바꿀 수 있다.

![블록/인라인 차이](/assets/20210303_143541.png)

- 위 메뉴는 `li` 안의 글자라 색이 잘 바뀌지만, 아래는 글자가 앵커(`a`) 태그 안에 소속되어 있어 색이 바뀌지 않는다.
- 아래를 바꾸려면 `ul li a` 태그를 대상으로 지정해야 한다.

```
ul li a {	/* 계층구조 따라 가면서 ul 태그 밑에 li 태그 밑에 a 태그 */
			text-decoration:none;
			color: orange;
		}
```

위처럼 넣어서 바꿔 본다.

![ul li a 적용 결과](/assets/20210303_143728.png)

`clear: both`는 이전에 지정한 `float` 속성을 모두 무효화시킨다.

## 시맨틱 태그(Semantic Tag)

`div` 대신 의미를 가진 시맨틱 태그(`section`, `article` 등)로 구역을 나눈다.

```
<section>	<!--  문서에서 구역을 나누는 역할의 태그  -->
		<article>	<!-- 기사(주제) 1건 -->
			<h2>본문1</h2>
			<div>
				<p>재산권의 행사는 공공복리에 적합하도록 하여야 한다. 정부는 회계연도마다 예산안을 편성하여 회계연도 개시
					90일전까지 국회에 제출하고, 국회는 회계연도 개시 30일전까지 이를 의결하여야 한다.</p>

				<p>대통령의 임기가 만료되는 때에는 임기만료 70일 내지 40일전에 후임자를 선거한다. 제2항과 제3항의 처분에
					대하여는 법원에 제소할 수 없다. 언론·출판에 대한 허가나 검열과 집회·결사에 대한 허가는 인정되지 아니한다.</p>
```

### 테이블 홀수/짝수 행 구분 - `nth-child`

```
tbody tr:nth-child(2n){	/*바디에 들어가는 부분 , 홀수 짝수행 구분해줘야.*/

	}
```

- `n`은 n번째 자식을 의미하며, `tr`의 몇 번째 자식인지 따져서 CSS를 적용하거나 적용하지 않는다.
- `n`이 0부터 시작한다고 생각하면 `2n`은 짝수로 올라간다.
- `tr`을 선택할 때 해당 내용을 적용시켜 올린다는 뜻이다.

> 실무에서는 이렇게 일일이 적용하기보다 라이브러리로 구현된 부분을 가져와 적용한다. 그래도 수정이나 구현을 하려면 어느 정도는 알아둬야 한다.
